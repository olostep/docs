Les parseurs sont un moyen de transformer des données non structurées en données structurées compatibles avec votre backend. En combinant les parseurs avec l'API Olostep (crawls, scrapes, batches), vous pouvez transformer n'importe quel site web en un appel API pour retourner le JSON dont vous avez besoin.

Les parseurs sont idéaux lorsque vous avez besoin de données à grande échelle de manière récurrente à partir des mêmes sites web. Cette approche est nettement plus économique et rapide comparée à [l'extraction LLM](/features/structured-content/llm-extraction) et ne retourne que le JSON requis.

Nous proposons des parseurs préconstruits pour des sites web et des cas d'utilisation populaires. Vous pouvez également créer vos propres parseurs en quelques minutes avec des LLMs en utilisant le [tableau de bord](https://www.olostep.com/dashboard/parsers) ou demander à notre équipe de le faire pour vous.

### Parseurs Préconstruits

Nous proposons plusieurs parseurs préconstruits pour des sites web populaires :
- Recherche Google : `@olostep/google-search`
- Produit Amazon : `@olostep/amazon-it-product`
- Extraction d'email : `@olostep/extract-emails`
- Extraction de calendriers : `@olostep/extract-calendars`
- Extraction de réseaux sociaux : `@olostep/extract-socials`
- Extraction de données TikTok : contactez-nous pour obtenir l'ID du parseur
- Google News : contactez-nous pour obtenir l'ID du parseur
- Google Maps : contactez-nous pour obtenir l'ID du parseur

### Exemple d'Utilisation

```python
import requests
import json

endpoint = "https://api.olostep.com/v1/scrapes"
payload = {
    "formats": ["json"],
    "parser": {"id": "@olostep/google-search"},
    "url_to_scrape": "https://www.google.com/search?q=alexander+the+great&gl=us&hl=en",
    "wait_before_scraping": 0,
}
headers = {
    "Authorization": "Bearer <YOUR-API-KEY>",
    "Content-Type": "application/json"
}

response = requests.request("POST", endpoint, json=payload, headers=headers)
print(json.dumps(response.json(), indent=4))
```

La réponse contient :

- **Informations de base sur la requête** : `id`, `object`, horodatage `created`, `url_to_scrape`
- **Objet résultat** avec des URLs pour accéder à différents formats de données
- **json_content** avec des résultats JSON structurés incluant :
- `searchParameters` : Informations sur la requête de recherche
- `knowledgeGraph` : Informations détaillées sur le sujet de recherche (lorsqu'elles sont disponibles)
- `organic` : Liste des résultats de recherche avec titre, lien, position et extrait
- `peopleAlsoAsk` : Questions connexes que les utilisateurs recherchent couramment
- `relatedSearches` : Requêtes de recherche suggérées connexes

`json_content` est la partie principale de la réponse avec les résultats JSON structurés. Tu peux accéder directement au contenu JSON depuis la réponse ou utiliser l'URL hébergée fournie dans la réponse.
