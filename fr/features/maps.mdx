Grâce au point de terminaison `/v1/maps` d'Olostep, tu peux obtenir tous les URLs d'un site web. Cela est utile pour la découverte de contenu, l'analyse de la structure du site (par exemple, SEO), ou pour décider quels URLs tu souhaites extraire ensuite.

- Obtenez tous les URLs d'un site web (y compris les sitemaps et les liens découverts)
- Utilisez des motifs spéciaux pour inclure/exclure des chemins (par exemple, `/blog/**`)
- Paginer les grandes réponses avec `cursor` (jusqu'à 10MB par réponse)
- Limiter le volume avec `top_n`

Pour les détails de l'API, consulte la [Référence API du point de terminaison Carte](/api-reference/maps/create).

## Utilisation

Envoie une requête POST avec l'`url` du site web. Optionnellement, passe `include_urls`, `exclude_urls` (motifs globaux), et `top_n`.

<CodeGroup>

```python Python
import requests
import json

endpoint = "https://api.olostep.com/v1/maps"

payload = {
    "url": "https://docs.olostep.com"
}
headers = {
    "Authorization": "Bearer <YOUR_API_KEY>",
    "Content-Type": "application/json"
}

response = requests.post(endpoint, json=payload, headers=headers)
print(json.dumps(response.json(), indent=2))
```

```js Node
const res = await fetch('https://api.olostep.com/v1/maps', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer <YOUR_API_KEY>',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ url: 'https://docs.olostep.com' })
})
console.log(await res.json())
```

```bash cURL
curl -s -X POST "https://api.olostep.com/v1/maps" \
  -H "Authorization: Bearer $OLOSTEP_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://docs.olostep.com"
  }'
```

</CodeGroup>

Le temps de réponse est généralement de quelques secondes mais peut prendre jusqu'à 120 secondes pour les sites web plus complexes. Il peut extraire tous les URLs d'un site web, y compris les backlinks et ceux non présents dans les Sitemaps. Tu peux également décider des chemins d'URLs que tu souhaites inclure ou exclure de la réponse.

Par défaut, le point de terminaison renvoie environ 100k URLs en un seul appel (10MB max). Si la réponse inclut plus de données, l'API renvoie un paramètre `cursor` qui peut être utilisé pour la pagination et obtenir les URLs suivantes. Pour plus de détails, consulte la [Référence API](/api-reference/maps/create#body-cursor).

Ce point de terminaison est particulièrement utile lorsque tu as besoin de :
- Découvrir toutes les pages de contenu d'un site web
- Analyser la structure et la hiérarchie du site
- Préparer les URLs pour un traitement par lots
- Décider quels URLs spécifiques extraire

Pour un contrôle plus précis sur les URLs retournés, tu peux utiliser les paramètres `include_urls` et `exclude_urls`.

### Exemple

Disons que depuis www.brex.com tu veux extraire tous les urls qui ont les chemins après `/product/` par exemple `https://www.brex.com/product/api/no-code` mais aussi inclure `www.brex.com/product`.
Tu peux utiliser le code suivant :

<CodeGroup>

    ```python maps.py
    import requests
    
    endpoint = "https://api.olostep.com/v1/maps"
    
    payload = {
        "url": "https://www.brex.com/",
        "include_urls": ["/product", "/product/**"],
        "top_n": 100000
    }
    
    headers = {
        "Authorization": "Bearer <YOUR_API_KEY>",
        "Content-Type": "application/json"
    }
    
    response = requests.post(endpoint, json=payload, headers=headers)
    
    print(response.text)
    ```

    ```js Node
    const endpoint = 'https://api.olostep.com/v1/maps'

    const res = await fetch(endpoint, {
      method: 'POST',
      headers: {
        'Authorization': 'Bearer <YOUR_API_KEY>',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        url: 'https://www.brex.com/',
        include_urls: ['/product', '/product/**'],
        top_n: 100000
      })
    })
    console.log(await res.text())
    ```

    ```bash cURL
    curl -s -X POST "https://api.olostep.com/v1/maps" \
      -H "Authorization: Bearer $OLOSTEP_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "url": "https://www.brex.com/",
        "include_urls": ["/product", "/product/**"],
        "top_n": 100000
      }'
    ```

</CodeGroup>

## Conclusion

Le point de terminaison des cartes est un outil puissant pour la découverte de contenu et l'analyse de site. Il fournit une liste complète des URLs d'un site web, te permettant d'extraire du contenu de pages spécifiques ou d'analyser la structure du site. Ce point de terminaison est particulièrement utile pour les professionnels du SEO, les spécialistes du marketing de contenu, les agents IA qui ont besoin d'analyser le contenu ou la structure d'un site web.
