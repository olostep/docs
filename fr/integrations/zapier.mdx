L'intégration Olostep Zapier apporte des capacités puissantes de scraping web à l'écosystème de plus de 8 000 applications de Zapier. Créez des flux de travail automatisés qui extraient, surveillent et traitent les données web sans écrire de code.

[Commencez avec Olostep sur Zapier →](https://zapier.com/apps/olostep/integrations)

## Installation

### 1. Trouver Olostep dans Zapier

Recherchez "Olostep" dans le répertoire d'applications Zapier ou lors de la création d'un nouveau Zap :

1. Allez sur [Olostep sur Zapier](https://zapier.com/apps/olostep/integrations)
2. Cliquez sur "Créer un Zap"
3. Recherchez "Olostep" dans le sélecteur d'applications
4. Sélectionnez l'application Olostep

Alternativement, visitez directement la [page d'intégration Olostep](https://zapier.com/apps/olostep/integrations) pour parcourir les actions disponibles et créer votre premier Zap.

### 2. Connectez votre compte

Lorsque vous utilisez Olostep pour la première fois dans un Zap, vous serez invité à connecter votre compte :

1. Cliquez sur "Se connecter à Olostep"
2. Entrez votre clé API Olostep
3. Cliquez sur "Oui, continuer vers Olostep"

Obtenez votre clé API depuis le [Tableau de bord Olostep](https://olostep.com/dashboard).

## Actions disponibles

### Scraper un site web

Extraire le contenu d'une URL unique. Prend en charge plusieurs formats et le rendu JavaScript.

**Cas d'utilisation :**
- Surveiller des pages spécifiques pour des changements
- Extraire des informations produit de sites e-commerce
- Rassembler des données d'articles de presse ou de blogs
- Récupérer du contenu pour l'agrégation de contenu

**Configuration :**

<ParamField path="URL à scraper" type="string" required>
  URL du site web à scraper (doit inclure http:// ou https://)
</ParamField>

<ParamField path="Format de sortie" type="dropdown" default="Markdown">
  Choisissez le format : Markdown, HTML, JSON ou Texte brut
</ParamField>

<ParamField path="Code pays" type="string">
  Code pays pour le contenu spécifique à la localisation (ex. "US", "GB", "CA")
</ParamField>

<ParamField path="Attendre avant de scraper" type="integer">
  Temps d'attente en millisecondes pour le rendu JavaScript (0-10000)
</ParamField>

<ParamField path="Parseur" type="string">
  ID de parseur optionnel pour une extraction spécialisée (ex. "@olostep/amazon-product")
</ParamField>

**Champs de sortie :**
- ID de scrape
- URL scrapée
- Contenu Markdown
- Contenu HTML
- Contenu JSON
- Contenu texte
- Statut
- Horodatage
- URL de capture d'écran (si disponible)
- Métadonnées de la page

**Exemples de flux de travail :**

<AccordionGroup>
  <Accordion title="Surveiller les prix des concurrents">
    **Déclencheur :** Programmation (Tous les jours à 9h)
    
    **Action :** Olostep - Scraper un site web
    - URL : Page produit du concurrent
    - Format : JSON
    - Parseur : @olostep/amazon-product
    
    **Action :** Google Sheets - Créer une ligne
    - Ajouter les données de prix au tableau de suivi
    
    **Action :** Gmail - Envoyer un email (Si le prix baisse)
    - Alerter l'équipe des changements de prix
  </Accordion>

  <Accordion title="Extraire et sauvegarder des articles de blog">
    **Déclencheur :** RSS par Zapier - Nouvel élément dans le flux
    
    **Action :** Olostep - Scraper un site web
    - URL : \{\{Post URL\}\}
    - Format : Markdown
    
    **Action :** Notion - Créer une page
    - Sauvegarder le contenu de l'article dans la base de données Notion
  </Accordion>

  <Accordion title="Enrichissement de leads">
    **Déclencheur :** Google Sheets - Nouvelle ligne
    
    **Action :** Olostep - Scraper un site web
    - URL : Site web de l'entreprise depuis la feuille
    - Format : Markdown
    
    **Action :** OpenAI - Compléter le texte
    - Extraire des informations sur l'entreprise à l'aide de l'IA
    
    **Action :** Google Sheets - Mettre à jour la ligne
    - Ajouter les données enrichies à la feuille
  </Accordion>
</AccordionGroup>

### Scraper des URLs en lot

Traitez plusieurs URLs en parallèle (jusqu'à 100 000 à la fois). Parfait pour l'extraction de données à grande échelle.

**Cas d'utilisation :**
- Scraper des catalogues de produits entiers
- Extraire des données de plusieurs résultats de recherche
- Traiter des listes d'URLs depuis des feuilles de calcul
- Extraction de contenu en masse

**Configuration :**

<ParamField path="URLs à scraper" type="text" required>
  Tableau JSON d'objets avec les champs url et custom_id. 
  
  Exemple : `[{"url":"https://example.com","custom_id":"site1"}]`
</ParamField>

<ParamField path="Format de sortie" type="dropdown" default="Markdown">
  Choisissez le format pour toutes les URLs : Markdown, HTML, JSON ou Texte brut
</ParamField>

<ParamField path="Code pays" type="string">
  Code pays pour le scraping spécifique à la localisation
</ParamField>

<ParamField path="Attendre avant de scraper" type="integer">
  Temps d'attente en millisecondes pour le rendu JavaScript
</ParamField>

<ParamField path="Parseur" type="string">
  ID de parseur optionnel pour une extraction spécialisée
</ParamField>

**Champs de sortie :**
- ID de lot (utilisez ceci pour récupérer les résultats plus tard)
- Statut
- Total des URLs
- Créé à
- Format demandé
- Code pays
- Parseur utilisé

**Exemples de flux de travail :**

<AccordionGroup>
  <Accordion title="Scraper un catalogue de produits">
    **Déclencheur :** Webhook - Recevoir une requête POST
    
    **Action :** Code par Zapier - Exécuter Python
    - Convertir CSV/liste en format tableau JSON
    
    **Action :** Olostep - Scraper des URLs en lot
    - URLs : \{\{tableau JSON de l'étape précédente\}\}
    - Format : JSON
    - Parseur : @olostep/amazon-product
    
    **Action :** Webhook - POST
    - Envoyer l'ID de lot à votre système pour récupération
  </Accordion>

  <Accordion title="Surveillance quotidienne de contenu">
    **Déclencheur :** Programmation - Tous les jours à 6h
    
    **Action :** Google Sheets - Obtenir des lignes
    - Récupérer les URLs à surveiller
    
    **Action :** Code par Zapier - Formater les URLs
    - Convertir en format de tableau de lot
    
    **Action :** Olostep - Scraper des URLs en lot
    - Traiter toutes les URLs en une fois
    
    **Action :** Slack - Envoyer un message
    - Notifier l'équipe que le scraping est terminé
  </Accordion>
</AccordionGroup>

### Créer un crawl

Découvrez et scrapez de manière autonome des sites web entiers en suivant les liens. Parfait pour les sites de documentation, les blogs et les dépôts de contenu.

**Cas d'utilisation :**
- Crawler et archiver des sites de documentation entiers
- Extraire tous les articles de blog d'un site web
- Construire des bases de connaissances à partir de contenu web
- Surveiller les changements de structure de site

**Configuration :**

<ParamField path="URL de départ" type="string" required>
  URL de départ pour le crawl (doit inclure http:// ou https://)
</ParamField>

<ParamField path="Nombre maximum de pages" type="integer" default="10">
  Nombre maximum de pages à crawler
</ParamField>

<ParamField path="Suivre les liens" type="boolean" default="true">
  Indique si les liens trouvés sur les pages doivent être suivis
</ParamField>

<ParamField path="Format de sortie" type="dropdown" default="Markdown">
  Format pour le contenu scrapé
</ParamField>

<ParamField path="Code pays" type="string">
  Code pays optionnel pour le crawling spécifique à la localisation
</ParamField>

<ParamField path="Parseur" type="string">
  ID de parseur optionnel pour une extraction de contenu spécialisée
</ParamField>

**Champs de sortie :**
- ID de crawl (utilisez ceci pour récupérer les résultats plus tard)
- Type d'objet
- Statut
- URL de départ
- Nombre maximum de pages
- Suivre les liens
- Horodatage de création
- Formats

**Exemples de flux de travail :**

<AccordionGroup>
  <Accordion title="Archiver un site de documentation">
    **Déclencheur :** Programmation - Mensuel le 1er à 0h
    
    **Action :** Olostep - Créer un crawl
    - URL de départ : https://docs.example.com
    - Pages max : 500
    - Suivre les liens : true
    - Format : Markdown
    
    **Action :** Webhook - POST
    - Envoyer l'ID de crawl à votre système d'archivage
    
    **Action :** Slack - Envoyer un message
    - Notifier l'équipe que le crawl est en cours
  </Accordion>

  <Accordion title="Analyse de contenu concurrentiel">
    **Déclencheur :** Programmation - Hebdomadaire le lundi à 9h
    
    **Action :** Olostep - Créer un crawl
    - URL de départ : URL du blog concurrent
    - Pages max : 100
    - Format : Markdown
    
    **Action :** Délai - Pour 10 minutes
    - Attendre la fin du crawl
    
    **Action :** Airtable - Créer des enregistrements
    - Stocker les données de crawl pour analyse
  </Accordion>
</AccordionGroup>

### Créer une carte

Extrayez toutes les URLs d'un site web pour la découverte de contenu et l'analyse de la structure du site.

**Cas d'utilisation :**
- Construire des sitemaps et des diagrammes de structure de site
- Découvrir toutes les pages avant un scraping en lot
- Trouver des pages cassées ou manquantes
- Audits et analyses SEO

**Configuration :**

<ParamField path="URL du site web" type="string" required>
  URL du site web pour extraire les liens (doit inclure http:// ou https://)
</ParamField>

<ParamField path="Requête de recherche" type="string">
  Requête de recherche optionnelle pour filtrer les URLs (ex. "blog")
</ParamField>

<ParamField path="Top N URLs" type="integer">
  Limiter le nombre d'URLs retournées
</ParamField>

<ParamField path="Inclure des motifs d'URL" type="string">
  Motifs globaux pour inclure des chemins spécifiques (ex. "/blog/**")
</ParamField>

<ParamField path="Exclure des motifs d'URL" type="string">
  Motifs globaux pour exclure des chemins spécifiques (ex. "/admin/**")
</ParamField>

**Champs de sortie :**
- ID de carte
- Type d'objet
- URL du site web
- Total des URLs trouvées
- URLs (tableau JSON)
- Requête de recherche
- Limite Top N

**Exemples de flux de travail :**

<AccordionGroup>
  <Accordion title="Découvrir et scraper">
    **Déclencheur :** Clic sur un bouton dans Zapier
    
    **Action :** Olostep - Créer une carte
    - URL : https://example.com
    - Motifs d'inclusion : /products/**
    - Top N : 500
    
    **Action :** Code par Zapier - Extraire les URLs
    - Analyser les URLs du résultat de la carte
    
    **Action :** Olostep - Scraper des URLs en lot
    - URLs : \{\{URLs de la carte\}\}
    - Format : JSON
    
    **Action :** Google Sheets - Créer des lignes
    - Ajouter toutes les données produit au tableau
  </Accordion>

  <Accordion title="Audit SEO du site">
    **Déclencheur :** Programmation - Mensuel
    
    **Action :** Olostep - Créer une carte
    - URL : Votre site web
    - Top N : 1000
    
    **Action :** Airtable - Créer des enregistrements
    - Stocker toutes les URLs pour suivi
    
    **Action :** Slack - Envoyer un message
    - Rapporter le total des pages trouvées
  </Accordion>
</AccordionGroup>

### Demander une réponse IA

Obtenez des réponses assistées par l'IA aux questions en utilisant la recherche web ou vos URLs de contexte fournies. Parfait pour l'automatisation de la recherche, la génération de contenu et l'analyse de données.

**Cas d'utilisation :**
- Automatisation de la recherche avec citations
- Résumer le contenu de plusieurs URLs
- Générer des rapports avec des références de sources
- Répondre à des questions en utilisant des pages web spécifiques
- Analyse et extraction de contenu

**Configuration :**

<ParamField path="Question" type="string" required>
  La question à laquelle vous souhaitez que Olostep Answers réponde
</ParamField>

<ParamField path="URLs de contexte (tableau JSON)" type="string">
  Tableau JSON optionnel d'URLs pour ancrer la réponse (ex. ["https://example.com/page1", "https://example.com/page2"])
</ParamField>

<ParamField path="Requête de recherche" type="string">
  Requête optionnelle pour récupérer des sources sur le web si aucune URL de contexte n'est fournie (ex. "site:example.com pricing")
</ParamField>

<ParamField path="Nombre de sources" type="integer">
  Limiter le nombre de sources utilisées pour générer la réponse
</ParamField>

<ParamField path="Format" type="dropdown" default="Markdown">
  Choisissez le format de la réponse : Markdown, JSON ou Texte brut
</ParamField>

<ParamField path="Inclure des citations" type="boolean" default="true">
  Indique si les citations/sources doivent être incluses dans la réponse
</ParamField>

**Champs de sortie :**
- ID de réponse
- Type d'objet
- Question
- Réponse (Texte)
- Réponse (Markdown)
- Réponse (JSON)
- Citations (tableau JSON avec URLs, titres, extraits)
- Horodatage de création
- Format
- URLs de contexte
- Requête de recherche
- Nombre de sources

**Exemples de flux de travail :**

  <Accordion title="Résumé de contenu à partir d'URLs">
    **Déclencheur :** Google Sheets - Nouvelle ligne (avec des URLs)
    
    **Action :** Olostep - Demander une réponse IA
    - Question : "Résumez les points clés de ces pages"
    - URLs de contexte : \{\{URLs depuis la feuille\}\}
    - Format : Markdown
    - Inclure des citations : true
    
    **Action :** Notion - Créer une page
    - Sauvegarder le résumé avec les liens sources
  </Accordion>

  <Accordion title="Intelligence concurrentielle">

## Flux de travail multi-étapes

### Pipeline complet de scraping de produits

Construisez un pipeline complet de données produit :

<Steps>
  <Step title="Découvrir les URLs de produits">

  </Step>
  
  <Step title="Surveiller les changements">
    Planifiez des scrapes quotidiens pour suivre les changements de prix/disponibilité

    - Alertez sur les changements significatifs
  </Step>
</Steps>

  </Step>
</Steps>

## Bonnes pratiques

<AccordionGroup>
  <Accordion title="Utiliser le traitement en lot pour plusieurs URLs">
    Lorsque vous scrapez plus de 3-5 URLs, utilisez **Scraper des URLs en lot** au lieu de plusieurs actions **Scraper un site web**. Le traitement en lot est :
    - Beaucoup plus rapide (traitement parallèle)
    - Plus rentable
    - Plus facile à gérer
    - Meilleur pour les limites de taux
  </Accordion>

  <Accordion title="Définir des temps d'attente appropriés">
    Pour les sites lourds en JavaScript, utilisez le paramètre "Attendre avant de scraper" :
    - Sites simples : 0-1000ms
    - Sites dynamiques : 2000-3000ms
    - JavaScript lourd : 5000-8000ms
    
    Testez avec différentes valeurs pour trouver le temps d'attente optimal.
  </Accordion>

  <Accordion title="Utiliser des parseurs spécialisés">
    Pour les sites web populaires (Amazon, LinkedIn, Google), utilisez des parseurs préconstruits :
    - Obtenez des données structurées automatiquement
    - Extraction plus fiable
    - Pas besoin de parsing personnalisé
    - Maintenu par Olostep
  </Accordion>

  <Accordion title="Filtrer avant de scraper">
    Utilisez l'action Filtre de Zapier pour éviter les scrapes inutiles :
    - Vérifiez si l'URL a changé
    - Vérifiez que les données n'ont pas été scrapées récemment
    - Appliquez une logique métier avant de scraper
    
    Cela économise des crédits API et du temps d'exécution.
  </Accordion>

  <Accordion title="Gérer les opérations asynchrones">
    Les opérations de lot, de crawl et de carte sont asynchrones :
    - Stockez l'ID retourné (batch_id, crawl_id, map_id)
    - Utilisez une action Délai si vous récupérez immédiatement
    - Envisagez des rappels webhook pour la complétion
    - Configurez des Zaps séparés pour la récupération
  </Accordion>

  <Accordion title="Stocker correctement les résultats">
    Choisissez le bon stockage en fonction de vos besoins :
    - **Google Sheets** : Suivi simple, collaboration d'équipe
    - **Airtable** : Données relationnelles, formatage riche
    - **Base de données** : Grande échelle, requêtes complexes
    - **Notion** : Base de connaissances, documentation
  </Accordion>

    - Utilisez les chemins d'erreur dans les Zaps
    - Envoyez des alertes à Slack/Email en cas d'échec
    - Suivez l'utilisation de l'API dans le tableau de bord Olostep
    - Enregistrez les métriques importantes
  </Accordion>

## Dépannage

  <Accordion title="Échec de l'authentification">
    **Erreur** : "Clé API invalide"
    
    **Solutions** :
    - Vérifiez la clé API depuis le [tableau de bord](https://olostep.com/dashboard)
    - Assurez-vous qu'il n'y a pas d'espaces supplémentaires dans la clé API
    - Reconnectez votre compte Olostep dans Zapier
    - Vérifiez que la clé API est active
  </Accordion>

  <Accordion title="Le scrape renvoie un contenu vide">
    **Erreur** : Les champs de contenu sont vides
    
    **Solutions** :
    - Augmentez le temps "Attendre avant de scraper"
    - Vérifiez si le site web nécessite une connexion
    - Essayez un format différent (HTML vs Markdown)
    - Vérifiez que l'URL est accessible
    - Vérifiez si le site bloque l'accès automatisé
  </Accordion>

  <Accordion title="Erreur de format de tableau de lot">
    **Erreur** : "Format JSON invalide pour le tableau de lot"
    
    **Solutions** :
    - Utilisez le format : `[{"url":"https://example.com","custom_id":"id1"}]`
    - Assurez-vous de la syntaxe JSON correcte
    - Utilisez une étape Code pour formater correctement les URLs
    - Testez le JSON avec un validateur en ligne
  </Accordion>

  <Accordion title="Limite de taux dépassée">
    **Erreur** : "Limite de taux dépassée"
    
    **Solutions** :
    - Espacer les exécutions de Zap avec des délais
    - Utilisez le traitement en lot au lieu de scrapes individuels
    - Mettez à niveau votre plan Olostep
    - Vérifiez la limite de taux dans le tableau de bord
  </Accordion>

  <Accordion title="URL non scrapée">
    **Erreur** : Des URLs spécifiques échouent à scraper
    
    **Solutions** :
    - Vérifiez le format de l'URL (inclure http:// ou https://)
    - Vérifiez si l'URL nécessite une authentification
    - Testez l'URL dans un navigateur d'abord
    - Essayez avec le paramètre de pays
    - Contactez le support pour les domaines bloqués
  </Accordion>
</AccordionGroup>

## Limitations de Zapier & Solutions

### Limites de tâches
Zapier a des limites de tâches basées sur votre plan. Chaque action Olostep compte pour 1 tâche.

**Solution** : Utilisez le traitement en lot pour scraper plusieurs URLs en une seule tâche.

### Temps d'exécution
Les Zaps expirent après 30 secondes. Les crawls et les grands lots peuvent prendre plus de temps.

**Solution** : Stockez l'ID et récupérez les résultats dans un Zap séparé ou utilisez des webhooks.

### Taille des données
Zapier a des limites sur la taille des données par tâche.

**Solution** : Utilisez des URLs hébergées dans la sortie pour récupérer le contenu volumineux séparément.

### Polling vs Instantané
Les déclencheurs sont basés sur le polling (vérification toutes les 5-15 minutes).

**Solution** : Utilisez des webhooks pour des notifications instantanées ou programmez à des moments spécifiques.
