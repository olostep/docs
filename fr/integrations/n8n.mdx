Le [nœud Olostep Web Scraper](https://n8n.io/integrations/olostep-web-scraper/) vérifié vous offre six opérations dans le constructeur visuel de n8n : scraper une URL, rechercher sur le web, obtenir des réponses IA, scraper par lots des milliers d'URLs, crawler un site ou cartographier tous ses liens.

[Voir sur n8n →](https://n8n.io/integrations/olostep-web-scraper/)

## Avant de commencer

- **Un compte Olostep avec une clé API :** [obtenez-en une gratuitement](https://olostep.com/dashboard), pas de carte de crédit requise. Vos 500 premiers crédits sont inclus.
- **n8n en cours d'exécution :** soit [n8n Cloud](https://n8n.io/cloud/) soit une instance auto-hébergée. Les nœuds communautaires doivent être activés (ils le sont par défaut sur la plupart des configurations).
- **Aucune programmation requise :** tout dans ce guide se fait via l'éditeur visuel de n8n.

---

## Configuration

<Steps>
  <Step title="Recherchez le nœud Olostep">
    Ouvrez n'importe quel workflow, cliquez sur **+**, et recherchez **Olostep**. Sélectionnez **Olostep Web Scraper** dans les résultats.

    ![Recherchez Olostep dans le sélecteur de nœuds n8n](/images/integrations/n8n/step-1.png)
  </Step>

  <Step title="Installez le nœud">
    Cliquez sur le résultat pour ouvrir le panneau de détails du nœud, puis cliquez sur **Installer le nœud**. n8n installera `n8n-nodes-olostep` et vous demandera de redémarrer. Faites-le avant de continuer.

    ![Détails du nœud Olostep Web Scraper avec le bouton Installer le nœud](/images/integrations/n8n/step-2.png)

    <Note>
      Si **Nœuds Communautaires** est désactivé pour votre espace de travail, un administrateur doit l'activer d'abord. Consultez le [guide des nœuds communautaires n8n](https://docs.n8n.io/integrations/community-nodes/installation/).
    </Note>
  </Step>

  <Step title="Ajoutez votre clé API">
  Ouvrez le nœud Olostep dans votre workflow, cliquez sur **Configurer l'identifiant** (dans l'onglet Paramètres), ajoutez votre clé API, et cliquez sur **Enregistrer**.

    ![Formulaire d'identifiants Olostep dans n8n avec le champ Clé API](/images/integrations/n8n/step-3.png)

    Obtenez votre clé depuis le [tableau de bord Olostep →](https://olostep.com/dashboard)
  </Step>

  <Step title="Connectez-le et exécutez">
    Connectez le nœud Olostep à un déclencheur et à toutes les étapes en aval, puis exécutez votre workflow.

    ![Canvas de workflow n8n avec le déclencheur de planification connecté au nœud Olostep](/images/integrations/n8n/step-4.png)
  </Step>
</Steps>

## Actions

<CardGroup cols={2}>
  <Card title="Scraper un site web" icon="file-lines">
    Extraire le contenu de n'importe quelle URL en tant que Markdown, HTML, JSON ou texte brut. Gère les pages rendues en JS avec des temps d'attente optionnels et un ciblage par pays.
  </Card>

  <Card title="Rechercher" icon="magnifying-glass">
    Effectuer une recherche web et obtenir des résultats structurés (titres, URLs et extraits) en JSON.
  </Card>

  <Card title="Réponses (IA)" icon="sparkles">
    Poser une question en langage naturel et obtenir une réponse avec des sources citées. Utile avant les nœuds LLM lorsque vous avez besoin de réponses fondées.
  </Card>

  <Card title="Scraper des URLs par lots" icon="layer-group">
    Soumettre jusqu'à 10 000 URLs en un seul travail, traitées en parallèle. Retourne un `batch_id`; récupérez les résultats de manière asynchrone.
  </Card>

  <Card title="Créer un crawl" icon="spider-web">
    Partir d'une URL, suivre les liens et scraper toutes les sous-pages. Idéal pour les sites de documentation, les blogs ou l'ingestion complète de sites. Retourne un `crawl_id`.
  </Card>

  <Card title="Créer une carte" icon="map">
    Obtenez chaque URL sur un site sans scraper le contenu. Utilisez-le pour la découverte avant un travail par lots. Retourne un `map_id`.
  </Card>
</CardGroup>

<Note>
  **Batch, Crawl, et Map sont asynchrones.** Stockez l'ID retourné et utilisez un nœud d'attente ou un second workflow pour récupérer les résultats une fois le traitement terminé.
</Note>

## Exemple de workflow : enrichissement de prospects depuis Google Sheets

**Ce qu'il fait :** Lorsque vous collez une URL d'entreprise dans une feuille Google, ce workflow scrape automatiquement le site web de l'entreprise, extrait les informations clés avec un nœud IA, et écrit les résultats dans la même ligne, transformant une feuille de calcul vide en une base de données de prospects remplie.

**Nœuds utilisés :** Déclencheur Google Sheets → Olostep Scrape Website → OpenAI → Code → Mise à jour Google Sheets

![Workflow d'enrichissement de prospects dans n8n : déclencheur Google Sheets connecté aux nœuds Olostep, OpenAI, Code et mise à jour Google Sheets](/images/integrations/n8n/workflow-example.png)

---

### Étape 1 : Configurez votre feuille Google

Créez une feuille avec ces colonnes : `Company URL`, `Industry`, `Description`, `Company Size`, `Enriched`. Le workflow lit depuis `Company URL` et remplit le reste.

### Étape 2 : Ajoutez un déclencheur Google Sheets

Dans n8n, ajoutez un nœud de déclencheur **Google Sheets**. Réglez l'événement sur **Row Added**, pointez-le vers votre feuille, et configurez-le pour surveiller la colonne `Company URL`. Maintenant, chaque fois que vous collez une nouvelle URL dans la feuille, ce workflow se déclenche.

### Étape 3 : Ajoutez Olostep Scrape Website

Connectez un nœud **Olostep Web Scraper** après le déclencheur. Configurez :
- **Action :** Scraper un site web
- **URL :** `{{ $json["Company URL"] }}` (récupère l'URL de la nouvelle ligne)
- **Format de sortie :** Markdown

Markdown fonctionne mieux ici car il élimine la navigation, les publicités et les modèles. Le nœud IA à l'étape suivante obtient un texte propre sur l'entreprise au lieu de bruit HTML brut.

### Étape 4 : Ajoutez un nœud OpenAI

Connectez un nœud **OpenAI**. Réglez le modèle sur `gpt-4o-mini` (rapide et économique pour les tâches d'extraction) et utilisez cette invite :

```
Vous êtes un chercheur commercial. Basé sur le contenu du site web de l'entreprise ci-dessous, extrayez :
1. Industrie (une phrase, par ex. "B2B SaaS", "E-commerce", "Santé")
2. Description de l'entreprise en une phrase (max 20 mots)
3. Taille estimée de l'entreprise (Startup / PME / Mid-market / Entreprise)

Retournez uniquement un objet JSON avec les clés : industry, description, company_size.

Contenu du site web :
{{ $json.markdownContent }}
```

Le champ `markdownContent` est ce que Olostep retourne du scrape, sous forme de texte brut propre.

### Étape 5 : Analysez la réponse IA et écrivez-la

Ajoutez un nœud **Code** pour analyser le JSON d'OpenAI :

```js
const parsed = JSON.parse($input.first().json.message.content);
return [{ json: parsed }];
```

Puis connectez un nœud **Google Sheets** réglé sur **Update Row**. Mappez les colonnes :
- `Industry` → `{{ $json.industry }}`
- `Description` → `{{ $json.description }}`
- `Company Size` → `{{ $json.company_size }}`
- `Enriched` → `Yes`

### Ce que vous obtenez

Collez une URL comme `https://notion.so` dans votre feuille, et en ~10 secondes la ligne se remplit :

| Company URL | Industry | Description | Company Size | Enriched |
|---|---|---|---|---|
| https://notion.so | Productivity SaaS | Espace de travail tout-en-un pour notes, docs et bases de données | Mid-market | Yes |

À partir de là, vous pouvez étendre ce workflow : ajoutez une notification Slack lorsque l'enrichissement est terminé, filtrez par industrie avant d'écrire, ou remplacez Google Sheets par HubSpot pour mettre à jour directement les contacts.

## Parseurs

Ajoutez un ID de parseur au champ **Parser** sur toute action Scrape ou Batch pour obtenir des données structurées au lieu de contenu brut :

| Parser | Extraits |
|--------|----------|
| `@olostep/amazon-product` | Titre, prix, note, avis, images, variantes |
| `@olostep/google-search` | Titres des résultats, URLs, extraits |
| `@olostep/google-maps` | Nom de l'entreprise, adresse, note, avis |
| `@olostep/extract-emails` | Adresses email de n'importe quelle page |
| `@olostep/extract-socials` | Liens de profils sociaux (X, GitHub, LinkedIn, etc.) |
| `@olostep/extract-calendars` | Liens Google Calendar et ICS |

Voir la liste complète dans le [store de parseurs Olostep →](https://www.olostep.com/store)

## Dépannage

<AccordionGroup>
  <Accordion title="Clé API rejetée">
    Copiez la clé directement depuis [olostep.com/dashboard](https://olostep.com/dashboard) sans espaces de fin. Supprimez et recréez l'identifiant dans n8n si l'erreur persiste.
  </Accordion>

  <Accordion title="Contenu scrappé vide">
    Augmentez **Attendre avant de scraper** (essayez 2000–5000ms pour les pages lourdes en JS). Confirmez que l'URL est accessible publiquement sans connexion. Si un domaine spécifique échoue constamment, contactez [info@olostep.com](mailto:info@olostep.com).
  </Accordion>

  <Accordion title="Erreur de format d'URL par lot">
    Le champ **URLs à scraper** attend un tableau JSON :
    ```json
    [
      { "url": "https://example.com/page-1", "custom_id": "p1" },
      { "url": "https://example.com/page-2", "custom_id": "p2" }
    ]
    ```
    Utilisez un nœud Code en amont pour construire ce tableau à partir de vos données si nécessaire.
  </Accordion>

  <Accordion title="Limite de débit atteinte">
    Ajoutez un nœud **Attendre** entre les étapes de scraping, ou passez à **Scraper des URLs par lots** au lieu de boucler des scrapes uniques. Vérifiez l'utilisation actuelle dans le [tableau de bord](https://olostep.com/dashboard).
  </Accordion>

  <Accordion title="Nœuds communautaires non visibles dans les paramètres">
    Sur n8n Cloud, les nœuds communautaires doivent être activés par un propriétaire d'espace de travail. Sur auto-hébergé, assurez-vous que `N8N_COMMUNITY_PACKAGES_ENABLED=true` est défini dans votre environnement. Voir le [guide d'installation de n8n](https://docs.n8n.io/integrations/community-nodes/installation/).
  </Accordion>
</AccordionGroup>
