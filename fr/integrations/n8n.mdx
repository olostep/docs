L'intégration Olostep n8n apporte des capacités puissantes de recherche web, de scraping et de crawling aux flux de travail n8n. Construisez des flux de travail automatisés qui recherchent, extraient et structurent des données web à partir de n'importe quel site sans écrire de code.

[Commencez avec Olostep sur n8n →](https://www.npmjs.com/package/n8n-nodes-olostep)

## Installation

### 1. Installer le Node

Installez le package de node Olostep via npm :

```bash
npm install n8n-nodes-olostep
```

Ensuite, redémarrez n8n pour charger le nouveau node.

Alternativement, suivez le [guide d'installation des nodes communautaires de n8n](https://docs.n8n.io/integrations/community-nodes/installation/) pour des instructions détaillées.

### 2. Connectez votre compte

Lorsque vous utilisez pour la première fois le node Olostep dans un flux de travail, vous devrez configurer les identifiants :

1. Ajoutez le node "Olostep Scrape" à votre flux de travail
2. Cliquez sur le node pour ouvrir ses paramètres
3. Cliquez sur "Créer un nouvel identifiant" ou sélectionnez un identifiant existant
4. Entrez votre clé API Olostep
5. Cliquez sur "Enregistrer" pour stocker l'identifiant

Obtenez votre clé API depuis le [Tableau de bord Olostep](https://olostep.com/dashboard).

## Actions disponibles

### Scraper un site web

Extraire le contenu d'une URL unique. Prend en charge plusieurs formats et le rendu JavaScript.

**Cas d'utilisation :**
- Surveiller les changements de pages spécifiques
- Extraire des informations produit de sites e-commerce
- Rassembler des données d'articles de presse ou de blogs
- Extraire du contenu pour l'agrégation de contenu

**Configuration :**

<ParamField path="URL à scraper" type="string" required>
  URL du site web à scraper (doit inclure http:// ou https://)
</ParamField>

<ParamField path="Format de sortie" type="dropdown" default="Markdown">
  Choisissez le format : Markdown, HTML, JSON ou Texte brut
</ParamField>

<ParamField path="Code pays" type="string">
  Code pays pour le contenu spécifique à un lieu (ex : "US", "GB", "CA")
</ParamField>

<ParamField path="Attendre avant de scraper" type="integer">
  Temps d'attente en millisecondes pour le rendu JavaScript (0-10000)
</ParamField>

<ParamField path="Parseur" type="string">
  ID de parseur optionnel pour une extraction spécialisée (ex : "@olostep/amazon-product")
</ParamField>

**Champs de sortie :**
- ID de scrape
- URL scrappée
- Contenu Markdown
- Contenu HTML
- Contenu JSON
- Contenu texte
- Statut
- Horodatage
- URL de capture d'écran (si disponible)
- Métadonnées de la page

**Exemples de flux de travail :**

<AccordionGroup>
  <Accordion title="Surveiller les prix des concurrents">
    **Déclencheur :** Planification (Tous les jours à 9h)
    
    **Action :** Olostep - Scraper un site web
    - URL : Page produit du concurrent
    - Format : JSON
    - Parseur : @olostep/amazon-product
    
    **Action :** Google Sheets - Ajouter une ligne
    - Ajouter les données de prix au tableau de suivi
    
    **Action :** Email - Envoyer un email (si le prix baisse)
    - Alerter l'équipe des changements de prix
  </Accordion>

  <Accordion title="Extraire et enregistrer des articles de blog">
    **Déclencheur :** Flux RSS - Nouvel élément
    
    **Action :** Olostep - Scraper un site web
    - URL : \{\{$json.link\}\}
    - Format : Markdown
    
    **Action :** Notion - Créer une page
    - Enregistrer le contenu de l'article dans la base de données Notion
  </Accordion>

  <Accordion title="Enrichissement de prospects">
    **Déclencheur :** Google Sheets - Nouvelle ligne
    
    **Action :** Olostep - Scraper un site web
    - URL : Site web de l'entreprise depuis la feuille
    - Format : Markdown
    
    **Action :** OpenAI - Compléter le texte
    - Extraire des informations sur l'entreprise à l'aide de l'IA
    
    **Action :** Google Sheets - Mettre à jour la ligne
    - Ajouter les données enrichies à la feuille
  </Accordion>
</AccordionGroup>

### Recherche

Recherchez sur le Web pour une requête donnée et obtenez des résultats structurés (résultats de recherche basés sur un parseur, non IA).

**Cas d'utilisation :**
- Flux de travail de recherche automatisée
- Découverte et enrichissement de prospects
- Analyse concurrentielle
- Recherche de contenu

**Configuration :**

<ParamField path="Requête" type="string" required>
  Requête de recherche
</ParamField>

**Sortie :** Renvoie des résultats de recherche structurés sous forme de JSON avec titres, URLs, extraits et métadonnées.

**Exemples de flux de travail :**

<AccordionGroup>
  <Accordion title="Recherche automatisée">
    **Déclencheur :** Planification (Tous les jours à 8h)
    
    **Action :** Olostep - Recherche
    - Requête : "derniers développements en IA"
    
    **Action :** Code - Traiter les résultats
    - Extraire et formater les informations clés
    
    **Action :** Notion - Créer des pages
    - Stocker les résultats de recherche
  </Accordion>

  <Accordion title="Découverte de prospects">
    **Déclencheur :** Manuel (Bouton)
    
    **Action :** Olostep - Recherche
    - Requête : "\{\{$json.searchTerm\}\}"
    
    **Action :** Airtable - Créer des enregistrements
    - Stocker les prospects avec les informations de contact
  </Accordion>
</AccordionGroup>

### Scraper des URLs en lot

Scraper jusqu'à 10k URLs en même temps. Parfait pour l'extraction de données à grande échelle.

**Cas d'utilisation :**
- Scraper des catalogues de produits entiers
- Extraire des données de plusieurs résultats de recherche
- Traiter des listes d'URLs à partir de feuilles de calcul
- Extraction de contenu en masse

**Configuration :**

<ParamField path="URLs à scraper" type="text" required>
  Tableau JSON d'objets avec les champs url et custom_id. 
  
  Exemple : `[{"url":"https://example.com","custom_id":"site1"}]`
</ParamField>

<ParamField path="Format de sortie" type="dropdown" default="Markdown">
  Choisissez le format pour toutes les URLs : Markdown, HTML, JSON ou Texte brut
</ParamField>

<ParamField path="Code pays" type="string">
  Code pays pour le scraping spécifique à un lieu
</ParamField>

<ParamField path="Attendre avant de scraper" type="integer">
  Temps d'attente en millisecondes pour le rendu JavaScript
</ParamField>

<ParamField path="Parseur" type="string">
  ID de parseur optionnel pour une extraction spécialisée
</ParamField>

**Champs de sortie :**
- ID de lot (utilisez ceci pour récupérer les résultats plus tard)
- Statut
- Total des URLs
- Créé à
- Format demandé
- Code pays
- Parseur utilisé

**Exemples de flux de travail :**

<AccordionGroup>
  <Accordion title="Scraper un catalogue de produits">
    **Déclencheur :** Webhook - Recevoir une requête POST
    
    **Action :** Code - Formater les URLs
    - Convertir CSV/liste en format tableau JSON
    
    **Action :** Olostep - Scraper des URLs en lot
    - URLs : \{\{$json.urlArray\}\}
    - Format : JSON
    - Parseur : @olostep/amazon-product
    
    **Action :** Webhook - POST
    - Envoyer l'ID de lot à votre système pour récupération
  </Accordion>

  <Accordion title="Surveillance quotidienne de contenu">
    **Déclencheur :** Planification - Tous les jours à 6h
    
    **Action :** Google Sheets - Lire les lignes
    - Récupérer les URLs à surveiller
    
    **Action :** Code - Formater les URLs
    - Convertir en format tableau de lot
    
    **Action :** Olostep - Scraper des URLs en lot
    - Traiter toutes les URLs en une fois
    
    **Action :** Slack - Envoyer un message
    - Notifier l'équipe que le scraping est terminé
  </Accordion>
</AccordionGroup>

### Créer un Crawl

Obtenez le contenu des sous-pages d'une URL. Découvrez et scrapez de manière autonome des sites entiers en suivant les liens. Parfait pour les sites de documentation, les blogs et les dépôts de contenu.

**Cas d'utilisation :**
- Crawler et archiver des sites de documentation entiers
- Extraire tous les articles de blog d'un site web
- Construire des bases de connaissances à partir de contenu web
- Surveiller les changements de structure de site

**Configuration :**

<ParamField path="URL de départ" type="string" required>
  URL de départ pour le crawl (doit inclure http:// ou https://)
</ParamField>

<ParamField path="Nombre maximum de pages" type="integer" default="10">
  Nombre maximum de pages à crawler
</ParamField>

<ParamField path="Suivre les liens" type="boolean" default="true">
  Indique si les liens trouvés sur les pages doivent être suivis
</ParamField>

<ParamField path="Format de sortie" type="dropdown" default="Markdown">
  Format pour le contenu scrappé
</ParamField>

<ParamField path="Code pays" type="string">
  Code pays optionnel pour le crawling spécifique à un lieu
</ParamField>

<ParamField path="Parseur" type="string">
  ID de parseur optionnel pour une extraction de contenu spécialisée
</ParamField>

**Champs de sortie :**
- ID de crawl (utilisez ceci pour récupérer les résultats plus tard)
- Type d'objet
- Statut
- URL de départ
- Nombre maximum de pages
- Suivre les liens
- Horodatage de création
- Formats

**Exemples de flux de travail :**

<AccordionGroup>
  <Accordion title="Archiver un site de documentation">
    **Déclencheur :** Planification - Mensuel le 1er à minuit
    
    **Action :** Olostep - Créer un Crawl
    - URL de départ : https://docs.example.com
    - Pages max : 500
    - Suivre les liens : true
    - Format : Markdown
    
    **Action :** Webhook - POST
    - Envoyer l'ID de crawl à votre système d'archivage
    
    **Action :** Slack - Envoyer un message
    - Notifier l'équipe que le crawl est en cours
  </Accordion>

  <Accordion title="Analyse de contenu concurrent">
    **Déclencheur :** Planification - Hebdomadaire le lundi à 9h
    
    **Action :** Olostep - Créer un Crawl
    - URL de départ : URL du blog du concurrent
    - Pages max : 100
    - Format : Markdown
    
    **Action :** Attendre - Pendant 10 minutes
    - Attendre que le crawl soit terminé
    
    **Action :** Airtable - Créer des enregistrements
    - Stocker les données de crawl pour analyse
  </Accordion>
</AccordionGroup>

### Créer une Carte

Obtenez toutes les URLs d'un site web. Extraire toutes les URLs d'un site pour la découverte de contenu et l'analyse de la structure du site.

**Cas d'utilisation :**
- Construire des sitemaps et des diagrammes de structure de site
- Découvrir toutes les pages avant le scraping en lot
- Trouver des pages cassées ou manquantes
- Audits et analyses SEO

**Configuration :**

<ParamField path="URL du site web" type="string" required>
  URL du site web pour extraire les liens (doit inclure http:// ou https://)
</ParamField>

<ParamField path="Requête de recherche" type="string">
  Requête de recherche optionnelle pour filtrer les URLs (ex : "blog")
</ParamField>

<ParamField path="Top N URLs" type="integer">
  Limiter le nombre d'URLs retournées
</ParamField>

<ParamField path="Inclure les motifs d'URL" type="string">
  Motifs globaux pour inclure des chemins spécifiques (ex : "/blog/**")
</ParamField>

<ParamField path="Exclure les motifs d'URL" type="string">
  Motifs globaux pour exclure des chemins spécifiques (ex : "/admin/**")
</ParamField>

**Champs de sortie :**
- ID de carte
- Type d'objet
- URL du site web
- Total des URLs trouvées
- URLs (tableau JSON)
- Requête de recherche
- Limite Top N

**Exemples de flux de travail :**

<AccordionGroup>
  <Accordion title="Découvrir et scraper">
    **Déclencheur :** Manuel (Bouton)
    
    **Action :** Olostep - Créer une Carte
    - URL : https://example.com
    - Inclure les motifs : /products/**
    - Top N : 500
    
    **Action :** Code - Extraire les URLs
    - Analyser les URLs à partir du résultat de la carte
    
    **Action :** Olostep - Scraper des URLs en lot
    - URLs : \{\{$json.urls\}\}
    - Format : JSON
    
    **Action :** Google Sheets - Ajouter des lignes
    - Ajouter toutes les données produit à la feuille de calcul
  </Accordion>

  <Accordion title="Audit SEO du site">
    **Déclencheur :** Planification - Mensuel
    
    **Action :** Olostep - Créer une Carte
    - URL : Votre site web
    - Top N : 1000
    
    **Action :** Airtable - Créer des enregistrements
    - Stocker toutes les URLs pour le suivi
    
    **Action :** Slack - Envoyer un message
    - Rapporter le total des pages trouvées
  </Accordion>
</AccordionGroup>

## Flux de travail multi-étapes

### Pipeline complet de scraping de produits

Construisez un pipeline complet de données produit :

<Steps>
  <Step title="Découvrir les URLs de produits">
    Utilisez **Créer une Carte** pour trouver toutes les pages produit sur le site cible
    - Inclure les motifs : `/products/**`
    - Exclure les motifs : `/cart/**`, `/checkout/**`
  </Step>
  
  <Step title="Traiter les produits en lot">
    Utilisez **Scraper des URLs en lot** pour extraire toutes les données produit
    - Format : JSON
    - Parseur : Parseur spécifique au produit si disponible
  </Step>
  
  <Step title="Stocker dans la base de données">
    Envoyez l'ID de lot à votre système ou attendez et récupérez les résultats
    - Utilisez Airtable, Google Sheets ou votre base de données
  </Step>
  
  <Step title="Surveiller les changements">
    Planifiez des scrapes quotidiens pour suivre les changements de prix/disponibilité
    - Comparez avec les données existantes
    - Alertez sur les changements significatifs
  </Step>
</Steps>

### Stratégie de contenu SEO

<Steps>
  <Step title="Cartographier les sites concurrents">

    - Extraire tous les articles de blog et pages de contenu
  </Step>
  
  <Step title="Scraper le contenu">
    Utilisez **Scraper des URLs en lot** pour obtenir le contenu complet
    - Format : Markdown pour une analyse facile
  </Step>
  
  <Step title="Analyse par IA">
    Utilisez OpenAI pour analyser les sujets et mots-clés
    - Identifier les lacunes de contenu
    - Trouver les sujets tendance
  </Step>

    - Planifiez votre stratégie de contenu
  </Step>
</Steps>

Ajoutez simplement l'ID du parseur au champ Parseur. Deux exemples :

```
Action : Olostep - Scraper un site web

  - Format : JSON
  - Parseur : @olostep/amazon-product

```
Action : Olostep - Scraper un site web

Le parseur extrait automatiquement les données structurées spécifiques à la tâche.

## Bonnes pratiques

<AccordionGroup>
  <Accordion title="Utiliser le traitement par lot pour plusieurs URLs">
    Lors du scraping de plus de 3-5 URLs, utilisez **Scraper des URLs en lot** au lieu de plusieurs actions **Scraper un site web**. Le traitement par lot est :
    - Beaucoup plus rapide (traitement parallèle)
    - Plus économique
    - Plus facile à gérer
    - Meilleur pour les limites de taux
  </Accordion>

  <Accordion title="Définir des temps d'attente appropriés">
    Pour les sites lourds en JavaScript, utilisez le paramètre "Attendre avant de scraper" :
    - Sites simples : 0-1000ms
    - Sites dynamiques : 2000-3000ms
    - JavaScript lourd : 5000-8000ms
    
    Testez avec différentes valeurs pour trouver le temps d'attente optimal.
  </Accordion>

  <Accordion title="Utiliser des parseurs spécialisés">
    Utilisez des parseurs pré-construits (ex : Amazon, Google, et parseurs spécifiques aux tâches du Store Olostep comme emails, réseaux sociaux, calendriers) :
    - Obtenez des données structurées automatiquement
    - Extraction plus fiable
    - Pas besoin de parsing personnalisé
    - Maintenu par Olostep
  </Accordion>

  <Accordion title="Filtrer avant de scraper">
    Utilisez le nœud IF de n8n pour éviter les scrapes inutiles :
    - Vérifiez si l'URL a changé
    - Vérifiez que les données n'ont pas été scrappées récemment
    - Appliquez la logique métier avant de scraper
    
    Cela économise des crédits API et du temps d'exécution.
  </Accordion>

  <Accordion title="Gérer les opérations asynchrones">
    Les opérations Batch, Crawl et Map sont asynchrones :
    - Stockez l'ID retourné (batch_id, crawl_id, map_id)
    - Utilisez un nœud Wait si vous récupérez immédiatement
    - Envisagez des rappels webhook pour la complétion
    - Configurez des flux de travail séparés pour la récupération
  </Accordion>

  <Accordion title="Stocker correctement les résultats">
    Choisissez le bon stockage en fonction de vos besoins :
    - **Google Sheets** : Suivi simple, collaboration d'équipe
    - **Airtable** : Données relationnelles, formatage riche
    - **Base de données** : Grande échelle, requêtes complexes
    - **Notion** : Base de connaissances, documentation

## Dépannage

<AccordionGroup>
  <Accordion title="Échec de l'authentification">
    **Erreur** : "Clé API invalide"
    
    **Solutions** :
    - Vérifiez la clé API depuis le [tableau de bord](https://olostep.com/dashboard)
    - Assurez-vous qu'il n'y a pas d'espaces supplémentaires dans la clé API
    - Recréez l'identifiant dans n8n
    - Vérifiez que la clé API est active
  </Accordion>

  <Accordion title="Le scrape renvoie un contenu vide">
    **Erreur** : Les champs de contenu sont vides
    
    **Solutions** :
    - Augmentez le temps "Attendre avant de scraper"
    - Vérifiez si le site web nécessite une connexion
    - Essayez un format différent (HTML vs Markdown)
    - Vérifiez que l'URL est accessible
    - Vérifiez si le site bloque l'accès automatisé
  </Accordion>

  <Accordion title="Erreur de format de tableau de lot">
    **Erreur** : "Format JSON invalide pour le tableau de lot"
    
    **Solutions** :
    - Utilisez le format : `[{"url":"https://example.com","custom_id":"id1"}]`
    - Assurez-vous de la syntaxe JSON correcte
    - Utilisez le nœud Code pour formater correctement les URLs
    - Testez le JSON avec un validateur en ligne
  </Accordion>

  <Accordion title="Limite de taux dépassée">
    **Erreur** : "Limite de taux dépassée"
    
    **Solutions** :
    - Espacer les exécutions de flux de travail avec des nœuds Wait
    - Utilisez le traitement par lot au lieu de scrapes individuels
    - Mettez à niveau votre plan Olostep
    - Vérifiez la limite de taux dans le tableau de bord
  </Accordion>

  <Accordion title="URL non scrappée">
    **Erreur** : Certaines URLs échouent à être scrappées
    
    **Solutions** :
    - Vérifiez le format de l'URL (inclure http:// ou https://)
    - Vérifiez si l'URL nécessite une authentification
    - Testez l'URL dans le navigateur d'abord
    - Essayez avec le paramètre de pays
    - Contactez le support pour les domaines bloqués
  </Accordion>
</AccordionGroup>

## Avantages de n8n

### Auto-hébergé
n8n est auto-hébergé, vous donnant un contrôle total sur vos flux de travail et vos données. Pas de verrouillage fournisseur, pas de données quittant votre infrastructure.

### Pas de limites de tâches
Contrairement aux plateformes d'automatisation basées sur le cloud, n8n n'impose pas de limites de tâches. Exécutez autant de flux de travail que vous le souhaitez sans coûts supplémentaires.

### Open Source
n8n est open source, vous permettant de le personnaliser et de l'étendre pour répondre à vos besoins spécifiques.

### Rentable
n8n auto-hébergé est gratuit, avec un hébergement cloud optionnel disponible. Payez uniquement pour l'utilisation de l'API Olostep.

## Tarification

Olostep facture en fonction de l'utilisation de l'API, indépendamment de n8n :

- **Scrapes** : Payez par scrape
- **Lots** : Payez par URL dans le lot
- **Crawls** : Payez par page crawlée
- **Cartes** : Payez par opération de carte

Consultez la tarification actuelle sur [olostep.com/pricing](https://olostep.com/pricing).

**n8n** : n8n auto-hébergé est gratuit. Hébergement cloud disponible avec des plans payants optionnels.
