L’application vérifiée [Olostep sur Relay](https://www.relay.app/apps/olostep/integrations) vous offre deux actions dans le constructeur visuel de Relay : scraper une URL ou mapper tous les liens sur un site.

[Voir sur Relay →](https://www.relay.app/apps/olostep/integrations)

## Avant de commencer

- **Un compte Olostep avec une clé API :** [obtenez-en une gratuitement](https://olostep.com/dashboard), aucune carte de crédit requise. Vos 500 premiers crédits sont inclus.
- **Un compte Relay :** créez-en un sur [relay.app](https://www.relay.app/).
- **Aucun codage requis :** tout dans ce guide se fait via l’éditeur visuel de Relay.

---

## Configuration

<Steps>
  <Step title="Trouver Olostep dans Relay">
    Ouvrez un workflow dans Relay, ajoutez une étape, et recherchez **Olostep**. Sélectionnez **Olostep** dans la liste des applications.

    ![Placeholder: Rechercher Olostep dans le sélecteur d'applications Relay](/images/integrations/relay/step-1.png)
  </Step>

  <Step title="Choisir une action">
    Après avoir sélectionné **Olostep**, choisissez l'une des actions Olostep disponibles dans la configuration de l'étape.

    ![Placeholder: Sélecteur d'action Olostep dans Relay](/images/integrations/relay/step-2.png)
  </Step>

  <Step title="Connecter votre compte">
    Cliquez sur **Connecter le compte**, collez votre clé API Olostep et autorisez la connexion. Relay enregistrera ce compte pour les futurs workflows.

    ![Placeholder: Modal de connexion de compte Relay avec champ de clé API Olostep](/images/integrations/relay/step-3.png)

    Obtenez votre clé depuis le [tableau de bord Olostep →](https://olostep.com/dashboard)
  </Step>

  <Step title="Configurer Scrape Website (URL + format)">
    - **URL à Scraper :** mappez depuis une étape précédente ou entrez manuellement
    - **Format de Sortie :** choisissez `Markdown`, `HTML`, `JSON`, ou `Texte`

    ![Placeholder: Champs de Scrape Website Olostep dans Relay avec URL et format de sortie](/images/integrations/relay/step-4.png)
  </Step>

  <Step title="Tester et publier">
    Exécutez un test pour vérifier la sortie, puis publiez ou activez le workflow une fois que les résultats semblent corrects.

    ![Placeholder: Sortie de test Relay et flux de publication](/images/integrations/relay/step-5.png)
  </Step>
</Steps>

## Actions

<CardGroup cols={2}>
  <Card title="Scrape Website" icon="file-lines">
    Extraire le contenu de n’importe quelle URL sous forme de Markdown, HTML, JSON, ou texte brut. Gère les pages rendues en JS avec des temps d'attente optionnels et un ciblage par pays.
  </Card>

  <Card title="Create Map" icon="map">
    Obtenez chaque URL sur un site sans scraper le contenu. Utilisez-le pour la découverte avant une tâche par lots. Retourne un `map_id`.
  </Card>
</CardGroup>

<Note>
  **Map est asynchrone.** Stockez le `map_id` retourné et utilisez un modèle de délai/sondage dans Relay pour récupérer les résultats une fois le traitement terminé.
</Note>

## Exemple de workflow : Scraping programmé de la page d'un concurrent

**Ce qu'il fait :** Selon un calendrier, ce workflow scrappe une page de concurrent et stocke un contenu propre que vous pouvez réutiliser dans les étapes suivantes de Relay.

**Nœuds utilisés :** Schedule -> Olostep Scrape Website

![Placeholder: Canvas Relay montrant les étapes de planification et de scraping Olostep](/images/integrations/relay/workflow-example.png)

---

### Étape 1 : Ajouter un déclencheur de planification

Créez un workflow et ajoutez un déclencheur de planification (par exemple, chaque jour de la semaine à 8h).

### Étape 2 : Ajouter Olostep Scrape Website

Ajoutez une étape **Olostep** et sélectionnez **Scrape Website**. Définissez :
- **URL :** `https://competitor.com/blog`
- **Format de Sortie :** Markdown

### Étape 3 : Tester et publier

Exécutez un test pour confirmer la sortie, puis publiez ou activez le workflow.

### Ce que vous obtenez

Chaque exécution produit un contenu de page propre prêt pour l'analyse ou le routage :

> **Résultat du Scraping**
> - URL source
> - Contenu récupéré dans le format sélectionné
> - Sortie de l'exécution horodatée dans Relay

## Parseurs

Ajoutez un ID de parseur dans le champ **Parser** de l’action Scrape pour obtenir des données structurées au lieu de contenu brut :

| Parseur | Extraits |
|--------|----------|
| `@olostep/amazon-product` | Titre, prix, note, avis, images, variantes |
| `@olostep/google-search` | Titres des résultats, URLs, extraits |
| `@olostep/google-maps` | Nom de l'entreprise, adresse, note, avis |
| `@olostep/extract-emails` | Adresses email de n'importe quelle page |
| `@olostep/extract-socials` | Liens de profils sociaux (X, GitHub, LinkedIn, etc.) |
| `@olostep/extract-calendars` | Liens Google Calendar et ICS |

Voir la liste complète dans le [store de parseurs Olostep →](https://www.olostep.com/store)

## Relay.app vs Zapier

Relay.app est une alternative solide à Zapier pour les workflows qui nécessitent une révision intégrée et une orchestration axée sur l'IA.

**Avantages de Relay.app :**
- **Humain dans la boucle :** étapes natives d'approbation et de révision dans le workflow
- **Conception de workflow axée sur l'IA :** plus facile d'ajouter des décisions et résumés IA en tant qu'étapes de premier ordre
- **Clarté du workflow :** constructeur visuel propre avec contexte d'exécution clair et sorties d'étape

**Quand Zapier peut être meilleur :**
- Vous avez besoin d'une couverture pour une application de niche uniquement disponible sur Zapier
- Votre équipe a déjà de nombreux Zaps existants et des outils opérationnels autour de Zapier

Si les deux plateformes prennent en charge votre pile, choisissez Relay lorsque la révision + la collaboration IA sont centrales à votre processus.

## Dépannage

<AccordionGroup>
  <Accordion title="Clé API rejetée">
    Copiez la clé directement depuis [olostep.com/dashboard](https://olostep.com/dashboard) sans espaces de fin. Reconnectez le compte Olostep dans Relay si l'erreur persiste.
  </Accordion>

  <Accordion title="Contenu scrappé vide">
    Augmentez **Attendre avant de scraper** (essayez 2000–5000ms pour les pages lourdes en JS). Confirmez que l'URL est publiquement accessible sans connexion. Si un domaine échoue systématiquement, contactez [info@olostep.com](mailto:info@olostep.com).
  </Accordion>

  <Accordion title="Limite de taux atteinte">
    Ajoutez des délais entre les exécutions intensives en scraping et réessayez avec un backoff pour les grands workflows. Vérifiez l'utilisation actuelle dans le [tableau de bord](https://olostep.com/dashboard).
  </Accordion>

  <Accordion title="L'étape de mappage prend plus de temps que prévu">
    Les maps sont asynchrones. Stockez d'abord le `map_id` retourné, puis récupérez les résultats dans une étape/exécution ultérieure une fois le traitement terminé.
  </Accordion>
</AccordionGroup>
