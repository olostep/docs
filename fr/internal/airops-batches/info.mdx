title: "Obtenir le statut du lot"

openapi: "GET /v1/batches-airops/{batch_id}"
---

## Aperçu

Retourne la phase actuelle et les métriques de progression pour un lot AirOps. Utilisez ceci pour le sondage si vous n'utilisez pas les webhooks.

## Champs de Réponse

<ResponseField name="id" type="string">
  L'identifiant du lot.
</ResponseField>

<ResponseField name="object" type="string">
  Toujours `"batch"`.
</ResponseField>

<ResponseField name="created_at" type="integer">
  Timestamp Unix (millisecondes) lorsque le lot a été créé.
</ResponseField>

<ResponseField name="total_urls" type="integer">
  Nombre total d'URLs soumises.
</ResponseField>

<ResponseField name="status" type="string">
  Statut global du workflow : `pending`, `in_progress`, `completed`, ou `failed`.
</ResponseField>

<ResponseField name="batch_mode" type="boolean">
  Si le mode lot LLM asynchrone est activé (`true`) ou le mode synchrone (`false`).
</ResponseField>

<ResponseField name="scraping" type="object">
  Objet de statut complet de la phase de scraping (voir [Webhooks](/internal/airops-batches/webhooks#scraping-status)).
</ResponseField>

<ResponseField name="analyzing" type="object">
  Objet de statut complet de la phase d'analyse (voir [Webhooks](/internal/airops-batches/webhooks#analyzing-status)).
</ResponseField>

<ResponseField name="enriching" type="object">
  Objet de statut complet de la phase d'enrichissement (voir [Webhooks](/internal/airops-batches/webhooks#enriching-status)).
</ResponseField>
