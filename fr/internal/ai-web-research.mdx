Une comparaison pratique des processeurs actuels Lite, Base et Core dans le nœud [AI Web Research de Gumloop](https://docs.gumloop.com/nodes/using_ai/ai_web_research) avec l’[API Answers](/features/answers) d’Olostep pour les clients de Gumloop.

### TL;DR
- **Latence** : `/answers` en moyenne ~5-15s ; processeur existant en moyenne 60-120s+
- **Actualité** : `/answers` utilise le scraping/crawling en direct (pas d'index obsolète) - plus à jour.
- **Précision** : `/answers` tend à être plus précis et exact que les processeurs existants, surtout pour les requêtes factuelles.
- **Portée** : Convient parfaitement pour les recherches factuelles `lite`/`base`/`core` (pas de recherche approfondie - `pro`/`ultra`).
- **Sortie** : JSON structuré avec sources citées ; facile à post-traiter.
- **Idée d'intégration** : Ajouter un nouveau processeur (par exemple, `core-fast`) qui redirige vers `/answers`.

### Comment nous avons testé
- Réutilisation des invites d'exemple des documents Gumloop dans trois catégories : Lite, Base, Core.
- Mesure du temps jusqu'à la première réponse complète et capture des exemples de sorties et de sources.
- Vérification que les réponses sont basées sur des données réelles du web.
- Comparaison de la sortie avec les processeurs existants dans le nœud [AI Web Research de Gumloop](https://docs.gumloop.com/nodes/using_ai/ai_web_research).

## Quelle est la raison derrière cela ?
L'API Answers d'Olostep, au lieu d'utiliser simplement un index web et de s'y référer, utilise l'infrastructure d'Olostep qui a été optimisée pour rechercher, explorer et extraire les données en temps réel, ce qui la rend plus précise et exacte pour ce type de tâches.

Elle n'est pas bien adaptée pour des tâches de recherche approfondie mais elle surpasse souvent les processeurs existants dans les catégories Lite, Base et Core.


## Prochaines étapes suggérées

Si cela est pertinent pour Gumloop et vos clients, envisagez d'ajouter un nouveau processeur pour Gumloop : **`core-fast`** soutenu par `/answers` pour les requêtes factuelles, basées sur des sources, où la latence est importante.

Si nous pouvons vous être d'une quelconque aide, faites-le nous savoir. Merci !
