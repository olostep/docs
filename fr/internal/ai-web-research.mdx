A PRACTICAL COMPARISON OF THE CURRENT LITE, BASE AND CORE PROCESSORS IN THE GUMLOOP [AI WEB RESEARCH NODE](HTTPS://DOCS.GUMLOOP.COM/NODES/USING_AI/AI_WEB_RESEARCH) WITH OLOSTEP’S [ANSWERS API](/FEATURES/ANSWERS/ANSWERS) FOR GUMLOOP CUSTOMERS.

### TL;DR
- **LATENCY**: `/ANSWERS` AVERAGES ~5-15S; EXISTING PROCESSOR ON AVERAGE 60-120S+
- **FRESHNESS**: `/ANSWERS` USES LIVE SCRAPING/CRAWLING (NO STALE INDEX) - MORE UP-TO-DATE.
- **ACCURACY**: `/ANSWERS` TENDS TO BE MORE ACCURATE AND PRECISE THAN THE EXISTING PROCESSORS, ESPECIALLY FOR FACTUAL QUERIES.
- **SCOPE**: GREAT FIT FOR `LITE`/`BASE`/`CORE` FACTUAL LOOKUPS (NOT DEEP RESEARCH - `PRO`/`ULTRA`).
- **OUTPUT**: STRUCTURED JSON WITH CITED SOURCES; EASY TO POST-PROCESS.
- **INTEGRATION IDEA**: ADD A NEW PROCESSOR (E.G., `CORE-FAST`) THAT ROUTES TO `/ANSWERS`.

### HOW WE TESTED
- REUSED EXAMPLE PROMPTS FROM GUMLOOP DOCS ACROSS THREE CATEGORIES: LITE, BASE, CORE.
- MEASURED TIME-TO-FIRST-COMPLETE-ANSWER AND CAPTURED EXAMPLE OUTPUTS AND SOURCES.
- VERIFIED ANSWERS ARE GROUNDED IN REAL WEB DATA.
- COMPARED THE OUTPUT WITH THE EXISTING PROCESSORS IN THE GUMLOOP [AI WEB RESEARCH NODE](HTTPS://DOCS.GUMLOOP.COM/NODES/USING_AI/AI_WEB_RESEARCH).
