---

## Panoramica

Crea un nuovo batch AirOps che:
1. Esegue il scraping di tutti gli URL forniti
2. Estrae automaticamente i metadati strutturati
3. Identifica gli elementi con campi mancanti
4. Arricchisce gli elementi incompleti tramite LLM
5. Restituisce risultati unificati per tutti gli URL

## Parametri della Richiesta

<ParamField body="items" type="array" required>
  Array di URL da elaborare. Ogni elemento richiede:
  - `url` - URL valido per il scraping
  - `custom_id` - Il tuo identificatore univoco (1-255 caratteri)
</ParamField>

<ParamField body="brands" type="array" default="[]">
  Nomi dei marchi da cercare nel contenuto della pagina. L'LLM rileverà le menzioni di questi marchi specifici e restituirà le corrispondenze nel campo `brand_mentions`.
  
  ```json
  {
    "brands": ["Acme Corp", "TechBrand", "SaaS Pro"]
  }
  ```
</ParamField>

<ParamField body="country" type="string" default="RANDOM">
  Paese per il scraping geo-mirato. Usa i codici ISO 3166-1 alpha-2 (`US`, `GB`, `DE`) o `RANDOM`.
</ParamField>

<ParamField body="webhook" type="string">
  URL per ricevere notifiche webhook. Tutti gli eventi del ciclo di vita per questo batch saranno inviati qui.
</ParamField>

<ParamField body="metadata" type="object">
  Coppie chiave-valore personalizzate associate al batch. Stile Stripe: massimo 50 chiavi, chiave ≤40 caratteri, valore ≤500 caratteri.
</ParamField>
