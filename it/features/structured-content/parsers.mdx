I parser sono un modo per trasformare dati non strutturati in dati strutturati compatibili con il tuo backend. Combinando i parser con l'API di Olostep (crawl, scrape, batch) è possibile trasformare qualsiasi sito web in una chiamata API per restituire il JSON di cui hai bisogno.

I parser sono ideali quando hai bisogno di dati su larga scala in modo ricorrente dagli stessi siti web. Questo approccio è significativamente più economico e veloce rispetto a [estrazione LLM](/features/structured-content/llm-extraction) e restituisce solo il JSON richiesto.

Offriamo parser pre-costruiti per siti web e casi d'uso popolari. Puoi anche creare i tuoi parser in pochi minuti con LLM utilizzando la [dashboard](https://www.olostep.com/dashboard/parsers) o chiedere al nostro team di farlo per te.

### Parser Pre-Costruiti

Offriamo diversi parser pre-costruiti per siti web popolari:
- Ricerca Google: `@olostep/google-search`
- Prodotto Amazon: `@olostep/amazon-it-product`
- Estrazione Email: `@olostep/extract-emails`
- Estrazione Calendari: `@olostep/extract-calendars`
- Estrazione Social: `@olostep/extract-socials`
- Estrazione dati TikTok: contattaci per ottenere l'ID del parser
- Google News: contattaci per ottenere l'ID del parser
- Google Maps: contattaci per ottenere l'ID del parser

### Esempio di Utilizzo

```python
import requests
import json

endpoint = "https://api.olostep.com/v1/scrapes"
payload = {
    "formats": ["json"],
    "parser": {"id": "@olostep/google-search"},
    "url_to_scrape": "https://www.google.com/search?q=alexander+the+great&gl=us&hl=en",
    "wait_before_scraping": 0,
}
headers = {
    "Authorization": "Bearer <YOUR-API-KEY>",
    "Content-Type": "application/json"
}

response = requests.request("POST", endpoint, json=payload, headers=headers)
print(json.dumps(response.json(), indent=4))
```

La risposta contiene:

- **Informazioni di base sulla richiesta**: `id`, `object`, timestamp `created`, `url_to_scrape`
- **Oggetto result** con URL per accedere a diversi formati dei dati
- **json_content** con risultati JSON strutturati che includono:
- `searchParameters`: Informazioni sulla query di ricerca
- `knowledgeGraph`: Informazioni dettagliate sul soggetto della ricerca (quando disponibili)
- `organic`: Elenco dei risultati di ricerca con titolo, link, posizione e snippet
- `peopleAlsoAsk`: Domande correlate che gli utenti cercano comunemente
- `relatedSearches`: Suggerimenti di ricerche correlate

`json_content` è la parte principale della risposta con i risultati JSON strutturati. Puoi accedere direttamente al contenuto JSON dalla risposta o utilizzare l'URL ospitato fornito nella risposta.
