**Pacchetto NPM:** [olostep-cli](https://www.npmjs.com/package/olostep-cli)

CLI per l’[API Olostep](https://www.olostep.com/): **map**, **answer**, **scrape**, **scrape-get**, **crawl**, **batch-scrape** e **batch-update**. Le risposte sono **JSON** (formattato) per script, CI e agent.

Installando questo pacchetto viene scaricato un **binario autonomo** per il tuo OS. Non serve Python per eseguire `olostep`.

<Note>
  Dopo l’installazione, un passaggio postinstall scarica il binario corretto dalle [release GitHub](https://github.com/olostep-api/CLI/releases) del pacchetto. Se fallisce, verifica che esista una release per questa versione del pacchetto e la tua piattaforma, poi reinstalla.
</Note>

## Installazione

**Requisiti:** Node.js **16+** (solo per `npm` / `npx`; la CLI è un binario nativo).

```bash
npm install -g olostep-cli
```

Senza installazione globale:

```bash
npx -y olostep-cli@latest --help
```

**Piattaforme:** macOS (Apple Silicon e Intel), Linux x64, Windows x64.

## Chiave API

Imposta **`OLOSTEP_API_KEY`** nell’ambiente o in un file `.env` nella directory da cui esegui `olostep`.

Crea le chiavi: [Olostep API Keys](https://www.olostep.com/dashboard/api-keys).

```bash
export OLOSTEP_API_KEY=your_key_here
olostep --help
```

## Output

| Flag | Comportamento |
| ---- | ------------- |
| `--out <path>` | Scrive JSON su file (crea le directory padre se necessario). |
| `--out -` | Scrive JSON solo su **stdout** (UTF-8, indentato). Utile per pipe e cattura da sottoprocessi. |

Avanzamento e log vanno su **stderr**, così stdout resta analizzabile.

```bash
olostep map "https://example.com" --top-n 20 --out - | jq '.urls[:10]'
olostep scrape "https://example.com" --out - | jq .result.markdown_content
```

Se ometti `--out`, i file vanno sotto `output/` (vedi [percorsi predefiniti per `--out`](#default-out-paths)).

## Comandi

Usa `olostep <comando> --help` per tutte le opzioni. La maggior parte accetta `--timeout` (HTTP, secondi).

### `map`: scoprire URL

| Opzione | Descrizione |
| ------- | ----------- |
| `--out` | Percorso file o `-` |
| `--top-n` | Numero massimo di URL |
| `--search-query` | Guida la scoperta |
| `--include-subdomain` / `--no-include-subdomain` | Sottodomini |
| `--include-url` / `--exclude-url` | Pattern ripetibili |
| `--cursor` | Paginazione |

Usa `--top-n`, non `--limit` (rimosso).

```bash
olostep map "https://example.com" --top-n 100 --search-query "blog"
```

### `answer`: risposta con ricerca

Esegue polling finché la risposta non è pronta.

| Opzione | Descrizione |
| ------- | ----------- |
| `--out` | File o `-` |
| `--json-format` | Suggerimento opzionale per output strutturato (stringa oggetto JSON) |
| `--poll-interval` / `--poll-timeout` | Polling (secondi) |

Usa `--json-format`, non `--model` (rimosso).

```bash
olostep answer "What does this company build?" --out answer.json
olostep answer "Extract facts" --json-format '{"company":"","year":""}' --out -
```

### `scrape`: singolo URL

**Formati:** `html`, `markdown`, `text`, `json`, `raw_pdf`, `screenshot` (separati da virgola; predefinito `markdown`).

| Opzione | Descrizione |
| ------- | ----------- |
| `--formats` | Separati da virgola |
| `--country` | Codice paese |
| `--wait-before-scraping` | Millisecondi di attesa prima dello scrape |
| `--payload-json` | Opzioni avanzate come stringa JSON |
| `--payload-file` | Stesso da file (non insieme a `--payload-json`) |

```bash
olostep scrape "https://example.com" --formats markdown,html
olostep scrape "https://example.com" --payload-file scrape-options.json --out -
```

### `scrape-get`: per ID scrape

```bash
olostep scrape-get "scrape_abc123" --out -
```

### `crawl`: crawl del sito

**Formati di recupero:** `markdown`, `html`, `json`.

Flag principali: `--max-pages`, `--max-depth`, `--include-subdomain`, `--include-external`, `--include-url`, `--exclude-url`, `--search-query`, `--top-n`, `--webhook`, `--crawl-timeout`, `--follow-robots-txt` / `--ignore-robots-txt`, `--formats`, `--pages-limit`, `--pages-search-query`, `--poll-seconds`, `--poll-timeout`, `--dry-run` (stampa payload, nessuna richiesta).

```bash
olostep crawl "https://docs.example.com" --max-pages 50 --formats markdown,html
olostep crawl "https://example.com" --max-pages 10 --dry-run
```

### `batch-scrape`: CSV

Colonne CSV: **`custom_id` o `id`**, e **`url`**.

| Opzione | Descrizione |
| ------- | ----------- |
| `--formats` | `markdown`, `html`, `json` |
| `--country` | Opzionale |
| `--parser-id` | Parser per estrazione strutturata |
| `--poll-seconds`, `--log-every`, `--items-limit` | Polling / impaginazione |
| `--dry-run` | Stampa solo il payload |

```bash
olostep batch-scrape urls.csv --formats markdown,html
```

### `batch-update`: metadati batch

Richiede **uno tra** `--metadata-json` o `--metadata-file` (oggetto JSON).

```bash
olostep batch-update "batch_abc123" --metadata-json '{"team":"growth"}'
```

## Percorsi predefiniti per `--out` {#default-out-paths}

Senza `--out`:

| Comando | Predefinito |
| ------- | ----------- |
| `map` | `output/map.json` |
| `answer` | `output/answer.json` |
| `scrape` | `output/scrape.json` |
| `scrape-get` | `output/scrape_get.json` |
| `crawl` | `output/crawl_results.json` |
| `batch-scrape` | `output/batch_results.json` |
| `batch-update` | `output/batch_update.json` |
