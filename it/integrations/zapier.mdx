## Installazione

### 1. Trova Olostep su Zapier

Cerca "Olostep" nella directory delle app di Zapier o quando crei un nuovo Zap:

1. Vai a [Olostep su Zapier](https://zapier.com/apps/olostep/integrations)
2. Clicca su "Create Zap"
3. Cerca "Olostep" nel selettore delle app
4. Seleziona l'app Olostep

In alternativa, visita direttamente la [pagina di integrazione Olostep](https://zapier.com/apps/olostep/integrations) per sfogliare le azioni disponibili e creare il tuo primo Zap.

### 2. Connetti il tuo account

Quando utilizzi Olostep per la prima volta in uno Zap, ti verrà chiesto di connettere il tuo account:

1. Clicca su "Sign in to Olostep"
2. Inserisci la tua chiave API di Olostep
3. Clicca su "Yes, Continue to Olostep"

Ottieni la tua chiave API dalla [Dashboard di Olostep](https://olostep.com/dashboard).

## Azioni Disponibili

### Scrape Website

Estrai contenuti da un singolo URL. Supporta più formati e il rendering JavaScript.

**Casi d'uso:**
- Monitora pagine specifiche per cambiamenti
- Estrai informazioni sui prodotti da siti e-commerce
- Raccogli dati da articoli di notizie o post di blog
- Recupera contenuti per l'aggregazione di contenuti

**Configurazione:**

<ParamField path="URL to Scrape" type="string" required>
  URL del sito web da raschiare (deve includere http:// o https://)
</ParamField>

<ParamField path="Output Format" type="dropdown" default="Markdown">
  Scegli il formato: Markdown, HTML, JSON o Testo Semplice
</ParamField>

<ParamField path="Country Code" type="string">
  Codice paese per contenuti specifici della località (es. "US", "GB", "CA")
</ParamField>

<ParamField path="Wait Before Scraping" type="integer">
  Tempo di attesa in millisecondi per il rendering JavaScript (0-10000)
</ParamField>

<ParamField path="Parser" type="string">
  ID parser opzionale per l'estrazione specializzata (es. "@olostep/amazon-product")
</ParamField>

**Campi di Output:**
- Scrape ID
- URL Raschiato
- Contenuto Markdown
- Contenuto HTML
- Contenuto JSON
- Contenuto Testo
- Stato
- Timestamp
- URL Screenshot (se disponibile)
- Metadati della Pagina

**Esempi di Flussi di Lavoro:**

<AccordionGroup>
  <Accordion title="Monitor Competitor Pricing">
    **Trigger:** Schedule (Ogni giorno alle 9 AM)
    
    **Azione:** Olostep - Scrape Website
    - URL: Pagina prodotto del concorrente
    - Formato: JSON
    - Parser: @olostep/amazon-product
    
    **Azione:** Google Sheets - Create Row
    - Aggiungi dati di prezzo al foglio di tracciamento
    
    **Azione:** Gmail - Send Email (Se il prezzo scende)
    - Avvisa il team sui cambiamenti di prezzo
  </Accordion>

  <Accordion title="Extract and Save Blog Posts">
    **Trigger:** RSS by Zapier - New Item in Feed
    
    **Azione:** Olostep - Scrape Website
    - URL: \{\{Post URL\}\}
    - Formato: Markdown
    
    **Azione:** Notion - Create Page
    - Salva il contenuto dell'articolo nel database di Notion
  </Accordion>

  <Accordion title="Lead Enrichment">
    **Trigger:** Google Sheets - New Row
    
    **Azione:** Olostep - Scrape Website
    - URL: Sito web aziendale dal foglio
    - Formato: Markdown
    
    **Azione:** OpenAI - Complete Text
    - Estrai informazioni aziendali utilizzando l'AI
    
    **Azione:** Google Sheets - Update Row
    - Aggiungi dati arricchiti al foglio
  </Accordion>
</AccordionGroup>

### Batch Scrape URLs

Elabora più URL in parallelo (fino a 100.000 contemporaneamente). Perfetto per l'estrazione di dati su larga scala.

**Casi d'uso:**
- Raschia interi cataloghi di prodotti
- Estrai dati da più risultati di ricerca
- Elabora elenchi di URL da fogli di calcolo
- Estrazione di contenuti in blocco

**Configurazione:**

<ParamField path="URLs to Scrape" type="text" required>
  Array JSON di oggetti con campi url e custom_id. 
  
  Esempio: `[{"url":"https://example.com","custom_id":"site1"}]`
</ParamField>

<ParamField path="Output Format" type="dropdown" default="Markdown">
  Scegli il formato per tutti gli URL: Markdown, HTML, JSON o Testo Semplice
</ParamField>

<ParamField path="Country Code" type="string">
  Codice paese per il raschiamento specifico della località
</ParamField>

<ParamField path="Wait Before Scraping" type="integer">
  Tempo di attesa in millisecondi per il rendering JavaScript
</ParamField>

<ParamField path="Parser" type="string">
  ID parser opzionale per l'estrazione specializzata
</ParamField>

**Campi di Output:**
- Batch ID (usalo per recuperare i risultati in seguito)
- Stato
- Totale URL
- Creato il
- Formato Richiesto
- Codice Paese
- Parser Usato

**Esempi di Flussi di Lavoro:**

<AccordionGroup>
  <Accordion title="Scrape Product Catalog">
    **Trigger:** Webhook - Receive POST Request
    
    **Azione:** Code by Zapier - Run Python
    - Converti CSV/elenco in formato array JSON
    
    **Azione:** Olostep - Batch Scrape URLs
    - URL: \{\{JSON array from previous step\}\}
    - Formato: JSON
    - Parser: @olostep/amazon-product
    
    **Azione:** Webhook - POST
    - Invia l'ID batch al tuo sistema per il recupero
  </Accordion>

  <Accordion title="Daily Content Monitoring">
    **Trigger:** Schedule - Ogni giorno alle 6 AM
    
    **Azione:** Google Sheets - Get Rows
    - Recupera gli URL da monitorare
    
    **Azione:** Code by Zapier - Format URLs
    - Converti in formato array batch
    
    **Azione:** Olostep - Batch Scrape URLs
    - Elabora tutti gli URL contemporaneamente
    
    **Azione:** Slack - Send Message
    - Notifica al team che il raschiamento è completo
  </Accordion>
</AccordionGroup>

### Create Crawl

Scopri e raschia autonomamente interi siti web seguendo i link. Perfetto per siti di documentazione, blog e repository di contenuti.

**Casi d'uso:**
- Esegui il crawling e archivia interi siti di documentazione
- Estrai tutti i post di blog da un sito web
- Costruisci basi di conoscenza da contenuti web
- Monitora i cambiamenti nella struttura del sito

**Configurazione:**

<ParamField path="Start URL" type="string" required>
  URL di partenza per il crawl (deve includere http:// o https://)
</ParamField>

<ParamField path="Maximum Pages" type="integer" default="10">
  Numero massimo di pagine da eseguire il crawling
</ParamField>

<ParamField path="Follow Links" type="boolean" default="true">
  Se seguire i link trovati nelle pagine
</ParamField>

<ParamField path="Output Format" type="dropdown" default="Markdown">
  Formato per il contenuto raschiato
</ParamField>

<ParamField path="Country Code" type="string">
  Codice paese opzionale per il crawling specifico della località
</ParamField>

<ParamField path="Parser" type="string">
  ID parser opzionale per l'estrazione di contenuti specializzati
</ParamField>

**Campi di Output:**
- Crawl ID (usalo per recuperare i risultati in seguito)
- Tipo di Oggetto
- Stato
- URL di Partenza
- Pagine Massime
- Segui Link
- Timestamp Creato
- Formati

**Esempi di Flussi di Lavoro:**

<AccordionGroup>
  <Accordion title="Archive Documentation Site">
    **Trigger:** Schedule - Mensile il 1° alle 12 AM
    
    **Azione:** Olostep - Create Crawl
    - URL di Partenza: https://docs.example.com
    - Pagine Massime: 500
    - Segui Link: true
    - Formato: Markdown
    
    **Azione:** Webhook - POST
    - Invia l'ID del crawl al tuo sistema di archiviazione
    
    **Azione:** Slack - Send Message
    - Notifica al team che il crawl è in corso
  </Accordion>

  <Accordion title="Competitor Content Analysis">
    **Trigger:** Schedule - Settimanale il lunedì alle 9 AM
    
    **Azione:** Olostep - Create Crawl
    - URL di Partenza: URL del blog del concorrente
    - Pagine Massime: 100
    - Formato: Markdown
    
    **Azione:** Delay - Per 10 minuti
    - Attendi il completamento del crawl
    
    **Azione:** Airtable - Create Records

  </Accordion>
</AccordionGroup>

### Create Map

Estrai tutti gli URL da un sito web per la scoperta di contenuti e l'analisi della struttura del sito.

**Casi d'uso:**
- Costruisci sitemap e diagrammi della struttura del sito
- Scopri tutte le pagine prima del raschiamento batch
- Trova pagine rotte o mancanti
- Audit e analisi SEO

**Configurazione:**

<ParamField path="Website URL" type="string" required>
  URL del sito web da cui estrarre i link (deve includere http:// o https://)
</ParamField>

<ParamField path="Search Query" type="string">
  Query di ricerca opzionale per filtrare gli URL (es. "blog")
</ParamField>

<ParamField path="Top N URLs" type="integer">
  Limita il numero di URL restituiti
</ParamField>

<ParamField path="Include URL Patterns" type="string">
  Modelli glob per includere percorsi specifici (es. "/blog/**")
</ParamField>

<ParamField path="Exclude URL Patterns" type="string">
  Modelli glob per escludere percorsi specifici (es. "/admin/**")
</ParamField>

**Campi di Output:**
- Map ID
- Tipo di Oggetto
- URL del Sito Web
- Totale URL Trovati
- URL (array JSON)
- Query di Ricerca
- Limite Top N

**Esempi di Flussi di Lavoro:**

<AccordionGroup>
  <Accordion title="Discover and Scrape">
    **Trigger:** Clic su Pulsante in Zapier
    
    **Azione:** Olostep - Create Map
    - URL: https://example.com
    - Include Patterns: /products/**
    - Top N: 500
    
    **Azione:** Code by Zapier - Extract URLs
    - Analizza gli URL dal risultato della mappa
    
    **Azione:** Olostep - Batch Scrape URLs
    - URL: \{\{URLs from map\}\}
    - Formato: JSON
    
    **Azione:** Google Sheets - Create Rows
    - Aggiungi tutti i dati dei prodotti al foglio di calcolo
  </Accordion>

  <Accordion title="SEO Site Audit">
    **Trigger:** Schedule - Mensile
    
    **Azione:** Olostep - Create Map
    - URL: Il tuo sito web
    - Top N: 1000
    
    **Azione:** Airtable - Create Records
    - Memorizza tutti gli URL per il tracciamento
    
    **Azione:** Slack - Send Message
    - Riporta il totale delle pagine trovate
  </Accordion>
</AccordionGroup>

### Ask AI Answer

Ottieni risposte potenziate dall'AI a domande utilizzando la ricerca web o i tuoi URL di contesto forniti. Perfetto per l'automazione della ricerca, la generazione di contenuti e l'analisi dei dati.

**Casi d'uso:**
- Automazione della ricerca con citazioni
- Riassumi contenuti da più URL
- Genera report con riferimenti alle fonti
- Rispondi a domande utilizzando pagine web specifiche
- Analisi e estrazione di contenuti

**Configurazione:**

<ParamField path="Question" type="string" required>
  La domanda a cui vuoi che Olostep Answers risponda
</ParamField>

<ParamField path="Context URLs (JSON Array)" type="string">
  Array JSON opzionale di URL per fondare la risposta (es. ["https://example.com/page1", "https://example.com/page2"])
</ParamField>

<ParamField path="Search Query" type="string">
  Query opzionale per recuperare fonti dal web se non vengono forniti URL di contesto (es. "site:example.com pricing")
</ParamField>

<ParamField path="Number of Sources" type="integer">
  Limita il numero di fonti utilizzate per generare la risposta
</ParamField>

<ParamField path="Format" type="dropdown" default="Markdown">
  Scegli il formato della risposta: Markdown, JSON o Testo Semplice
</ParamField>

<ParamField path="Include Citations" type="boolean" default="true">
  Se includere citazioni/fonti nella risposta
</ParamField>

**Campi di Output:**
- Answer ID
- Tipo di Oggetto
- Domanda
- Risposta (Testo)
- Risposta (Markdown)
- Risposta (JSON)
- Citazioni (array JSON con URL, titoli, frammenti)
- Timestamp Creato
- Formato
- URL di Contesto
- Query di Ricerca
- Numero di Fonti

**Esempi di Flussi di Lavoro:**

## Flussi di Lavoro Multi-Step

### Pipeline Completa di Scraping Prodotti

Costruisci una pipeline completa di dati sui prodotti:

<Steps>
  <Step title="Scopri URL dei Prodotti">
    Usa **Create Map** per trovare tutte le pagine dei prodotti sul sito target
    - Include patterns: `/products/**`
    - Exclude patterns: `/cart/**`, `/checkout/**`
  </Step>
  
  <Step title="Elaborazione Batch dei Prodotti">
    Usa **Batch Scrape URLs** per estrarre tutti i dati dei prodotti
    - Formato: JSON
    - Parser: Parser specifico per il prodotto se disponibile
  </Step>
  
  <Step title="Memorizza nel Database">
    Invia l'ID batch al tuo sistema o attendi e recupera i risultati
    - Usa Airtable, Google Sheets o il tuo database
  </Step>
  
  <Step title="Monitora i Cambiamenti">
    Pianifica raschiamenti giornalieri per monitorare i cambiamenti di prezzo/disponibilità
    - Confronta con i dati esistenti
    - Avvisa sui cambiamenti significativi
  </Step>
</Steps>

    - Pianifica la tua strategia di contenuti
  </Step>
</Steps>

## Best Practices

<AccordionGroup>
  <Accordion title="Usa l'Elaborazione Batch per Più URL">
    Quando raschi più di 3-5 URL, usa **Batch Scrape URLs** invece di più azioni **Scrape Website**. L'elaborazione batch è:
    - Molto più veloce (elaborazione parallela)
    - Più conveniente
    - Più facile da gestire
    - Migliore per i limiti di velocità
  </Accordion>

  <Accordion title="Imposta Tempi di Attesa Appropriati">
    Per i siti pesanti in JavaScript, usa il parametro "Wait Before Scraping":
    - Siti semplici: 0-1000ms
    - Siti dinamici: 2000-3000ms
    - JavaScript pesante: 5000-8000ms
    
    Testa con valori diversi per trovare il tempo di attesa ottimale.
  </Accordion>

  <Accordion title="Usa Parser Specializzati">
    Per siti web popolari (Amazon, LinkedIn, Google), usa parser pre-costruiti:
    - Ottieni dati strutturati automaticamente
    - Estrazione più affidabile
    - Nessun bisogno di parsing personalizzato
    - Manutenzione da parte di Olostep
  </Accordion>

  <Accordion title="Filtra Prima di Raschiare">
    Usa l'azione Filter di Zapier per evitare raschiamenti non necessari:
    - Verifica se l'URL è cambiato
    - Verifica che i dati non siano stati raschiati di recente
    - Applica logica aziendale prima del raschiamento
    
    Questo risparmia crediti API e tempo di esecuzione.
  </Accordion>

  <Accordion title="Gestisci Operazioni Asincrone">
    Le operazioni Batch, Crawl e Map sono asincrone:
    - Memorizza l'ID restituito (batch_id, crawl_id, map_id)
    - Usa un'azione Delay se recuperi immediatamente
    - Considera i callback webhook per il completamento
    - Imposta Zap separati per il recupero
  </Accordion>

  <Accordion title="Memorizza i Risultati Correttamente">
    Scegli l'archiviazione giusta in base alle tue esigenze:
    - **Google Sheets**: Tracciamento semplice, collaborazione del team
    - **Airtable**: Dati relazionali, formattazione ricca
    - **Database**: Su larga scala, query complesse
    - **Notion**: Base di conoscenza, documentazione
  </Accordion>

  <Accordion title="Monitora e Avvisa">

## Risoluzione dei Problemi

<AccordionGroup>
  <Accordion title="Autenticazione Fallita">
    **Errore**: "Chiave API non valida"
    
    **Soluzioni**:
    - Controlla la chiave API dalla [dashboard](https://olostep.com/dashboard)
    - Assicurati che non ci siano spazi extra nella chiave API
    - Riconnetti il tuo account Olostep in Zapier
    - Verifica che la chiave API sia attiva
  </Accordion>

  <Accordion title="Il Raschiamento Restituisce Contenuto Vuoto">
    **Errore**: I campi di contenuto sono vuoti
    
    **Soluzioni**:
    - Aumenta il tempo di "Wait Before Scraping"
    - Controlla se il sito richiede il login
    - Prova un formato diverso (HTML vs Markdown)
    - Verifica che l'URL sia accessibile
    - Controlla se il sito blocca l'accesso automatico
  </Accordion>

  <Accordion title="Errore di Formato Array Batch">
    **Errore**: "Formato JSON non valido per l'array batch"
    
    **Soluzioni**:
    - Usa il formato: `[{"url":"https://example.com","custom_id":"id1"}]`
    - Assicurati della corretta sintassi JSON
    - Usa il passaggio Code per formattare correttamente gli URL
    - Testa il JSON con un validatore online
  </Accordion>

  <Accordion title="Limite di Velocità Superato">
    **Errore**: "Limite di velocità superato"
    
    **Soluzioni**:
    - Distanza le esecuzioni degli Zap con ritardi
    - Usa l'elaborazione batch invece di raschiamenti individuali
    - Aggiorna il tuo piano Olostep
    - Controlla il limite di velocità nella dashboard
  </Accordion>

  <Accordion title="URL Non Raschiato">
    **Errore**: URL specifici non vengono raschiati
    
    **Soluzioni**:
    - Verifica il formato dell'URL (includi http:// o https://)
    - Controlla se l'URL richiede autenticazione
    - Testa l'URL prima nel browser
    - Prova con il parametro del paese
    - Contatta il supporto per domini bloccati
  </Accordion>
</AccordionGroup>

## Limitazioni di Zapier e Soluzioni Alternative

### Limiti di Task
Zapier ha limiti di task basati sul tuo piano. Ogni azione Olostep conta come 1 task.

**Soluzione Alternativa**: Usa l'elaborazione batch per raschiare più URL come un singolo task.

### Tempo di Esecuzione
Gli Zap scadono dopo 30 secondi. I crawl e i batch grandi possono richiedere più tempo.

**Soluzione Alternativa**: Memorizza l'ID e recupera i risultati in uno Zap separato o usa webhook.

### Dimensione dei Dati
Zapier ha limiti sulla dimensione dei dati per task.

**Soluzione Alternativa**: Usa URL ospitati nell'output per recuperare separatamente contenuti di grandi dimensioni.

### Polling vs Istantaneo
I trigger sono basati su polling (controllo ogni 5-15 minuti).

**Soluzione Alternativa**: Usa webhook per notifiche istantanee o pianifica a orari specifici.
