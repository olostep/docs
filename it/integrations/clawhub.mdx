Il tuo agente OpenClaw può leggere la documentazione, ma non può leggere il web. Il [plugin Olostep Web Agent](https://clawhub.ai/plugins/olostep-web-agent) risolve questo problema — cerca, estrae, esplora e raccoglie dati strutturati da qualsiasi sito web, direttamente all'interno del flusso di lavoro del tuo agente.

Un'installazione ti offre **13 competenze** per compiti di alto livello (debug di un errore da thread live di StackOverflow, scrittura di codice di integrazione dalla documentazione attuale, ricerca di strumenti con confronti strutturati) e un **server MCP con 9 strumenti** per l'accesso programmato diretto. SPAs pesanti in JS, CAPTCHA, Cloudflare, proxy residenziali — gestiti automaticamente.

## Installazione

```bash
clawhub install olostep
```

Ecco fatto. Se preferisci collegare manualmente il server MCP, aggiungi questo alla configurazione di OpenClaw:

```json
{
  "mcpServers": {
    "olostep": {
      "command": "npx",
      "args": ["-y", "olostep-mcp"],
      "env": {
        "OLOSTEP_API_KEY": "your-api-key-here"
      }
    }
  }
}
```

Ottieni una chiave API gratuita su [olostep.com/auth](https://olostep.com/auth) — 500 richieste/mese, nessuna carta di credito.

## Strumenti MCP

Il server MCP incluso (`olostep-mcp`) fornisce al tuo agente 9 strumenti che può chiamare direttamente. Usali quando hai bisogno di un controllo dettagliato oltre ciò che le competenze di flusso di lavoro forniscono.

| Strumento | Cosa fa |
|------|-------------|
| `scrape_website` | Estrai un singolo URL in markdown, HTML, JSON o testo |
| `get_webpage_content` | Recupera una pagina web come markdown pulito, pronto per LLM |
| `search_web` | Cerca nel web live, ottieni risposte sintetizzate dall'AI |
| `google_search` | Dati strutturati di Google SERP — risultati organici, PAA, knowledge graph |
| `answers` | Fai una domanda, ottieni una risposta citata con output JSON strutturato opzionale |
| `batch_scrape_urls` | Estrai fino a 10.000 URL in parallelo con rendering completo |
| `create_crawl` | Esplora un sito web seguendo i link da un URL iniziale |
| `create_map` | Scopri tutti gli URL su un sito web, filtrabili per pattern e query |
| `get_website_urls` | Trova e recupera URL rilevanti da un dominio specifico |

## Come si Presenta nella Pratica

### "Perché sta fallendo?" — Debug dal web live

Incolli `ECONNRESET quando chiami l'endpoint webhook di Stripe` nel tuo agente. Cerca nei problemi di GitHub e su StackOverflow per quell'errore esatto, estrae i tre thread più rilevanti e restituisce una soluzione concreta — non "controlla le impostazioni di rete", ma la configurazione di timeout effettiva che l'ha risolto per altri sviluppatori che hanno incontrato lo stesso ostacolo.

### "Scrivi l'integrazione" — Codice dalla documentazione attuale, non da dati di addestramento obsoleti

Hai bisogno di integrare un'API di pagamento. La competenza `docs-to-code` estrae il riferimento API corrente — quello pubblicato oggi, non la versione su cui il tuo modello è stato addestrato sei mesi fa — e scrive codice funzionante usando i parametri e gli endpoint che esistono effettivamente.

### "Quale dovrei usare?" — Confronti strutturati di strumenti

Valutando ORM? Confrontando fornitori di autenticazione? La competenza `research` cerca in più fonti, estrae pagine di prezzi reali e matrici di caratteristiche, e restituisce una tabella di confronto strutturata con citazioni. Ottieni una raccomandazione supportata da ciò che è effettivamente sul sito di ciascun prodotto, non dalla popolarità nei dati di addestramento.

### Costruisci una pipeline RAG da qualsiasi sito di documentazione

```
map   → scopri ogni URL sul sito di documentazione
batch → estrai tutte le pagine in parallelo come markdown pulito
      → alimenta nel tuo archivio vettoriale
```

Due competenze, una pipeline. Funziona su siti di documentazione con centinaia o migliaia di pagine.

### Estrai dati strutturati su larga scala

```
map   → trova tutti gli URL di prodotto, elenco o lavoro
batch → estrai ogni pagina con un parser pre-costruito → JSON tipizzato
      → invia al tuo database, API, o file di seed
```

### Migra a una nuova versione del framework

Stai aggiornando Next.js dalla versione 13 alla 15. La competenza `migrate-code` estrae la guida ufficiale alla migrazione, estrae ogni cambiamento critico con pattern di codice prima/dopo, e riscrive i tuoi file per adattarli — basato sulla documentazione reale, non sulla migliore ipotesi del modello.

## Parser Pre-costruiti

Passa il parametro `parser` a qualsiasi chiamata di estrazione e ottieni JSON tipizzato invece di contenuto grezzo. Nessuna definizione di schema necessaria — questi gestiscono l'estrazione per te.

| Parser | Restituisce |
|--------|---------|
| `@olostep/google-search` | Risultati organici, knowledge graph, People Also Ask |
| `@olostep/amazon-it-product` | Prezzo, valutazione, caratteristiche, disponibilità |
| `@olostep/extract-emails` | Ogni indirizzo email trovato sulla pagina |
| `@olostep/extract-calendars` | Eventi del calendario strutturati |
| `@olostep/extract-socials` | Link ai profili dei social media |
