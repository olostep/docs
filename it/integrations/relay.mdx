## Installazione

### 1. Trova Olostep in Relay.app

Cerca "Olostep" in Relay.app quando crei un nuovo workflow:

1. Vai a [Olostep su Relay.app](https://www.relay.app/apps/olostep/integrations)
2. Clicca su "Inizia gratis" o "Crea Workflow"
3. Cerca "Olostep" nel selettore di app
4. Seleziona l'app Olostep

In alternativa, visita direttamente la [pagina di integrazione di Olostep](https://www.relay.app/apps/olostep/integrations) per esplorare le azioni disponibili e creare il tuo primo workflow.

### 2. Connetti il tuo Account

Quando usi per la prima volta Olostep in un workflow, ti verrà chiesto di connettere il tuo account:

1. Clicca su "Connetti Olostep"
2. Inserisci la tua chiave API di Olostep
3. Autorizza la connessione

Ottieni la tua chiave API dalla [Dashboard di Olostep](https://olostep.com/dashboard).

## Azioni Disponibili

### Scrape URL

Estrai contenuto da un singolo URL. Supporta formati multipli e rendering JavaScript.

**Casi d'Uso:**
- Monitora pagine specifiche per cambiamenti
- Estrai informazioni sui prodotti da siti e-commerce
- Raccogli dati da articoli di notizie o post di blog
- Recupera contenuti per aggregazione di contenuti

**Configurazione:**

<ParamField path="URL" type="string" required>
  URL del sito web da scrappare (deve includere http:// o https://)
</ParamField>

<ParamField path="Format" type="dropdown" default="Markdown">
  Scegli il formato: Markdown, HTML, JSON o Testo Semplice
</ParamField>

<ParamField path="Country Code" type="string">
  Codice paese per contenuti specifici per località (es. "US", "GB", "CA")
</ParamField>

<ParamField path="Wait Before Scraping" type="integer">
  Tempo di attesa in millisecondi per il rendering JavaScript (0-10000)
</ParamField>

<ParamField path="Parser" type="string">
  ID parser opzionale per estrazione specializzata (es. "@olostep/amazon-product")
</ParamField>

**Campi di Output:**
- Scrape ID
- URL Scrappato
- Contenuto Markdown
- Contenuto HTML
- Contenuto JSON
- Contenuto Testo
- Stato
- Timestamp
- URL Screenshot (se disponibile)
- Metadata Pagina

**Esempi di Workflow:**

<AccordionGroup>
  <Accordion title="Monitoraggio Prezzi Competitor">
    **Trigger:** Programma (Ogni giorno alle 9 AM)
    
    **Azione:** Olostep - Scrape URL
    - URL: Pagina prodotto del competitor
    - Formato: JSON
    - Parser: @olostep/amazon-product
    
    **Azione:** Google Sheets - Crea Riga
    - Aggiungi dati di prezzo al foglio di monitoraggio
    
    **Human-in-the-Loop:** Rivedi cambiamenti di prezzo
    - Approva cali di prezzo significativi
    
    **Azione:** Slack - Invia Messaggio
    - Avvisa il team sui cambiamenti di prezzo approvati
  </Accordion>

  <Accordion title="Estrai e Salva Post di Blog">
    **Trigger:** Feed RSS - Nuovo Elemento
    
    **Azione:** Olostep - Scrape URL
    - URL: \{\{Post URL\}\}
    - Formato: Markdown
    
    **Azione:** Notion - Crea Pagina
    - Salva il contenuto dell'articolo nel database di Notion
  </Accordion>

  <Accordion title="Arricchimento Lead con AI">
    **Trigger:** Google Sheets - Nuova Riga
    
    **Azione:** Olostep - Scrape URL
    - URL: Sito web dell'azienda dal foglio
    - Formato: Markdown
    
    **Azione AI:** OpenAI - Estrai Dati
    - Estrai: dimensione azienda, settore, prodotti
    
    **Azione:** Google Sheets - Aggiorna Riga
    - Aggiungi dati arricchiti al foglio
  </Accordion>
</AccordionGroup>

## Vantaggi di Relay.app

Relay.app offre diversi vantaggi rispetto alle piattaforme di automazione tradizionali:

### Workflow Human-in-the-Loop

Aggiungi passaggi di revisione manuale direttamente nei tuoi workflow:

```
1. Olostep scrappa il sito web
2. Passaggio di Revisione Umana ← Rivedi e approva i dati
3. Salva nel database solo se approvato
```

Perfetto per:
- Controllo qualità prima dell'elaborazione dei dati
- Giudizio umano sui contenuti estratti
- Workflow di approvazione per dati sensibili
- Revisione di riassunti generati dall'AI

### Funzionalità AI Integrate

Usa l'AI direttamente nei tuoi workflow senza integrazioni esterne:

- **Decisioni AI**: Lascia che l'AI decida i percorsi del workflow in base ai contenuti scrappati
- **Filtraggio Intelligente**: Filtra e categorizza automaticamente i dati scrappati
- **Riassunto Contenuti**: Riassumi articoli lunghi prima di memorizzarli
- **Estrazione Dati**: Usa l'AI per estrarre dati strutturati da contenuti non strutturati

### Migliore UX e Testing

L'interfaccia user-friendly di Relay.app rende più facile la costruzione dei workflow:

- Informazioni di testing chiare
- Messaggi di errore utili
- Builder di workflow visivo
- Debugging e risoluzione problemi facili

## Esempi di Workflow Popolari

### Monitoraggio Prezzi E-commerce con Revisione

Monitora i prezzi dei competitor con approvazione umana:

```
Trigger: Programma (Ogni ora)
↓
Azione: Olostep - Scrape URL
  - URL: Pagina prodotto del competitor
  - Formato: JSON
  - Parser: @olostep/amazon-product
↓
Passaggio di Revisione Umana
  - Rivedi cambiamento di prezzo
  - Approva o rifiuta
↓
Azione: Slack - Invia Messaggio (Se approvato)
  - Avviso: "Prezzo cambiato a $\{\{price\}\}"
```

### Aggregazione Contenuti con AI

Aggrega e migliora contenuti da più fonti:

```
Trigger: Google Sheets - Nuova Riga
↓
Azione: Olostep - Scrape URL
  - URL: \{\{URL dal foglio\}\}
  - Formato: Markdown
↓
Azione AI: Riassumi Contenuto
  - Genera riassunto
↓
Azione: Airtable - Crea Record
  - Memorizza articolo con riassunto AI
```

### Pipeline di Arricchimento Lead

Arricchisci i dati dei lead con informazioni web e AI:

```
Trigger: HubSpot - Nuovo Contatto
↓
Azione: Olostep - Scrape URL
  - URL: \{\{Sito web aziendale\}\}
  - Formato: Markdown
↓
Azione AI: Estrai Info Azienda
  - Estrai: dimensione azienda, settore, prodotti
↓
Passaggio di Revisione Umana
  - Verifica l'accuratezza dell'estrazione AI
↓
Azione: HubSpot - Aggiorna Contatto
  - Aggiungi dati arricchiti verificati
```

Perfetto per la raccolta e il monitoraggio dei dati:

**Casi d'Uso:**
- Fogli di monitoraggio prezzi
- Database di arricchimento lead
- Inventario contenuti
- Fogli di analisi competitor

### Airtable

Costruisci potenti database con dati scrappati:

```
1. Olostep scrappa o esegue crawling
2. AI - Estrai dati strutturati
3. Airtable - Crea Record
```

**Casi d'Uso:**
- Cataloghi di prodotti
- Database di ricerca
- Calendari di contenuti
- Database di link

### Slack

Ricevi notifiche istantanee con workflow di approvazione:

```
1. Olostep monitora la pagina
2. Filtro - Verifica cambiamenti
3. Passaggio di Approvazione Umana
4. Slack - Invia Messaggio (se approvato)
```

**Casi d'Uso:**
- Avvisi di calo prezzo (con approvazione)
- Notifiche di aggiornamento contenuti
- Monitoraggio errori
- Digest giornalieri

### HubSpot / Salesforce

Arricchisci automaticamente i dati CRM:

```
1. Nuovo contatto aggiunto
2. Olostep scrappa il sito web aziendale
3. AI estrae informazioni chiave
4. Passaggio di Revisione Umana
5. CRM - Aggiorna contatto (se approvato)
```

**Casi d'Uso:**
- Arricchimento lead con controllo qualità
- Ricerca aziendale
- Intelligenza competitiva
- Mappatura account

### Notion

Costruisci basi di conoscenza con contenuti revisionati:

1. Olostep esegue crawling della documentazione
2. AI - Analizza e struttura il contenuto
3. Passaggio di Revisione Umana
4. Notion - Crea Pagine (contenuti approvati)

**Casi d'Uso:**
- Specchi di documentazione

## Risoluzione dei Problemi

<AccordionGroup>
  <Accordion title="Autenticazione Fallita">
    **Errore**: "Chiave API non valida"
    
    **Soluzioni**:
    - Controlla la chiave API dal [dashboard](https://olostep.com/dashboard)
    - Assicurati che non ci siano spazi extra nella chiave API
    - Riconnetti il tuo account Olostep in Relay.app
    - Verifica che la chiave API sia attiva
  </Accordion>

  <Accordion title="Scrape Restituisce Contenuto Vuoto">
    **Errore**: I campi di contenuto sono vuoti
    
    **Soluzioni**:
    - Aumenta il tempo di "Wait Before Scraping"
    - Controlla se il sito richiede il login
    - Prova un formato diverso (HTML vs Markdown)
    - Verifica che l'URL sia accessibile
    - Controlla se il sito blocca l'accesso automatizzato
  </Accordion>

  <Accordion title="Limite di Velocità Superato">
    **Errore**: "Limite di velocità superato"
    
    **Soluzioni**:
    - Distanza le esecuzioni del workflow con ritardi
    - Usa l'elaborazione in batch invece di scrape individuali
    - Aggiorna il tuo piano Olostep
    - Controlla il limite di velocità nel dashboard
  </Accordion>

  <Accordion title="URL Non Scrappato">
    **Errore**: URL specifici non vengono scrappati
    
    **Soluzioni**:
    - Verifica il formato dell'URL (includi http:// o https://)
    - Controlla se l'URL richiede autenticazione
    - Testa l'URL prima nel browser
    - Prova con il parametro del paese
    - Contatta il supporto per domini bloccati
  </Accordion>
</AccordionGroup>

## Relay.app vs Zapier

Relay.app è un'alternativa moderna a Zapier con diversi vantaggi:

**Vantaggi di Relay.app:**
- **Human-in-the-Loop**: Passaggi di revisione manuale integrati
- **AI Integrata**: Funzionalità AI integrate nei workflow
- **Migliore UX**: Interfaccia più intuitiva e messaggi di errore più chiari
- **Prezzi Migliori**: Più conveniente per molti casi d'uso
- **Integrazioni Più Profonde**: Integrazioni più complete con app popolari

**Quando Scegliere Zapier:**
- Necessità di integrazione con app meno comunemente usate
- Preferenza per una piattaforma più consolidata
- Richiesta di funzionalità specifiche solo di Zapier

Per confronti dettagliati, vedi [confronto di Relay.app con Zapier](https://www.relay.app/blog/zapier-alternatives).
