De Olostep n8n integratie brengt krachtige webzoek-, scraping- en crawlingmogelijkheden naar n8n-workflows. Bouw geautomatiseerde workflows die webgegevens van elke website zoeken, extraheren en structureren zonder code te schrijven.

[Begin met Olostep op n8n →](https://www.npmjs.com/package/n8n-nodes-olostep)

## Installatie

### 1. Installeer de Node

Installeer het Olostep node-pakket via npm:

```bash
npm install n8n-nodes-olostep
```

Herstart vervolgens n8n om de nieuwe node te laden.

Volg anders de [n8n community nodes installatiehandleiding](https://docs.n8n.io/integrations/community-nodes/installation/) voor gedetailleerde instructies.

### 2. Verbind Je Account

Wanneer je de Olostep node voor het eerst in een workflow gebruikt, moet je referenties configureren:

1. Voeg de "Olostep Scrape" node toe aan je workflow
2. Klik op de node om de instellingen te openen
3. Klik op "Nieuwe Referentie Maken" of selecteer bestaande referenties
4. Voer je Olostep API-sleutel in
5. Klik op "Opslaan" om de referentie op te slaan

Verkrijg je API-sleutel van het [Olostep Dashboard](https://olostep.com/dashboard).

## Beschikbare Acties

### Website Scrapen

Extraheer inhoud van een enkele URL. Ondersteunt meerdere formaten en JavaScript-rendering.

**Gebruiksscenario's:**
- Monitor specifieke pagina's op wijzigingen
- Extraheer productinformatie van e-commerce sites
- Verzamel gegevens van nieuwsartikelen of blogposts
- Haal inhoud op voor contentaggregatie

**Configuratie:**

<ParamField path="URL om te Scrapen" type="string" required>
  Website URL om te scrapen (moet http:// of https:// bevatten)
</ParamField>

<ParamField path="Uitvoerformaat" type="dropdown" default="Markdown">
  Kies formaat: Markdown, HTML, JSON, of Plain Text
</ParamField>

<ParamField path="Landcode" type="string">
  Landcode voor locatie-specifieke inhoud (bijv. "US", "GB", "CA")
</ParamField>

<ParamField path="Wacht Voor Scraping" type="integer">
  Wachttijd in milliseconden voor JavaScript-rendering (0-10000)
</ParamField>

<ParamField path="Parser" type="string">
  Optionele parser ID voor gespecialiseerde extractie (bijv. "@olostep/amazon-product")
</ParamField>

**Uitvoer Velden:**
- Scrape ID
- Gescrapete URL
- Markdown Inhoud
- HTML Inhoud
- JSON Inhoud
- Tekst Inhoud
- Status
- Tijdstempel
- Screenshot URL (indien beschikbaar)
- Paginametadata

### Batch Scrape URLs

Scrape tot 10k urls tegelijkertijd. Perfect voor grootschalige data-extractie.

**Gebruiksscenario's:**
- Scrape volledige productcatalogi
- Extraheer gegevens van meerdere zoekresultaten
- Verwerk lijsten van URLs uit spreadsheets
- Bulk inhoudsextractie

**Configuratie:**

<ParamField path="URLs om te Scrapen" type="text" required>
  JSON array van objecten met url en custom_id velden. 
  
  Voorbeeld: `[{"url":"https://example.com","custom_id":"site1"}]`
</ParamField>

<ParamField path="Uitvoerformaat" type="dropdown" default="Markdown">
  Kies formaat voor alle URLs: Markdown, HTML, JSON, of Plain Text
</ParamField>

<ParamField path="Landcode" type="string">
  Landcode voor locatie-specifieke scraping
</ParamField>

<ParamField path="Wacht Voor Scraping" type="integer">
  Wachttijd in milliseconden voor JavaScript-rendering
</ParamField>

<ParamField path="Parser" type="string">
  Optionele parser ID voor gespecialiseerde extractie
</ParamField>

**Uitvoer Velden:**
- Batch ID (gebruik dit om later resultaten op te halen)
- Status
- Totaal URLs
- Aangemaakt Op
- Aangevraagd Formaat
- Landcode
- Gebruikte Parser

**Voorbeeld Workflows:**

<AccordionGroup>
  <Accordion title="Scrape Productcatalogus">
    **Trigger:** Webhook - Ontvang POST Verzoek
    
    **Actie:** Code - Formatteer URLs
    - Converteer CSV/lijst naar JSON array formaat
    
    **Actie:** Olostep - Batch Scrape URLs
    - URLs: \{\{$json.urlArray\}\}
    - Formaat: JSON
    - Parser: @olostep/amazon-product
    
    **Actie:** Webhook - POST
    - Stuur batch ID naar je systeem voor ophalen
  </Accordion>

  <Accordion title="Dagelijkse Inhoudsmonitoring">
    **Trigger:** Schema - Elke dag om 6 uur
    
    **Actie:** Google Sheets - Lees Rijen
    - Haal URLs op om te monitoren
    
    **Actie:** Code - Formatteer URLs
    - Converteer naar batch array formaat
    
    **Actie:** Olostep - Batch Scrape URLs
    - Verwerk alle URLs tegelijk
    
    **Actie:** Slack - Stuur Bericht
    - Informeer team dat scraping voltooid is
  </Accordion>
</AccordionGroup>

### Crawl Maken

Verkrijg de inhoud van subpagina's van een URL. Ontdek en scrape autonoom volledige websites door links te volgen. Perfect voor documentatiesites, blogs en inhoudsrepositories.

**Gebruiksscenario's:**
- Crawl en archiveer volledige documentatiesites
- Extraheer alle blogposts van een website
- Bouw kennisbanken van webinhoud
- Monitor wijzigingen in websitestructuur

**Configuratie:**

<ParamField path="Start URL" type="string" required>
  Start-URL voor de crawl (moet http:// of https:// bevatten)
</ParamField>

<ParamField path="Maximaal Aantal Pagina's" type="integer" default="10">

</ParamField>

<ParamField path="Volg Links" type="boolean" default="true">
  Of links op pagina's gevolgd moeten worden
</ParamField>

<ParamField path="Uitvoerformaat" type="dropdown" default="Markdown">
  Formaat voor gescrapete inhoud
</ParamField>

<ParamField path="Landcode" type="string">
  Optionele landcode voor locatie-specifieke crawling
</ParamField>

<ParamField path="Parser" type="string">
  Optionele parser ID voor gespecialiseerde inhoudsextractie
</ParamField>

**Uitvoer Velden:**
- Crawl ID (gebruik dit om later resultaten op te halen)
- Object Type
- Status
- Start URL
- Maximum Aantal Pagina's
- Volg Links
- Aangemaakt Tijdstempel
- Formaten

### Kaart Maken

Verkrijg alle URLs op een website. Extraheer alle URLs van een website voor inhoudsontdekking en site-structuuranalyse.

**Gebruiksscenario's:**
- Bouw sitemaps en site-structuurdiagrammen
- Ontdek alle pagina's voordat je batch scraping doet
- Vind gebroken of ontbrekende pagina's

**Configuratie:**

<ParamField path="Website URL" type="string" required>

</ParamField>

<ParamField path="Zoekopdracht" type="string">

</ParamField>

  Beperk het aantal geretourneerde URLs
</ParamField>

**Uitvoer Velden:**

## Multi-Step Workflows

### Volledige Product Scraping Pipeline

Bouw een uitgebreide productgegevenspipeline:

<Steps>
  <Step title="Ontdek Product URLs">
    Gebruik **Kaart Maken** om alle productpagina's op de doelwebsite te vinden
    - Inclusief patronen: `/products/**`
    - Uitsluit patronen: `/cart/**`, `/checkout/**`
  </Step>
  
  <Step title="Batch Verwerk Producten">
    Gebruik **Batch Scrape URLs** om alle productgegevens te extraheren
    - Formaat: JSON
    - Parser: Product-specifieke parser indien beschikbaar
  </Step>
  
  <Step title="Opslaan in Database">
    Stuur batch ID naar je systeem of wacht en haal resultaten op
    - Gebruik Airtable, Google Sheets, of je database
  </Step>
  
  <Step title="Monitor voor Wijzigingen">
    Plan dagelijkse scrapes om prijs/beschikbaarheidswijzigingen te volgen
    - Vergelijk met bestaande gegevens
    - Waarschuw bij significante wijzigingen
  </Step>
</Steps>

### SEO Inhoudsstrategie

Analyseer concurrenten en plan inhoud:

<Steps>
  <Step title="Kaart Concurrent Sites">
    Gebruik **Kaart Maken** op concurrent websites
    - Extraheer alle blogposts en inhoudspagina's
  </Step>
  
  <Step title="Scrape Inhoud">
    Gebruik **Batch Scrape URLs** om volledige inhoud te krijgen
    - Formaat: Markdown voor eenvoudige analyse
  </Step>
  
  <Step title="AI Analyse">
    Gebruik OpenAI om onderwerpen en trefwoorden te analyseren
    - Identificeer inhoudsgaten
    - Vind trending onderwerpen
  </Step>
  
  <Step title="Creëer Inhoudskalender">
    Voeg inzichten toe aan Notion of Airtable
    - Plan je inhoudsstrategie
  </Step>
</Steps>

## Best Practices

<AccordionGroup>
  <Accordion title="Gebruik Batchverwerking voor Meerdere URLs">
    Wanneer je meer dan 3-5 URLs scrapt, gebruik **Batch Scrape URLs** in plaats van meerdere **Scrape Website** acties. Batchverwerking is:
    - Veel sneller (parallelle verwerking)
    - Kosteneffectiever
    - Makkelijker te beheren
    - Beter voor snelheidslimieten
  </Accordion>

  <Accordion title="Stel Geschikte Wachttijden In">
    Voor JavaScript-zware sites, gebruik de "Wacht Voor Scraping" parameter:
    - Eenvoudige sites: 0-1000ms
    - Dynamische sites: 2000-3000ms
    - Zware JavaScript: 5000-8000ms
    
    Test met verschillende waarden om de optimale wachttijd te vinden.
  </Accordion>

  <Accordion title="Gebruik Gespecialiseerde Parsers">
    Gebruik vooraf gebouwde parsers (bijv. Amazon, Google, en taak-specifieke parsers uit de Olostep Store zoals e-mails, socials, kalenders):
    - Krijg automatisch gestructureerde gegevens
    - Betrouwbaardere extractie
    - Geen behoefte aan aangepaste parsing
    - Onderhouden door Olostep
  </Accordion>

  <Accordion title="Filter Voor Scraping">
    Gebruik n8n's IF node om onnodige scrapes te vermijden:
    - Controleer of URL is veranderd
    - Verifieer dat gegevens niet recent zijn gescrapet
    - Pas bedrijfslogica toe voor scraping
    
    Dit bespaart API-credits en uitvoeringstijd.
  </Accordion>

  <Accordion title="Behandel Asynchrone Operaties">
    Batch, Crawl, en Map operaties zijn asynchroon:
    - Sla de geretourneerde ID op (batch_id, crawl_id, map_id)
    - Gebruik een Wacht node als je direct ophaalt
    - Overweeg webhook callbacks voor voltooiing
    - Stel aparte workflows in voor ophalen
  </Accordion>

  <Accordion title="Sla Resultaten Correct Op">
    Kies de juiste opslag op basis van je behoeften:
    - **Google Sheets**: Eenvoudige tracking, team samenwerking
    - **Airtable**: Relationele gegevens, rijke opmaak
    - **Database**: Grootschalig, complexe queries
    - **Notion**: Kennisbank, documentatie
  </Accordion>

  <Accordion title="Monitor en Waarschuw">
    Stel monitoring in voor je scraping workflows:
    - Gebruik Foutworkflows in n8n
    - Stuur waarschuwingen naar Slack/E-mail bij fouten
    - Volg API-gebruik in Olostep dashboard
    - Log belangrijke statistieken
  </Accordion>
</AccordionGroup>

## Probleemoplossing

<AccordionGroup>
  <Accordion title="Authenticatie Mislukt">
    **Fout**: "Ongeldige API-sleutel"
    
    **Oplossingen**:
    - Controleer API-sleutel van [dashboard](https://olostep.com/dashboard)
    - Zorg ervoor dat er geen extra spaties in de API-sleutel staan
    - Maak de referentie opnieuw aan in n8n
    - Verifieer dat de API-sleutel actief is
  </Accordion>

  <Accordion title="Scrape Geeft Lege Inhoud Terug">
    **Fout**: Inhoudsvelden zijn leeg
    
    **Oplossingen**:
    - Verhoog "Wacht Voor Scraping" tijd
    - Controleer of website inloggen vereist
    - Probeer een ander formaat (HTML vs Markdown)
    - Verifieer dat URL toegankelijk is
    - Controleer of site geautomatiseerde toegang blokkeert
  </Accordion>

  <Accordion title="Batch Array Formaat Fout">
    **Fout**: "Ongeldig JSON-formaat voor batch array"
    
    **Oplossingen**:
    - Gebruik formaat: `[{"url":"https://example.com","custom_id":"id1"}]`
    - Zorg voor correcte JSON-syntaxis
    - Gebruik Code node om URLs correct te formatteren
    - Test JSON met online validator
  </Accordion>

  <Accordion title="Snelheidslimiet Overschreden">
    **Fout**: "Snelheidslimiet overschreden"
    
    **Oplossingen**:
    - Spreid workflow-uitvoeringen met Wacht nodes
    - Gebruik batchverwerking in plaats van individuele scrapes
    - Upgrade je Olostep-plan
    - Controleer snelheidslimiet in dashboard
  </Accordion>

  <Accordion title="URL Niet Gescrapet">
    **Fout**: Specifieke URLs falen bij scrapen
    
    **Oplossingen**:
    - Verifieer URL-formaat (inclusief http:// of https://)
    - Controleer of URL authenticatie vereist
    - Test URL eerst in browser
    - Probeer met landparameter
    - Neem contact op met ondersteuning voor geblokkeerde domeinen
  </Accordion>
</AccordionGroup>

## n8n Voordelen

### Zelf-Gehost
n8n is zelf-gehost, waardoor je volledige controle hebt over je workflows en gegevens. Geen vendor lock-in, geen gegevens die je infrastructuur verlaten.

### Geen Taaklimieten
In tegenstelling tot cloud-gebaseerde automatiseringsplatforms, legt n8n geen taaklimieten op. Voer zoveel workflows uit als je nodig hebt zonder extra kosten.

### Open Source
n8n is open source, waardoor je het kunt aanpassen en uitbreiden om aan je specifieke behoeften te voldoen.

### Kosteneffectief
Zelf-gehoste n8n is gratis, met optionele cloudhosting beschikbaar. Betaal alleen voor het gebruik van de Olostep API.

## Prijzen

Olostep rekent op basis van API-gebruik, onafhankelijk van n8n:

- **Scrapes**: Betaal per scrape
- **Batches**: Betaal per URL in batch
- **Crawls**: Betaal per gecrawlde pagina
- **Kaarten**: Betaal per kaartoperatie

Bekijk de huidige prijzen op [olostep.com/pricing](https://olostep.com/pricing).

**n8n**: Zelf-gehoste n8n is gratis. Cloudhosting beschikbaar met optionele betaalde plannen.
