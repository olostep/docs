Je OpenClaw-agent kan documentatie lezen, maar het kan het web niet lezen. De [Olostep Web Agent plugin](https://clawhub.ai/plugins/olostep-web-agent) lost dat op — zoek, schraap, kruip en extraheer gestructureerde gegevens van elke website, direct binnen je agent workflow.

Eén installatie geeft je **13 vaardigheden** voor taken op hoog niveau (debug een fout van live StackOverflow threads, schrijf integratiecode vanuit actuele documentatie, onderzoek tools met gestructureerde vergelijkingen) en een **9-tool MCP-server** voor directe programmatische toegang. JS-zware SPA's, CAPTCHA's, Cloudflare, residentiële proxies — automatisch afgehandeld.

## Installatie

```bash
clawhub install olostep
```

Dat is alles. Als je de MCP-server handmatig wilt aansluiten, voeg dit toe aan je OpenClaw-configuratie:

```json
{
  "mcpServers": {
    "olostep": {
      "command": "npx",
      "args": ["-y", "olostep-mcp"],
      "env": {
        "OLOSTEP_API_KEY": "your-api-key-here"
      }
    }
  }
}
```

Verkrijg een gratis API-sleutel op [olostep.com/auth](https://olostep.com/auth) — 500 verzoeken/maand, geen creditcard nodig.

## MCP Tools

De meegeleverde MCP-server (`olostep-mcp`) geeft je agent 9 tools die het direct kan aanroepen. Gebruik deze wanneer je fijnmazige controle nodig hebt die verder gaat dan wat de workflowvaardigheden bieden.

| Tool | Wat het doet |
|------|-------------|
| `scrape_website` | Schraap een enkele URL naar markdown, HTML, JSON of tekst |
| `get_webpage_content` | Haal een webpagina op als schone, LLM-klaar markdown |
| `search_web` | Zoek op het live web, krijg AI-gesynthetiseerde antwoorden |
| `google_search` | Gestructureerde Google SERP-gegevens — organische resultaten, PAA, kennisgrafiek |
| `answers` | Stel een vraag, krijg een geciteerd antwoord met optionele gestructureerde JSON-output |
| `batch_scrape_urls` | Schraap tot 10.000 URL's parallel met volledige weergave |
| `create_crawl` | Kruip een website door links te volgen vanaf een start-URL |
| `create_map` | Ontdek alle URL's op een website, filterbaar op patroon en query |
| `get_website_urls` | Vind en haal relevante URL's op van een specifiek domein |

## Hoe Dit Er In De Praktijk Uitziet

### "Waarom faalt dit?" — Debug vanaf het live web

Je plakt `ECONNRESET bij het aanroepen van Stripe webhook endpoint` in je agent. Het zoekt GitHub-issues en StackOverflow voor die exacte fout, schraapt de drie meest relevante threads en geeft een concrete oplossing — niet "controleer je netwerkinstellingen," maar de daadwerkelijke timeout-configuratie die het oploste voor andere ontwikkelaars die tegen dezelfde muur aanliepen.

### "Schrijf de integratie" — Code van actuele documentatie, niet van verouderde trainingsgegevens

Je moet een betalings-API integreren. De `docs-to-code` vaardigheid schraapt de huidige API-referentie — degene die vandaag is gepubliceerd, niet de versie waarop je model zes maanden geleden is getraind — en schrijft werkende code met de parameters en eindpunten die daadwerkelijk bestaan.

### "Welke moet ik gebruiken?" — Gestructureerde toolvergelijkingen

ORM's evalueren? Auth-providers vergelijken? De `research` vaardigheid zoekt meerdere bronnen, schraapt echte prijspagina's en functiematrices en geeft een gestructureerde vergelijkingstabel met citaten. Je krijgt een aanbeveling ondersteund door wat er daadwerkelijk op de website van elk product staat, niet door de populariteit van trainingsgegevens.

### Bouw een RAG-pijplijn van elke documentatiesite

```
map   → ontdek elke URL op de documentatiesite
batch → schraap alle pagina's parallel als schone markdown
      → voer in je vectorstore
```

Twee vaardigheden, één pijplijn. Werkt op documentatiesites met honderden of duizenden pagina's.

### Extraheer gestructureerde gegevens op schaal

```
map   → vind alle product-, lijst- of vacature-URL's
batch → schraap elke pagina met een vooraf gebouwde parser → getypte JSON
      → voer in je database, API of seed-bestanden
```

### Migreer naar een nieuwe frameworkversie

Je upgrade van Next.js 13 naar 15. De `migrate-code` vaardigheid schraapt de officiële migratiehandleiding, extraheert elke brekende verandering met voor/na-codepatronen en herschrijft je bestanden om te passen — gebaseerd op de echte documentatie, niet op de beste gok van het model.

## Vooraf Gebouwde Parsers

Geef de `parser` parameter door aan elke schraapoproep en krijg getypte JSON terug in plaats van ruwe inhoud. Geen schema-definitie nodig — deze verzorgen de extractie voor je.

| Parser | Geeft terug |
|--------|------------|
| `@olostep/google-search` | Organische resultaten, kennisgrafiek, People Also Ask |
| `@olostep/amazon-it-product` | Prijs, beoordeling, functies, beschikbaarheid |
| `@olostep/extract-emails` | Elk e-mailadres gevonden op de pagina |
| `@olostep/extract-calendars` | Gestructureerde kalenderevenementen |
| `@olostep/extract-socials` | Links naar sociale mediaprofielen |
