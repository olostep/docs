## Overzicht

Creëert een nieuwe AirOps batch die:
1. Alle opgegeven URL's scrapt
2. Gestructureerde metadata automatisch extraheert
3. Items met ontbrekende velden identificeert
4. Onvolledige items verrijkt via LLM
5. Geünificeerde resultaten voor alle URL's retourneert

## Verzoekparameters

<ParamField body="items" type="array" required>
  Array van URL's om te verwerken. Elk item vereist:
  - `url` - Geldige URL om te scrapen
  - `custom_id` - Jouw unieke identificatie (1-255 tekens)
</ParamField>

<ParamField body="brands" type="array" default="[]">
  Merknamen om naar te zoeken in de pagina-inhoud. De LLM zal vermeldingen van deze specifieke merken detecteren en overeenkomsten retourneren in het `brand_mentions` veld.
  
  ```json
  {
    "brands": ["Acme Corp", "TechBrand", "SaaS Pro"]
  }
  ```
</ParamField>

<ParamField body="country" type="string" default="RANDOM">
  Land voor geo-gerichte scraping. Gebruik ISO 3166-1 alpha-2 codes (`US`, `GB`, `DE`) of `RANDOM`.
</ParamField>

<ParamField body="webhook" type="string">
  URL om webhookmeldingen te ontvangen. Alle levenscyclusgebeurtenissen voor deze batch worden hierheen gestuurd.
</ParamField>

<ParamField body="metadata" type="object">
  Aangepaste sleutel-waarde paren gekoppeld aan de batch. Stripe-stijl: max 50 sleutels, sleutel ≤40 tekens, waarde ≤500 tekens.
</ParamField>
