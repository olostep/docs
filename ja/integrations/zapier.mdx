OlostepのZapier統合は、Zapierの8,000以上のアプリのエコシステムに強力なウェブスクレイピング機能をもたらします。コードを書くことなくウェブデータを抽出、監視、処理する自動化ワークフローを構築します。

[OlostepをZapierで始める →](https://zapier.com/apps/olostep/integrations)

## 特徴

この統合は、自動化されたウェブデータ抽出とAI駆動のリサーチのための5つの強力なアクションを提供します：

## インストール

### 1. ZapierでOlostepを見つける

Zapierアプリディレクトリで「Olostep」を検索するか、新しいZapを作成するときに：

1. [ZapierでのOlostep](https://zapier.com/apps/olostep/integrations)にアクセス
2. 「Create Zap」をクリック
3. アプリセレクターで「Olostep」を検索
4. Olostepアプリを選択

または、[Olostep統合ページ](https://zapier.com/apps/olostep/integrations)に直接アクセスして、利用可能なアクションを閲覧し、最初のZapを作成します。

### 2. アカウントを接続

Zapで初めてOlostepを使用するとき、アカウントを接続するよう求められます：

1. 「Sign in to Olostep」をクリック
2. Olostep APIキーを入力
3. 「Yes, Continue to Olostep」をクリック

APIキーは[Olostepダッシュボード](https://olostep.com/dashboard)から取得できます。

## 利用可能なアクション

### ウェブサイトをスクレイプ

単一のURLからコンテンツを抽出。複数の形式とJavaScriptレンダリングをサポート。

**使用例:**
- 特定のページの変更を監視
- eコマースサイトから商品情報を抽出
- ニュース記事やブログ投稿からデータを収集
- コンテンツ集約のためのコンテンツを引き出す

**設定:**

<ParamField path="スクレイプするURL" type="string" required>
  スクレイプするウェブサイトのURL（http://またはhttps://を含む必要があります）
</ParamField>

<ParamField path="出力形式" type="dropdown" default="Markdown">
  形式を選択：Markdown、HTML、JSON、またはプレーンテキスト
</ParamField>

<ParamField path="国コード" type="string">
  ロケーション固有のコンテンツのための国コード（例："US", "GB", "CA"）
</ParamField>

<ParamField path="スクレイプ前の待機時間" type="integer">
  JavaScriptレンダリングのための待機時間（0-10000ミリ秒）
</ParamField>

<ParamField path="パーサー" type="string">
  特殊な抽出のためのオプションのパーサーID（例："@olostep/amazon-product"）
</ParamField>

**出力フィールド:**
- スクレイプID
- スクレイプされたURL
- Markdownコンテンツ
- HTMLコンテンツ
- JSONコンテンツ
- テキストコンテンツ
- ステータス
- タイムスタンプ
- スクリーンショットURL（利用可能な場合）
- ページメタデータ

**例ワークフロー:**

<AccordionGroup>
  <Accordion title="競合他社の価格を監視">
    **トリガー:** スケジュール（毎日午前9時）
    
    **アクション:** Olostep - ウェブサイトをスクレイプ
    - URL: 競合他社の製品ページ
    - 形式: JSON
    - パーサー: @olostep/amazon-product
    
    **アクション:** Google Sheets - 行を作成
    - 価格データを追跡スプレッドシートに追加
    
    **アクション:** Gmail - メールを送信（価格が下がった場合）
    - 価格変更についてチームに通知
  </Accordion>

  <Accordion title="ブログ投稿を抽出して保存">
    **トリガー:** RSS by Zapier - フィード内の新しいアイテム
    
    **アクション:** Olostep - ウェブサイトをスクレイプ
    - URL: \{\{投稿URL\}\}
    - 形式: Markdown
    
    **アクション:** Notion - ページを作成
    - 記事コンテンツをNotionデータベースに保存
  </Accordion>

  <Accordion title="リードエンリッチメント">
    **トリガー:** Google Sheets - 新しい行
    
    **アクション:** Olostep - ウェブサイトをスクレイプ
    - URL: シートからの会社のウェブサイト
    - 形式: Markdown
    
    **アクション:** OpenAI - テキストを完了
    - AIを使用して会社情報を抽出
    
    **アクション:** Google Sheets - 行を更新
    - 強化されたデータをシートに追加
  </Accordion>
</AccordionGroup>

### URLを一括スクレイプ

複数のURLを並行して処理（最大100,000件同時）。大規模なデータ抽出に最適。

**使用例:**
- 製品カタログ全体をスクレイプ
- 複数の検索結果からデータを抽出
- スプレッドシートからのURLリストを処理
- 大量のコンテンツ抽出

**設定:**

<ParamField path="スクレイプするURL" type="text" required>
  urlとcustom_idフィールドを持つオブジェクトのJSON配列。 
  
  例: `[{"url":"https://example.com","custom_id":"site1"}]`
</ParamField>

<ParamField path="出力形式" type="dropdown" default="Markdown">
  すべてのURLの形式を選択：Markdown、HTML、JSON、またはプレーンテキスト
</ParamField>

<ParamField path="国コード" type="string">
  ロケーション固有のスクレイピングのための国コード
</ParamField>

<ParamField path="スクレイプ前の待機時間" type="integer">
  JavaScriptレンダリングのための待機時間
</ParamField>

<ParamField path="パーサー" type="string">
  特殊な抽出のためのオプションのパーサーID
</ParamField>

**出力フィールド:**
- バッチID（後で結果を取得するために使用）
- ステータス
- 総URL数
- 作成日時
- 要求された形式
- 国コード
- 使用されたパーサー

**例ワークフロー:**

<AccordionGroup>
  <Accordion title="製品カタログをスクレイプ">
    **トリガー:** Webhook - POSTリクエストを受信
    
    **アクション:** Code by Zapier - Pythonを実行
    - CSV/リストをJSON配列形式に変換
    
    **アクション:** Olostep - URLを一括スクレイプ
    - URL: \{\{前のステップからのJSON配列\}\}
    - 形式: JSON
    - パーサー: @olostep/amazon-product
    
    **アクション:** Webhook - POST
    - バッチIDをシステムに送信して取得
  </Accordion>

  <Accordion title="毎日のコンテンツ監視">
    **トリガー:** スケジュール - 毎日午前6時
    
    **アクション:** Google Sheets - 行を取得
    - 監視するURLを取得
    
    **アクション:** Code by Zapier - URLをフォーマット
    - バッチ配列形式に変換
    
    **アクション:** Olostep - URLを一括スクレイプ
    - すべてのURLを一度に処理
    
    **アクション:** Slack - メッセージを送信
    - スクレイピング完了をチームに通知
  </Accordion>
</AccordionGroup>

### クロールを作成

リンクをたどってウェブサイト全体を自律的に発見しスクレイプ。ドキュメントサイト、ブログ、コンテンツリポジトリに最適。

**使用例:**
- ドキュメントサイト全体をクロールしてアーカイブ
- ウェブサイトからすべてのブログ投稿を抽出
- ウェブコンテンツからナレッジベースを構築
- ウェブサイト構造の変更を監視

**設定:**

<ParamField path="開始URL" type="string" required>
  クロールの開始URL（http://またはhttps://を含む必要があります）
</ParamField>

<ParamField path="最大ページ数" type="integer" default="10">
  クロールする最大ページ数
</ParamField>

<ParamField path="リンクをたどる" type="boolean" default="true">
  ページで見つかったリンクをたどるかどうか
</ParamField>

<ParamField path="出力形式" type="dropdown" default="Markdown">
  スクレイプされたコンテンツの形式
</ParamField>

<ParamField path="国コード" type="string">
  ロケーション固有のクロールのためのオプションの国コード
</ParamField>

<ParamField path="パーサー" type="string">
  特殊なコンテンツ抽出のためのオプションのパーサーID
</ParamField>

**出力フィールド:**
- クロールID（後で結果を取得するために使用）
- オブジェクトタイプ
- ステータス
- 開始URL
- 最大ページ数
- リンクをたどる
- 作成タイムスタンプ
- 形式

**例ワークフロー:**

<AccordionGroup>
  <Accordion title="ドキュメントサイトをアーカイブ">
    **トリガー:** スケジュール - 毎月1日の午前12時
    
    **アクション:** Olostep - クロールを作成
    - 開始URL: https://docs.example.com
    - 最大ページ数: 500
    - リンクをたどる: true
    - 形式: Markdown
    
    **アクション:** Webhook - POST
    - クロールIDをアーカイブシステムに送信
    
    **アクション:** Slack - メッセージを送信
    - クロールが進行中であることをチームに通知
  </Accordion>

  <Accordion title="競合他社のコンテンツ分析">
    **トリガー:** スケジュール - 毎週月曜日の午前9時
    
    **アクション:** Olostep - クロールを作成
    - 開始URL: 競合他社のブログURL
    - 最大ページ数: 100
    - 形式: Markdown
    
    **アクション:** Delay - 10分間待機
    - クロールが完了するまで待機
    
    **アクション:** Airtable - レコードを作成
    - 分析のためにクロールデータを保存
  </Accordion>
</AccordionGroup>

### マップを作成

コンテンツ発見とサイト構造分析のためにウェブサイトからすべてのURLを抽出。

**使用例:**
- サイトマップとサイト構造図を構築
- 一括スクレイプ前にすべてのページを発見
- 壊れたページや欠落したページを見つける
- SEO監査と分析

**設定:**

<ParamField path="ウェブサイトURL" type="string" required>
  リンクを抽出するウェブサイトのURL（http://またはhttps://を含む必要があります）
</ParamField>

<ParamField path="検索クエリ" type="string">
  URLをフィルタリングするためのオプションの検索クエリ（例："blog"）
</ParamField>

<ParamField path="上位N件のURL" type="integer">
  返されるURLの数を制限
</ParamField>

<ParamField path="含むURLパターン" type="string">
  特定のパスを含むためのグロブパターン（例："/blog/**"）
</ParamField>

<ParamField path="除外URLパターン" type="string">
  特定のパスを除外するためのグロブパターン（例："/admin/**"）
</ParamField>

**出力フィールド:**
- マップID
- オブジェクトタイプ
- ウェブサイトURL
- 発見された総URL数
- URL（JSON配列）
- 検索クエリ
- 上位N件の制限

**例ワークフロー:**

<AccordionGroup>
  <Accordion title="発見してスクレイプ">
    **トリガー:** Zapierでのボタンクリック
    
    **アクション:** Olostep - マップを作成
    - URL: https://example.com
    - 含むパターン: /products/**
    - 上位N件: 500
    
    **アクション:** Code by Zapier - URLを抽出
    - マップ結果からURLを解析
    
    **アクション:** Olostep - URLを一括スクレイプ
    - URL: \{\{マップからのURL\}\}
    - 形式: JSON
    
    **アクション:** Google Sheets - 行を作成
    - すべての製品データをスプレッドシートに追加
  </Accordion>

  <Accordion title="SEOサイト監査">
    **トリガー:** スケジュール - 毎月
    
    **アクション:** Olostep - マップを作成
    - URL: あなたのウェブサイト
    - 上位N件: 1000
    
    **アクション:** Airtable - レコードを作成
    - 追跡のためにすべてのURLを保存
    
    **アクション:** Slack - メッセージを送信
    - 発見された総ページ数を報告
  </Accordion>
</AccordionGroup>

### AIに質問

ウェブ検索や提供されたコンテキストURLを使用して質問にAI駆動の回答を得る。リサーチ自動化、コンテンツ生成、データ分析に最適。

**使用例:**
- 引用付きのリサーチ自動化
- 複数のURLからコンテンツを要約
- ソース参照付きでレポートを生成
- 特定のウェブページを使用して質問に回答
- コンテンツ分析と抽出

**設定:**

<ParamField path="質問" type="string" required>
  Olostep Answersに回答してもらいたい質問
</ParamField>

<ParamField path="コンテキストURL（JSON配列）" type="string">
  回答を基にするためのオプションのJSON配列のURL（例：["https://example.com/page1", "https://example.com/page2"]）
</ParamField>

<ParamField path="検索クエリ" type="string">
  コンテキストURLが提供されていない場合にウェブからソースを取得するためのオプションのクエリ（例："site:example.com pricing"）
</ParamField>

<ParamField path="ソースの数" type="integer">
  回答を生成するために使用するソースの数を制限
</ParamField>

<ParamField path="形式" type="dropdown" default="Markdown">
  回答形式を選択：Markdown、JSON、またはプレーンテキスト
</ParamField>

<ParamField path="引用を含める" type="boolean" default="true">
  応答に引用/ソースを含めるかどうか
</ParamField>

**出力フィールド:**
- 回答ID
- オブジェクトタイプ
- 質問
- 回答（テキスト）
- 回答（Markdown）
- 回答（JSON）
- 引用（URL、タイトル、スニペットを含むJSON配列）
- 作成タイムスタンプ
- 形式
- コンテキストURL
- 検索クエリ
- ソースの数

**例ワークフロー:**

## 人気のワークフロー例

### Eコマース価格監視

競合他社の価格を監視し、即時アラートを受け取る：

```
トリガー: スケジュール（毎時）
↓
アクション: Olostep - ウェブサイトをスクレイプ
  - URL: 競合他社の製品ページ
  - 形式: JSON
  - パーサー: @olostep/amazon-product
↓
アクション: フィルター（価格が変更された場合のみ続行）
↓
アクション: Slack - メッセージを送信
  - アラート: "価格が$\{\{price\}\}に変更されました"
```

### コンテンツ集約

複数のソースからコンテンツを集約：

```
トリガー: Google Sheets - 新しい行
↓
アクション: Olostep - ウェブサイトをスクレイプ
  - URL: \{\{シートからのURL\}\}

アクション: Airtable - レコードを作成
  - 記事を要約付きで保存
```

### リードエンリッチメントパイプライン

ウェブ情報でリードデータを強化：

```
トリガー: HubSpot - 新しい連絡先
↓
アクション: Olostep - ウェブサイトをスクレイプ
  - URL: \{\{会社のウェブサイト\}\}
  - 形式: Markdown

アクション: HubSpot - 連絡先を更新
  - 強化されたデータを連絡先に追加
```

### リサーチ自動化

複数のソースからのリサーチを自動化：

```
トリガー: Airtable - 新しいレコード

アクション: Olostep - マップを作成

  - 形式: Markdown
↓
アクション: Notion - ページを作成
  - リサーチデータベースを作成
```

### ソーシャルメディア監視

言及とコンテンツを追跡：

```
トリガー: スケジュール（6時間ごと）
↓
アクション: Olostep - ウェブサイトをスクレイプ
  - URL: ニュースサイトの検索ページ
  - 形式: HTML
↓
アクション: Code - 言及を抽出
  - ブランドの言及を見つける
↓
アクション: Google Sheets - 行を作成
  - タイムスタンプ付きで言及を記録
```

## マルチステップワークフロー

### 完全な製品スクレイピングパイプライン

包括的な製品データパイプラインを構築：

<Steps>
  <Step title="製品URLを発見">
    **マップを作成**を使用してターゲットウェブサイト上のすべての製品ページを見つける
    - 含むパターン: `/products/**`
    - 除外パターン: `/cart/**`, `/checkout/**`
  </Step>
  
  <Step title="製品を一括処理">
    **URLを一括スクレイプ**を使用してすべての製品データを抽出
    - 形式: JSON
    - パーサー: 製品固有のパーサーが利用可能な場合
  </Step>
  
  <Step title="データベースに保存">
    バッチIDをシステムに送信するか、待機して結果を取得
    - Airtable、Google Sheets、またはデータベースを使用
  </Step>
  
  <Step title="変更を監視">

## ベストプラクティス

<AccordionGroup>
  <Accordion title="複数のURLにはバッチ処理を使用">
    3-5以上のURLをスクレイプする場合は、複数の**ウェブサイトをスクレイプ**アクションの代わりに**URLを一括スクレイプ**を使用します。バッチ処理は：
    - はるかに速い（並行処理）
    - よりコスト効果が高い
    - 管理が簡単
    - レート制限に優しい
  </Accordion>

  <Accordion title="適切な待機時間を設定">
    JavaScriptが多いサイトには、「スクレイプ前の待機時間」パラメータを使用：
    - シンプルなサイト: 0-1000ms
    - 動的サイト: 2000-3000ms
    - 重いJavaScript: 5000-8000ms
    
    最適な待機時間を見つけるために異なる値でテストしてください。
  </Accordion>

  <Accordion title="専門パーサーを使用">
    人気のあるウェブサイト（Amazon、LinkedIn、Google）には事前構築されたパーサーを使用：
    - 構造化データを自動的に取得
    - より信頼性の高い抽出
    - カスタムパーシングの必要なし
    - Olostepによって維持
  </Accordion>

  <Accordion title="スクレイプ前にフィルタリング">
    不要なスクレイプを避けるためにZapierのフィルターアクションを使用：
    - URLが変更されたかどうかを確認
    - 最近スクレイプされていないことを確認
    - スクレイプ前にビジネスロジックを適用
    
    これによりAPIクレジットと実行時間が節約されます。
  </Accordion>

  <Accordion title="非同期操作を処理">
    バッチ、クロール、マップ操作は非同期です：
    - 返されたID（batch_id、crawl_id、map_id）を保存
    - すぐに取得する場合はDelayアクションを使用
    - 完了時のWebhookコールバックを検討
    - 取得のために別のZapを設定
  </Accordion>

  <Accordion title="結果を適切に保存">
    ニーズに応じて適切なストレージを選択：
    - **Google Sheets**: シンプルな追跡、チームコラボレーション
    - **Airtable**: リレーショナルデータ、リッチフォーマット
    - **データベース**: 大規模、複雑なクエリ
    - **Notion**: ナレッジベース、ドキュメント
  </Accordion>

  <Accordion title="監視とアラート">
    スクレイピングワークフローの監視を設定：
    - Zap内のエラーパスを使用
    - 失敗時にSlack/Emailにアラートを送信
    - OlostepダッシュボードでAPI使用状況を追跡
    - 重要なメトリクスをログ
  </Accordion>
</AccordionGroup>

## トラブルシューティング

<AccordionGroup>
  <Accordion title="認証失敗">
    **エラー**: "無効なAPIキー"
    
    **解決策**:
    - [ダッシュボード](https://olostep.com/dashboard)からAPIキーを確認
    - APIキーに余分なスペースがないことを確認
    - ZapierでOlostepアカウントを再接続
    - APIキーがアクティブであることを確認
  </Accordion>

  <Accordion title="スクレイプが空のコンテンツを返す">
    **エラー**: コンテンツフィールドが空
    
    **解決策**:
    - 「スクレイプ前の待機時間」を増やす
    - ウェブサイトがログインを必要とするか確認
    - 異なる形式（HTML vs Markdown）を試す
    - URLがアクセス可能であることを確認
    - サイトが自動アクセスをブロックしているか確認
  </Accordion>

  <Accordion title="バッチ配列形式エラー">
    **エラー**: "バッチ配列のJSON形式が無効"
    
    **解決策**:
    - 形式を使用: `[{"url":"https://example.com","custom_id":"id1"}]`
    - 正しいJSON構文を確認
    - URLを正しくフォーマットするためにCodeステップを使用
    - オンラインバリデーターでJSONをテスト
  </Accordion>

  <Accordion title="レート制限超過">
    **エラー**: "レート制限超過"
    
    **解決策**:
    - Zapの実行を遅延で間隔を空ける
    - 個別のスクレイプの代わりにバッチ処理を使用
    - Olostepプランをアップグレード
    - ダッシュボードでレート制限を確認
  </Accordion>

  <Accordion title="URLがスクレイプされない">
    **エラー**: 特定のURLがスクレイプに失敗
    
    **解決策**:
    - URL形式を確認（http://またはhttps://を含む）
    - URLが認証を必要とするか確認
    - まずブラウザでURLをテスト
    - 国パラメータで試す
    - ブロックされたドメインについてサポートに連絡
  </Accordion>
</AccordionGroup>

## Zapierの制限と回避策

### タスク制限
Zapierにはプランに基づくタスク制限があります。各Olostepアクションは1タスクとしてカウントされます。

**回避策**: 複数のURLを単一のタスクとしてスクレイプするためにバッチ処理を使用。

### 実行時間
Zapは30秒後にタイムアウトします。クロールや大規模なバッチはそれ以上かかる場合があります。

**回避策**: IDを保存し、別のZapで結果を取得するか、Webhookを使用。

### データサイズ
Zapierにはタスクごとのデータサイズ制限があります。

**回避策**: 出力でホストされたURLを使用して、大きなコンテンツを別途取得。

### ポーリング対インスタント
トリガーはポーリングベース（5-15分ごとにチェック）です。

**回避策**: インスタント通知にはWebhookを使用するか、特定の時間にスケジュール。

## 価格

OlostepはAPI使用量に基づいて課金され、Zapierとは独立しています：

- **スクレイプ**: スクレイプごとに支払い
- **バッチ**: バッチ内のURLごとに支払い
- **クロール**: クロールされたページごとに支払い
- **マップ**: マップ操作ごとに支払い

現在の価格は[olostep.com/pricing](https://olostep.com/pricing)で確認してください。

**Zapierプラン**: Zapを実行するにはアクティブなZapierプランも必要です。
