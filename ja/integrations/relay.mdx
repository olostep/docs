## インストール

### 1. Relay.appでOlostepを見つける

新しいワークフローを作成する際にRelay.appで「Olostep」を検索します：

1. [Relay.appでのOlostep](https://www.relay.app/apps/olostep/integrations)に移動
2. 「無料で始める」または「ワークフローを作成」をクリック
3. アプリセレクターで「Olostep」を検索
4. Olostepアプリを選択

または、[Olostep統合ページ](https://www.relay.app/apps/olostep/integrations)に直接アクセスして、利用可能なアクションを閲覧し、最初のワークフローを作成します。

### 2. アカウントを接続する

ワークフローで初めてOlostepを使用する際に、アカウント接続を求められます：

1. 「Olostepを接続」をクリック
2. Olostep APIキーを入力
3. 接続を承認

APIキーは[Olostepダッシュボード](https://olostep.com/dashboard)から取得できます。

## 利用可能なアクション

### URLをスクレイプ

単一のURLからコンテンツを抽出します。複数のフォーマットとJavaScriptレンダリングをサポート。

**ユースケース:**
- 特定のページの変更を監視
- eコマースサイトから商品情報を抽出
- ニュース記事やブログ投稿からデータを収集
- コンテンツ集約のためのコンテンツを引き出す

**設定:**

<ParamField path="URL" type="string" required>
  スクレイプするウェブサイトのURL（http://またはhttps://を含む必要があります）
</ParamField>

<ParamField path="Format" type="dropdown" default="Markdown">
  フォーマットを選択：Markdown、HTML、JSON、またはプレーンテキスト
</ParamField>

<ParamField path="Country Code" type="string">
  ロケーション固有のコンテンツのための国コード（例："US", "GB", "CA"）
</ParamField>

<ParamField path="Wait Before Scraping" type="integer">
  JavaScriptレンダリングのための待機時間（ミリ秒単位、0-10000）
</ParamField>

<ParamField path="Parser" type="string">
  特殊な抽出のためのオプションのパーサーID（例："@olostep/amazon-product"）
</ParamField>

**出力フィールド:**
- スクレイプID
- スクレイプされたURL
- Markdownコンテンツ
- HTMLコンテンツ
- JSONコンテンツ
- テキストコンテンツ
- ステータス
- タイムスタンプ
- スクリーンショットURL（利用可能な場合）
- ページメタデータ

**例のワークフロー:**

## Relay.appの利点

Relay.appは従来の自動化プラットフォームに比べていくつかの利点を提供します：

### 人間のループワークフロー

ワークフローに手動レビューのステップを直接追加：

```
1. Olostepがウェブサイトをスクレイプ
2. 人間のレビューのステップ ← データをレビューして承認
3. 承認された場合のみデータベースに保存
```

最適な用途：
- データ処理前の品質管理
- 抽出されたコンテンツに対する人間の判断
- 機密データの承認ワークフロー
- AI生成の要約のレビュー

### 組み込みAI機能

外部統合なしでワークフローにAIを使用：

- **AI意思決定**: スクレイプされたコンテンツに基づいてワークフローパスをAIに決定させる
- **スマートフィルタリング**: スクレイプされたデータを自動的にフィルタリングおよび分類
- **コンテンツ要約**: 保存前に長い記事を要約
- **データ抽出**: 構造化されていないコンテンツからAIを使用して構造化データを抽出

### 優れたUXとテスト

Relay.appの使いやすいインターフェースでワークフローの構築が簡単に：

- 明確なテスト情報
- 役立つエラーメッセージ
- ビジュアルワークフロービルダー
- 簡単なデバッグとトラブルシューティング

## 人気のワークフロー例

### レビュー付きのEコマース価格監視

人間の承認を伴う競合価格の監視：

```
トリガー: スケジュール（毎時）
↓
アクション: Olostep - URLをスクレイプ
  - URL: 競合製品ページ
  - フォーマット: JSON
  - パーサー: @olostep/amazon-product
↓
人間のレビューのステップ
  - 価格変更を確認
  - 承認または拒否
↓
アクション: Slack - メッセージを送信（承認された場合）
  - アラート: "価格が$\{\{price\}\}に変更されました"
```

複数のソースからコンテンツを集約し、強化：

```

↓
アクション: Olostep - URLをスクレイプ
  - URL: \{\{シートからのURL\}\}
  - フォーマット: Markdown
↓
AIアクション: コンテンツを要約
  - 要約を生成
↓
アクション: Airtable - レコードを作成
  - AIの要約を含む記事を保存
```

### リードエンリッチメントパイプライン

ウェブ情報とAIを使用してリードデータを強化：

```
トリガー: HubSpot - 新しい連絡先
↓
アクション: Olostep - URLをスクレイプ
  - URL: \{\{会社のウェブサイト\}\}
  - フォーマット: Markdown
↓
AIアクション: 会社情報を抽出
  - 抽出: 会社の規模、業界、製品
↓
人間のレビューのステップ
  - AI抽出の精度を確認
↓
アクション: HubSpot - 連絡先を更新
  - 検証されたエンリッチデータを追加
```

## 人気アプリとの統合

### Google Sheets

データ収集と追跡に最適：

```
1. Olostepがウェブサイトをスクレイプ
2. AI - データを処理してフォーマット
3. 人間のレビュー（オプション）
4. Google Sheets - 行を作成/更新
```

**ユースケース:**
- 価格追跡スプレッドシート
- リードエンリッチメントデータベース
- コンテンツインベントリ
- 競合分析シート

### Airtable

スクレイプされたデータで強力なデータベースを構築：

```
1. Olostepがスクレイプまたはクロール
2. AI - 構造化データを抽出
3. Airtable - レコードを作成
```

**ユースケース:**
- 製品カタログ
- リサーチデータベース
- コンテンツカレンダー
- リンクデータベース

### Slack

承認ワークフローで即時通知を受け取る：

```
1. Olostepがページを監視
2. フィルタ - 変更を確認
3. 人間の承認ステップ
4. Slack - メッセージを送信（承認された場合）
```

**ユースケース:**
- 価格下落アラート（承認付き）
- コンテンツ更新通知
- エラーモニタリング
- 日次ダイジェスト

### HubSpot / Salesforce

CRMデータを自動的にエンリッチ：

```
1. 新しい連絡先が追加される
2. Olostepが会社のウェブサイトをスクレイプ
3. AIが主要情報を抽出
4. 人間のレビューのステップ
5. CRM - 連絡先を更新（承認された場合）
```

**ユースケース:**
- 品質管理を伴うリードエンリッチメント
- 会社リサーチ
- 競合インテリジェンス
- アカウントマッピング

### Notion

レビュー済みコンテンツでナレッジベースを構築：

```
1. Olostepがドキュメントをクロール
2. AI - コンテンツを解析して構造化
3. 人間のレビューのステップ
4. Notion - ページを作成（承認されたコンテンツ）
```

**ユースケース:**
- ドキュメントミラー
- リサーチリポジトリ
- コンテンツライブラリ
- チームウィキ

## トラブルシューティング

<AccordionGroup>
  <Accordion title="認証失敗">
    **エラー**: "無効なAPIキー"
    
    **解決策**:
    - [ダッシュボード](https://olostep.com/dashboard)からAPIキーを確認
    - APIキーに余分なスペースがないことを確認
    - Relay.appでOlostepアカウントを再接続
    - APIキーがアクティブであることを確認
  </Accordion>

  <Accordion title="スクレイプが空のコンテンツを返す">
    **エラー**: コンテンツフィールドが空
    
    **解決策**:
    - 「スクレイプ前の待機時間」を増やす
    - ウェブサイトがログインを必要とするか確認
    - 異なるフォーマットを試す（HTML対Markdown）
    - URLがアクセス可能であることを確認
    - サイトが自動アクセスをブロックしていないか確認
  </Accordion>

  <Accordion title="レート制限を超えた">
    **エラー**: "レート制限を超えた"
    
    **解決策**:
    - ワークフローの実行を遅延で間隔をあける
    - 個別のスクレイプの代わりにバッチ処理を使用
    - Olostepプランをアップグレード
    - ダッシュボードでレート制限を確認
  </Accordion>

  <Accordion title="URLがスクレイプされない">
    **エラー**: 特定のURLがスクレイプに失敗
    
    **解決策**:
    - URL形式を確認（http://またはhttps://を含む）
    - URLが認証を必要とするか確認
    - まずブラウザでURLをテスト
    - 国のパラメータで試す
    - ブロックされたドメインについてサポートに連絡
  </Accordion>
</AccordionGroup>

## Relay.app vs Zapier

Relay.appはZapierに対するモダンな代替手段で、いくつかの利点があります：

**Relay.appの利点:**
- **人間のループ**: 組み込みの手動レビューのステップ
- **組み込みAI**: ワークフローに統合されたAI機能
- **より良いUX**: より直感的なインターフェースと明確なエラーメッセージ
- **より良い価格設定**: 多くのユースケースでよりコスト効果が高い
- **より深い統合**: 人気のあるアプリとのより包括的な統合

**Zapierを選ぶとき:**
- あまり一般的でないアプリとの統合が必要
- より確立されたプラットフォームを好む
- Zapier専用の特定の機能が必要

詳細な比較については、[Relay.appのZapier比較](https://www.relay.app/blog/zapier-alternatives)を参照してください。

## 価格設定

OlostepはAPIの使用に基づいて料金を請求し、Relay.appとは独立しています：

- **スクレイプ**: スクレイプごとに支払い
- **バッチ**: バッチ内のURLごとに支払い
- **クロール**: クロールされたページごとに支払い
- **マップ**: マップ操作ごとに支払い

現在の価格は[olostep.com/pricing](https://olostep.com/pricing)で確認してください。

**Relay.appプラン**: ワークフローを実行するにはアクティブなRelay.appプランも必要です。
