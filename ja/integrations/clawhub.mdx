あなたのOpenClawエージェントはドキュメントを読むことができますが、ウェブを読むことはできません。[Olostep Web Agentプラグイン](https://clawhub.ai/plugins/olostep-web-agent)がその問題を解決します。エージェントのワークフロー内で直接、任意のウェブサイトから検索、スクレイプ、クロール、構造化データの抽出が可能です。

1回のインストールで、高レベルのタスクを実行するための**13のスキル**（ライブのStackOverflowスレッドからエラーをデバッグ、現在のドキュメントからインテグレーションコードを書く、構造化比較でツールを調査）と、直接プログラム的にアクセスするための**9つのツールを備えたMCPサーバー**を提供します。JSが多用されたSPA、CAPTCHA、Cloudflare、住宅プロキシなども自動的に処理されます。

## インストール

```bash
clawhub install olostep
```

これで完了です。MCPサーバーを手動で接続したい場合は、次の内容をOpenClawの設定に追加してください：

```json
{
  "mcpServers": {
    "olostep": {
      "command": "npx",
      "args": ["-y", "olostep-mcp"],
      "env": {
        "OLOSTEP_API_KEY": "your-api-key-here"
      }
    }
  }
}
```

[olostep.com/auth](https://olostep.com/auth)で無料のAPIキーを取得できます — 月500リクエスト、クレジットカード不要。

## MCPツール

バンドルされたMCPサーバー（`olostep-mcp`）は、エージェントに直接呼び出せる9つのツールを提供します。ワークフロースキルが提供する以上の詳細な制御が必要な場合に使用します。

| ツール | 機能 |
|------|-------------|
| `scrape_website` | 単一のURLをMarkdown、HTML、JSON、またはテキストにスクレイプ |
| `get_webpage_content` | ウェブページをクリーンでLLM対応のMarkdownとして取得 |
| `search_web` | ライブウェブを検索し、AI合成回答を取得 |
| `google_search` | 構造化されたGoogle SERPデータ — オーガニック結果、PAA、ナレッジグラフ |
| `answers` | 質問をし、引用付きの回答を取得し、オプションで構造化されたJSON出力を得る |
| `batch_scrape_urls` | フルレンダリングで最大10,000のURLを並行してスクレイプ |
| `create_crawl` | 開始URLからリンクをたどり、ウェブサイトをクロール |
| `create_map` | ウェブサイト上のすべてのURLを発見し、パターンとクエリでフィルタリング可能 |
| `get_website_urls` | 特定のドメインから関連するURLを見つけて取得 |

## 実際の使用例

### 「なぜ失敗しているのか？」 — ライブウェブからデバッグ

`ECONNRESET when calling Stripe webhook endpoint`をエージェントに貼り付けます。エージェントはGitHubの問題とStackOverflowを検索し、その正確なエラーを探し、最も関連性の高い3つのスレッドをスクレイプし、具体的な修正を返します — 「ネットワーク設定を確認してください」ではなく、同じ問題に直面した他の開発者が実際に解決したタイムアウト設定です。

### 「インテグレーションを書いて」 — 古いトレーニングデータではなく、最新のドキュメントからコードを生成

支払いAPIを統合する必要があります。`docs-to-code`スキルは、今日公開された最新のAPIリファレンスをスクレイプし、実際に存在するパラメータとエンドポイントを使用して動作するコードを書きます。

### 「どれを使うべきか？」 — 構造化されたツール比較

ORMを評価していますか？認証プロバイダーを比較していますか？`research`スキルは複数のソースを検索し、実際の価格ページと機能マトリックスをスクレイプし、引用付きの構造化比較表を返します。製品のウェブサイトに実際にある情報に基づいた推奨を得ることができます。

### 任意のドキュメントサイトからRAGパイプラインを構築

```
map   → ドキュメントサイト上のすべてのURLを発見
batch → すべてのページをクリーンなMarkdownとして並行してスクレイプ
      → ベクトルストアにフィード
```

2つのスキル、1つのパイプライン。数百または数千のページを持つドキュメントサイトで機能します。

### 大規模に構造化データを抽出

```
map   → すべての商品、リスト、または求人URLを発見
batch → 事前構築されたパーサーで各ページをスクレイプ → 型付きJSON
      → データベース、API、またはシードファイルにパイプ
```

### 新しいフレームワークバージョンに移行

Next.js 13から15にアップグレードしています。`migrate-code`スキルは公式の移行ガイドをスクレイプし、すべての破壊的変更を抽出し、コードパターンを前後にしてファイルを書き換えます — モデルの推測ではなく、実際のドキュメントに基づいています。

## 事前構築されたパーサー

任意のスクレイプ呼び出しに`parser`パラメータを渡すと、生のコンテンツではなく型付きJSONが返されます。スキーマ定義は不要 — これらが抽出を処理します。

| パーサー | 返される内容 |
|--------|---------|
| `@olostep/google-search` | オーガニック結果、ナレッジグラフ、People Also Ask |
| `@olostep/amazon-it-product` | 価格、評価、機能、在庫状況 |
| `@olostep/extract-emails` | ページ上で見つかったすべてのメールアドレス |
| `@olostep/extract-calendars` | 構造化されたカレンダーイベント |
| `@olostep/extract-socials` | ソーシャルメディアのプロフィールリンク |
