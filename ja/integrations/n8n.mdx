Olostepのn8nインテグレーションは、強力なウェブ検索、スクレイピング、クローリング機能をn8nワークフローに提供します。コードを書くことなく、任意のウェブサイトからデータを検索、抽出、構造化する自動化されたワークフローを構築できます。

[Olostepをn8nで始める →](https://www.npmjs.com/package/n8n-nodes-olostep)

## 特徴

このインテグレーションは、ウェブデータの自動抽出のための6つの強力な操作を提供します：

## インストール

### 1. ノードをインストール

npmを通じてOlostepノードパッケージをインストール：

```bash
npm install n8n-nodes-olostep
```

その後、n8nを再起動して新しいノードを読み込みます。

または、詳細な手順については[n8nコミュニティノードのインストールガイド](https://docs.n8n.io/integrations/community-nodes/installation/)を参照してください。

### 2. アカウントを接続

ワークフローで初めてOlostepノードを使用する際には、資格情報を設定する必要があります：

1. ワークフローに「Olostep Scrape」ノードを追加
2. ノードをクリックして設定を開く
3. 「新しい資格情報を作成」または既存の資格情報を選択
4. Olostep APIキーを入力
5. 「保存」をクリックして資格情報を保存

APIキーは[Olostepダッシュボード](https://olostep.com/dashboard)から取得できます。

## 利用可能なアクション

### ウェブサイトをスクレイプ

単一のURLからコンテンツを抽出。複数の形式とJavaScriptレンダリングをサポート。

**ユースケース:**
- 特定のページの変更を監視
- ECサイトから商品情報を抽出
- ニュース記事やブログ投稿からデータを収集
- コンテンツ集約のためのコンテンツを取得

**設定:**

<ParamField path="スクレイプするURL" type="string" required>
  スクレイプするウェブサイトのURL（http://またはhttps://を含める必要があります）
</ParamField>

<ParamField path="出力形式" type="dropdown" default="Markdown">
  形式を選択：Markdown、HTML、JSON、またはプレーンテキスト
</ParamField>

<ParamField path="国コード" type="string">
  ロケーション特定のコンテンツのための国コード（例："US"、"GB"、"CA"）
</ParamField>

<ParamField path="スクレイプ前の待機時間" type="integer">
  JavaScriptレンダリングのための待機時間（0-10000ミリ秒）
</ParamField>

<ParamField path="パーサー" type="string">
  特殊な抽出のためのオプションのパーサーID（例："@olostep/amazon-product"）
</ParamField>

**出力フィールド:**
- スクレイプID
- スクレイプされたURL
- Markdownコンテンツ
- HTMLコンテンツ
- JSONコンテンツ
- テキストコンテンツ
- ステータス
- タイムスタンプ
- スクリーンショットURL（利用可能な場合）
- ページメタデータ

**例ワークフロー:**

<AccordionGroup>
  <Accordion title="競合価格の監視">
    **トリガー:** スケジュール（毎日午前9時）
    
    **アクション:** Olostep - ウェブサイトをスクレイプ
    - URL: 競合製品ページ
    - 形式: JSON
    - パーサー: @olostep/amazon-product
    
    **アクション:** Google Sheets - 行を追加
    - 価格データを追跡スプレッドシートに追加
    
    **アクション:** メール - メール送信（価格が下がった場合）
    - 価格変動をチームに通知
  </Accordion>

  <Accordion title="ブログ投稿の抽出と保存">
    **トリガー:** RSSフィード - 新しいアイテム
    
    **アクション:** Olostep - ウェブサイトをスクレイプ
    - URL: \{\{$json.link\}\}
    - 形式: Markdown
    
    **アクション:** Notion - ページを作成
    - 記事コンテンツをNotionデータベースに保存
  </Accordion>

  <Accordion title="リードエンリッチメント">
    **トリガー:** Google Sheets - 新しい行
    
    **アクション:** Olostep - ウェブサイトをスクレイプ
    - URL: シートからの会社ウェブサイト
    - 形式: Markdown
    
    **アクション:** OpenAI - テキストを完了
    - AIを使用して会社情報を抽出
    
    **アクション:** Google Sheets - 行を更新
    - エンリッチされたデータをシートに追加
  </Accordion>
</AccordionGroup>

### 検索

指定されたクエリでウェブを検索し、構造化された結果を取得（非AI、パーサーベースの検索結果）。

**ユースケース:**
- 自動化されたリサーチワークフロー
- リードの発見とエンリッチメント
- 競合分析
- コンテンツリサーチ

**設定:**

<ParamField path="クエリ" type="string" required>
  検索クエリ
</ParamField>

**出力:** タイトル、URL、スニペット、メタデータを含むJSON形式の構造化された検索結果を返します。

**例ワークフロー:**

<AccordionGroup>
  <Accordion title="自動化されたリサーチ">
    **トリガー:** スケジュール（毎日午前8時）
    
    **アクション:** Olostep - 検索
    - クエリ: "最新のAI開発"
    
    **アクション:** コード - 結果を処理
    - 主要情報を抽出しフォーマット
    
    **アクション:** Notion - ページを作成
    - リサーチ結果を保存
  </Accordion>

  <Accordion title="リード発見">
    **トリガー:** 手動（ボタン）
    
    **アクション:** Olostep - 検索
    - クエリ: "\{\{$json.searchTerm\}\}"
    
    **アクション:** Airtable - レコードを作成
    - 連絡先情報と共にリードを保存
  </Accordion>
</AccordionGroup>

### URLをバッチスクレイプ

最大10,000のURLを同時にスクレイプ。大規模なデータ抽出に最適。

**ユースケース:**
- 製品カタログ全体をスクレイプ
- 複数の検索結果からデータを抽出
- スプレッドシートからURLリストを処理
- バルクコンテンツ抽出

**設定:**

<ParamField path="スクレイプするURL" type="text" required>
  urlとcustom_idフィールドを持つオブジェクトのJSON配列。 
  
  例: `[{"url":"https://example.com","custom_id":"site1"}]`
</ParamField>

<ParamField path="出力形式" type="dropdown" default="Markdown">
  すべてのURLの形式を選択：Markdown、HTML、JSON、またはプレーンテキスト
</ParamField>

<ParamField path="国コード" type="string">
  ロケーション特定のスクレイピングのための国コード
</ParamField>

<ParamField path="スクレイプ前の待機時間" type="integer">
  JavaScriptレンダリングのための待機時間
</ParamField>

<ParamField path="パーサー" type="string">
  特殊な抽出のためのオプションのパーサーID
</ParamField>

**出力フィールド:**
- バッチID（後で結果を取得するために使用）
- ステータス
- 合計URL
- 作成日時
- リクエストされた形式
- 国コード
- 使用されたパーサー

**例ワークフロー:**

<AccordionGroup>
  <Accordion title="製品カタログをスクレイプ">
    **トリガー:** Webhook - POSTリクエストを受信
    
    **アクション:** コード - URLをフォーマット
    - CSV/リストをJSON配列形式に変換
    
    **アクション:** Olostep - URLをバッチスクレイプ
    - URL: \{\{$json.urlArray\}\}
    - 形式: JSON
    - パーサー: @olostep/amazon-product
    
    **アクション:** Webhook - POST
    - バッチIDをシステムに送信して取得
  </Accordion>

  <Accordion title="毎日のコンテンツ監視">
    **トリガー:** スケジュール - 毎日午前6時
    
    **アクション:** Google Sheets - 行を読み込む
    - 監視するURLを取得
    
    **アクション:** コード - URLをフォーマット
    - バッチ配列形式に変換
    
    **アクション:** Olostep - URLをバッチスクレイプ
    - すべてのURLを一度に処理
    
    **アクション:** Slack - メッセージを送信
    - スクレイピングが完了したことをチームに通知
  </Accordion>
</AccordionGroup>

### クローリングを作成

URLのサブページのコンテンツを取得。リンクをたどってウェブサイト全体を自律的に発見しスクレイプ。ドキュメントサイト、ブログ、コンテンツリポジトリに最適。

**ユースケース:**
- ドキュメントサイト全体をクロールしてアーカイブ
- ウェブサイトからすべてのブログ投稿を抽出
- ウェブコンテンツからナレッジベースを構築
- ウェブサイト構造の変更を監視

**設定:**

<ParamField path="開始URL" type="string" required>
  クローリングの開始URL（http://またはhttps://を含める必要があります）
</ParamField>

<ParamField path="最大ページ数" type="integer" default="10">
  クロールする最大ページ数
</ParamField>

<ParamField path="リンクをたどる" type="boolean" default="true">
  ページで見つかったリンクをたどるかどうか
</ParamField>

<ParamField path="出力形式" type="dropdown" default="Markdown">
  スクレイプされたコンテンツの形式
</ParamField>

<ParamField path="国コード" type="string">
  ロケーション特定のクロールのためのオプションの国コード
</ParamField>

<ParamField path="パーサー" type="string">
  特殊なコンテンツ抽出のためのオプションのパーサーID
</ParamField>

**出力フィールド:**
- クローリングID（後で結果を取得するために使用）
- オブジェクトタイプ
- ステータス
- 開始URL
- 最大ページ数
- リンクをたどる
- 作成タイムスタンプ
- 形式

**例ワークフロー:**

<AccordionGroup>
  <Accordion title="ドキュメントサイトをアーカイブ">
    **トリガー:** スケジュール - 毎月1日の午前0時
    
    **アクション:** Olostep - クローリングを作成
    - 開始URL: https://docs.example.com
    - 最大ページ数: 500
    - リンクをたどる: true
    - 形式: Markdown
    
    **アクション:** Webhook - POST
    - クローリングIDをアーカイブシステムに送信
    
    **アクション:** Slack - メッセージを送信
    - クローリングが進行中であることをチームに通知
  </Accordion>

  <Accordion title="競合コンテンツ分析">
    **トリガー:** スケジュール - 毎週月曜日の午前9時
    
    **アクション:** Olostep - クローリングを作成
    - 開始URL: 競合ブログURL
    - 最大ページ数: 100
    - 形式: Markdown
    
    **アクション:** 待機 - 10分間待機
    - クローリングが完了するのを待つ
    
    **アクション:** Airtable - レコードを作成
    - 分析のためにクローリングデータを保存
  </Accordion>
</AccordionGroup>

### マップを作成

ウェブサイト上のすべてのURLを取得。コンテンツ発見とサイト構造分析のためにウェブサイトからすべてのURLを抽出。

**ユースケース:**
- サイトマップとサイト構造図を作成
- バッチスクレイピング前にすべてのページを発見
- 壊れたページや欠落したページを見つける
- SEO監査と分析

**設定:**

<ParamField path="ウェブサイトURL" type="string" required>
  リンクを抽出するウェブサイトのURL（http://またはhttps://を含める必要があります）
</ParamField>

<ParamField path="検索クエリ" type="string">
  URLをフィルタリングするためのオプションの検索クエリ（例："blog"）
</ParamField>

<ParamField path="上位N URL" type="integer">
  返されるURLの数を制限
</ParamField>

<ParamField path="含めるURLパターン" type="string">
  特定のパスを含めるためのグロブパターン（例："/blog/**"）
</ParamField>

<ParamField path="除外するURLパターン" type="string">
  特定のパスを除外するためのグロブパターン（例："/admin/**"）
</ParamField>

**出力フィールド:**
- マップID
- オブジェクトタイプ
- ウェブサイトURL
- 見つかったURLの合計
- URL（JSON配列）
- 検索クエリ
- 上位N制限

**例ワークフロー:**

<AccordionGroup>
  <Accordion title="発見とスクレイプ">
    **トリガー:** 手動（ボタン）
    
    **アクション:** Olostep - マップを作成
    - URL: https://example.com
    - 含めるパターン: /products/**
    - 上位N: 500
    
    **アクション:** コード - URLを抽出
    - マップ結果からURLを解析
    
    **アクション:** Olostep - URLをバッチスクレイプ
    - URL: \{\{$json.urls\}\}
    - 形式: JSON
    
    **アクション:** Google Sheets - 行を追加
    - すべての製品データをスプレッドシートに追加
  </Accordion>

  <Accordion title="SEOサイト監査">
    **トリガー:** スケジュール - 毎月
    
    **アクション:** Olostep - マップを作成
    - URL: あなたのウェブサイト
    - 上位N: 1000
    
    **アクション:** Airtable - レコードを作成
    - すべてのURLを追跡用に保存
    
    **アクション:** Slack - メッセージを送信
    - 見つかったページの合計を報告
  </Accordion>
</AccordionGroup>

## 人気のワークフロー例

### Eコマース価格監視

競合価格を監視し、即時アラートを受け取る：

```
トリガー: スケジュール（毎時）
↓
アクション: Olostep - ウェブサイトをスクレイプ
  - URL: 競合製品ページ
  - 形式: JSON
  - パーサー: @olostep/amazon-product
↓
アクション: IF - 価格が変わったか確認
↓
アクション: Slack - メッセージを送信
  - アラート: "価格が$\{\{price\}\}に変更されました"
```

### コンテンツ集約

複数のソースからコンテンツを集約：

```
トリガー: Google Sheets - 新しい行
↓
アクション: Olostep - ウェブサイトをスクレイプ
  - URL: \{\{$json.url\}\}
  - 形式: Markdown
↓
アクション: OpenAI - 要約
  - コンテンツを要約
↓
アクション: Airtable - レコードを作成
  - 要約付きの記事を保存
```

### リードエンリッチメントパイプライン

ウェブ情報でリードデータをエンリッチ：

```
トリガー: HubSpot - 新しいコンタクト
↓
アクション: Olostep - ウェブサイトをスクレイプ
  - URL: \{\{$json.companyWebsite\}\}
  - 形式: Markdown
↓
アクション: OpenAI - データを抽出
  - 抽出: 会社規模、業界、製品
↓
アクション: HubSpot - コンタクトを更新
  - エンリッチされたデータをコンタクトに追加
```

### リサーチ自動化

複数のソースからリサーチを自動化：

```
トリガー: Airtable - 新しいレコード
↓
アクション: Olostep - マップを作成
  - URL: リサーチ対象ウェブサイト
  - 含める: /research/**
↓
アクション: コード - URLを解析
↓
アクション: Olostep - URLをバッチスクレイプ
  - URL: \{\{$json.discoveredUrls\}\}
  - 形式: Markdown
↓
アクション: Notion - ページを作成
  - リサーチデータベースを作成
```

### ソーシャルメディア監視

言及とコンテンツを追跡：

```
トリガー: スケジュール（6時間ごと）
↓
アクション: Olostep - ウェブサイトをスクレイプ
  - URL: ニュースサイトの検索ページ
  - 形式: HTML
↓
アクション: コード - 言及を抽出
  - ブランドの言及を見つける
↓
アクション: Google Sheets - 行を追加
  - タイムスタンプ付きで言及を記録
```

## マルチステップワークフロー

### 完全な製品スクレイピングパイプライン

包括的な製品データパイプラインを構築：

<Steps>
  <Step title="製品URLを発見">
    **マップを作成**を使用して、ターゲットウェブサイト上のすべての製品ページを見つける
    - 含めるパターン: `/products/**`
    - 除外するパターン: `/cart/**`, `/checkout/**`
  </Step>
  
  <Step title="製品をバッチ処理">
    **URLをバッチスクレイプ**を使用して、すべての製品データを抽出
    - 形式: JSON
    - パーサー: 製品特化のパーサーがあれば使用
  </Step>
  
  <Step title="データベースに保存">
    バッチIDをシステムに送信するか、待機して結果を取得
    - Airtable、Google Sheets、またはデータベースを使用
  </Step>
  
  <Step title="変更を監視">
    毎日のスクレイプをスケジュールして、価格/在庫の変更を追跡
    - 既存データと比較
    - 重要な変更をアラート
  </Step>
</Steps>

### SEOコンテンツ戦略

競合を分析し、コンテンツを計画：

<Steps>
  <Step title="競合サイトをマップ">
    **マップを作成**を使用して競合ウェブサイトをマップ
    - すべてのブログ投稿とコンテンツページを抽出
  </Step>
  
  <Step title="コンテンツをスクレイプ">
    **URLをバッチスクレイプ**を使用して完全なコンテンツを取得
    - 形式: Markdownで簡単に分析
  </Step>
  
  <Step title="AI分析">
    OpenAIを使用してトピックとキーワードを分析
    - コンテンツギャップを特定
    - トレンドトピックを見つける
  </Step>
  
  <Step title="コンテンツカレンダーを作成">
    NotionまたはAirtableにインサイトを追加
    - コンテンツ戦略を計画
  </Step>
</Steps>

```
  - URL: https://www.amazon.com/dp/PRODUCT_ID
  - 形式: JSON
  - パーサー: @olostep/amazon-product
```

```
アクション: Olostep - ウェブサイトをスクレイプ
  - URL: https://example.com/contact
  - 形式: JSON
  - パーサー: @olostep/extract-emails
```

パーサーは自動的にタスクに特化した構造化データを抽出します。

## ベストプラクティス

<AccordionGroup>
  <Accordion title="複数のURLにはバッチ処理を使用">
    3-5以上のURLをスクレイプする場合は、複数の**ウェブサイトをスクレイプ**アクションの代わりに**URLをバッチスクレイプ**を使用してください。バッチ処理は：
    - はるかに高速（並列処理）
    - よりコスト効果が高い
    - 管理が容易
    - レート制限に優しい
  </Accordion>

  <Accordion title="適切な待機時間を設定">
    JavaScriptが多用されているサイトには、「スクレイプ前の待機時間」パラメータを使用：
    - シンプルなサイト: 0-1000ms
    - 動的サイト: 2000-3000ms
    - 重いJavaScript: 5000-8000ms
    
    最適な待機時間を見つけるために異なる値でテストしてください。
  </Accordion>

  <Accordion title="特殊なパーサーを使用">
    事前構築されたパーサー（例：Amazon、Google、およびOlostepストアからのタスク特化のパーサー）を使用：
    - 構造化データを自動的に取得
    - より信頼性の高い抽出
    - カスタム解析の必要なし
    - Olostepによって維持
  </Accordion>

  <Accordion title="スクレイプ前にフィルタリング">
    不要なスクレイプを避けるためにn8nのIFノードを使用：
    - URLが変更されたか確認
    - データが最近スクレイプされていないか確認
    - スクレイプ前にビジネスロジックを適用
    
    これによりAPIクレジットと実行時間を節約できます。
  </Accordion>

  <Accordion title="非同期操作を処理">
    バッチ、クローリング、マップ操作は非同期です：
    - 返されたID（batch_id、crawl_id、map_id）を保存
    - 即時取得する場合はWaitノードを使用
    - 完了のためのWebhookコールバックを検討
    - 取得のための別のワークフローを設定
  </Accordion>

  <Accordion title="結果を適切に保存">
    ニーズに応じて適切なストレージを選択：
    - **Google Sheets**: シンプルな追跡、チームコラボレーション
    - **Airtable**: リレーショナルデータ、リッチフォーマット
    - **データベース**: 大規模、複雑なクエリ
    - **Notion**: ナレッジベース、ドキュメント
  </Accordion>

  <Accordion title="監視とアラート">
    スクレイピングワークフローの監視を設定：
    - n8nのエラーワークフローを使用
    - 失敗時にSlack/Emailにアラートを送信
    - OlostepダッシュボードでAPI使用量を追跡
    - 重要なメトリクスをログ
  </Accordion>
</AccordionGroup>

## トラブルシューティング

<AccordionGroup>
  <Accordion title="認証失敗">
    **エラー**: "無効なAPIキー"
    
    **解決策**:
    - [ダッシュボード](https://olostep.com/dashboard)からAPIキーを確認
    - APIキーに余分なスペースがないことを確認
    - n8nで資格情報を再作成
    - APIキーがアクティブであることを確認
  </Accordion>

  <Accordion title="スクレイプが空のコンテンツを返す">
    **エラー**: コンテンツフィールドが空
    
    **解決策**:
    - 「スクレイプ前の待機時間」を増やす
    - ウェブサイトがログインを必要としているか確認
    - 異なる形式を試す（HTML対Markdown）
    - URLがアクセス可能であることを確認
    - サイトが自動アクセスをブロックしていないか確認
  </Accordion>

  <Accordion title="バッチ配列形式エラー">
    **エラー**: "バッチ配列のJSON形式が無効"
    
    **解決策**:
    - 形式を使用: `[{"url":"https://example.com","custom_id":"id1"}]`
    - 正しいJSON構文を確認
    - コードノードを使用してURLを正しくフォーマット
    - オンラインバリデーターでJSONをテスト
  </Accordion>

  <Accordion title="レート制限超過">
    **エラー**: "レート制限超過"
    
    **解決策**:
    - Waitノードでワークフローの実行を間隔を空ける
    - 個別のスクレイプの代わりにバッチ処理を使用
    - Olostepプランをアップグレード
    - ダッシュボードでレート制限を確認
  </Accordion>

  <Accordion title="URLがスクレイプされない">
    **エラー**: 特定のURLがスクレイプに失敗
    
    **解決策**:
    - URL形式を確認（http://またはhttps://を含める）
    - URLが認証を必要としているか確認
    - まずブラウザでURLをテスト
    - 国パラメータを試す
    - ブロックされたドメインについてサポートに連絡
  </Accordion>
</AccordionGroup>

## n8nの利点

### セルフホスト
n8nはセルフホスト型で、ワークフローとデータを完全にコントロールできます。ベンダーロックインなし、データがインフラストラクチャを離れることはありません。

### タスク制限なし
クラウドベースの自動化プラットフォームとは異なり、n8nはタスク制限を課しません。追加コストなしで必要なだけワークフローを実行できます。

### オープンソース
n8nはオープンソースであり、特定のニーズに合わせてカスタマイズおよび拡張できます。

### コスト効果
セルフホスト型のn8nは無料で、オプションでクラウドホスティングが利用可能です。Olostep APIの使用料のみを支払います。

## 料金

Olostepはn8nとは独立してAPI使用量に基づいて課金します：

- **スクレイプ**: スクレイプごとに支払い
- **バッチ**: バッチ内のURLごとに支払い
- **クローリング**: クロールされたページごとに支払い
- **マップ**: マップ操作ごとに支払い

最新の料金は[olostep.com/pricing](https://olostep.com/pricing)で確認できます。

**n8n**: セルフホスト型のn8nは無料です。クラウドホスティングはオプションの有料プランで利用可能です。
