Über den Olostep `/v1/maps` Endpunkt können Sie alle URLs auf einer Website abrufen. Dies ist nützlich für die Inhaltserkennung, die Analyse der Seitenstruktur (z.B. SEO) oder um zu entscheiden, welche URLs Sie als nächstes scrapen möchten.

- Erhalten Sie alle URLs auf einer Website (einschließlich Sitemaps und entdeckter Links)
- Verwenden Sie spezielle Muster, um Pfade einzuschließen/auszuschließen (z.B. `/blog/**`)
- Paginieren Sie große Antworten mit `cursor` (bis zu 10MB pro Antwort)
- Begrenzen Sie das Volumen mit `top_n`

Für API-Details siehe die [Map Endpoint API Referenz](/api-reference/maps/create).

## Verwendung

Senden Sie eine POST-Anfrage mit der Website-`url`. Optional können Sie `include_urls`, `exclude_urls` (Glob-Muster) und `top_n` übergeben.

<CodeGroup>

```python Python
import requests
import json

endpoint = "https://api.olostep.com/v1/maps"

payload = {
    "url": "https://docs.olostep.com"
}
headers = {
    "Authorization": "Bearer <YOUR_API_KEY>",
    "Content-Type": "application/json"
}

response = requests.post(endpoint, json=payload, headers=headers)
print(json.dumps(response.json(), indent=2))
```

```js Node
const res = await fetch('https://api.olostep.com/v1/maps', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer <YOUR_API_KEY>',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ url: 'https://docs.olostep.com' })
})
console.log(await res.json())
```

```bash cURL
curl -s -X POST "https://api.olostep.com/v1/maps" \
  -H "Authorization: Bearer $OLOSTEP_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://docs.olostep.com"
  }'
```

</CodeGroup>

Die Antwortzeit liegt typischerweise im Sekundenbereich, kann aber bei komplexeren Websites bis zu 120 Sekunden dauern. Es können alle URLs von einer Website extrahiert werden, sogar Backlinks und solche, die nicht in den Sitemaps vorhanden sind. Sie können auch entscheiden, welche URL-Pfade Sie in die Antwort ein- oder ausschließen möchten.

Standardmäßig gibt der Endpunkt etwa 100.000 URLs in einem einzigen Aufruf zurück (maximal 10MB). Wenn die Antwort mehr Daten enthält, gibt die API einen `cursor`-Parameter zurück, der für die Paginierung und das Abrufen der nachfolgenden URLs verwendet werden kann. Für weitere Details siehe die [API-Referenz](/api-reference/maps/create#body-cursor).

Dieser Endpunkt ist besonders nützlich, wenn Sie:
- Alle Inhaltsseiten auf einer Website entdecken möchten
- Die Seitenstruktur und Hierarchie analysieren möchten
- URLs für die Stapelverarbeitung vorbereiten möchten
- Entscheiden möchten, welche spezifischen URLs gescrapt werden sollen

Für eine genauere Kontrolle über die zurückgegebenen URLs können Sie die Parameter `include_urls` und `exclude_urls` verwenden.

### Beispiel

Angenommen, Sie möchten von www.brex.com alle URLs extrahieren, die die Pfade nach `/product/` haben, z.B. `https://www.brex.com/product/api/no-code`, aber auch `www.brex.com/product` einbeziehen.
Sie können den folgenden Code verwenden:

<CodeGroup>

    ```python maps.py
    import requests
    
    endpoint = "https://api.olostep.com/v1/maps"
    
    payload = {
        "url": "https://www.brex.com/",
        "include_urls": ["/product", "/product/**"],
        "top_n": 100000
    }
    
    headers = {
        "Authorization": "Bearer <YOUR_API_KEY>",
        "Content-Type": "application/json"
    }
    
    response = requests.post(endpoint, json=payload, headers=headers)
    
    print(response.text)
    ```

    ```js Node
    const endpoint = 'https://api.olostep.com/v1/maps'

    const res = await fetch(endpoint, {
      method: 'POST',
      headers: {
        'Authorization': 'Bearer <YOUR_API_KEY>',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        url: 'https://www.brex.com/',
        include_urls: ['/product', '/product/**'],
        top_n: 100000
      })
    })
    console.log(await res.text())
    ```

    ```bash cURL
    curl -s -X POST "https://api.olostep.com/v1/maps" \
      -H "Authorization: Bearer $OLOSTEP_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "url": "https://www.brex.com/",
        "include_urls": ["/product", "/product/**"],
        "top_n": 100000
      }'
    ```

</CodeGroup>

## Fazit

Der Maps-Endpunkt ist ein leistungsstarkes Werkzeug zur Inhaltserkennung und Seitenanalyse. Er bietet eine umfassende Liste von URLs auf einer Website, sodass Sie Inhalte von bestimmten Seiten extrahieren oder die Seitenstruktur analysieren können. Dieser Endpunkt ist besonders nützlich für SEO-Profis, Content-Marketer und KI-Agenten, die Website-Inhalte oder -Strukturen analysieren müssen.

## Preisgestaltung

Die Karte kostet 1 Kredit. Für jede zusätzlichen 1000 URLs, die in der Antwort zurückgegeben werden, wird ein zusätzlicher Kredit berechnet.
