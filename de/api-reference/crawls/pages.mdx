## Übersicht

Diese API-Endpunkt ermöglicht es Ihnen, die Liste der Seiten für einen bestimmten Crawl abzurufen. Dies ist nützlich, um die gesammelten Daten während eines Crawls zu analysieren oder zu verarbeiten.

## Anfrage

**GET** `/v1/crawls/{crawl_id}/pages`

### Pfadparameter

- `crawl_id` *(erforderlich)*: Die eindeutige Kennung des Crawls, für den die Seiten abgerufen werden sollen.

### Beispielanfrage

```http
GET /v1/crawls/12345/pages HTTP/1.1
Host: api.olostep.com
Authorization: Bearer <Ihr-API-Token>
```

## Antwort

Bei Erfolg gibt der Endpunkt eine Liste der Seiten zurück, die während des angegebenen Crawls erfasst wurden.

### Erfolgsantwort

- **Statuscode**: `200 OK`
- **Inhaltstyp**: `application/json`

```json
{
  "pages": [
    {
      "url": "https://example.com/page1",
      "status": 200,
      "title": "Beispielseite 1"
    },
    {
      "url": "https://example.com/page2",
      "status": 404,
      "title": "Seite nicht gefunden"
    }
  ]
}
```

### Fehlerantworten

- **Statuscode**: `404 Not Found` - Wenn der angegebene `crawl_id` nicht existiert.
- **Statuscode**: `401 Unauthorized` - Wenn die Authentifizierung fehlschlägt.

## Hinweise

- Stellen Sie sicher, dass Sie ein gültiges API-Token verwenden, um auf diesen Endpunkt zuzugreifen.
- Die Antwort enthält nur die Seiten, die während des spezifischen Crawls erfasst wurden.
