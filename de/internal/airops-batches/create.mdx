## Übersicht

Erstellt einen neuen AirOps Batch, der:
1. Alle angegebenen URLs abruft
2. Strukturierte Metadaten automatisch extrahiert
3. Elemente mit fehlenden Feldern identifiziert
4. Unvollständige Elemente über LLM anreichert
5. Vereinheitlichte Ergebnisse für alle URLs zurückgibt

## Anforderungsparameter

<ParamField body="items" type="array" required>
  Array von zu verarbeitenden URLs. Jedes Element erfordert:
  - `url` - Gültige URL zum Abrufen
  - `custom_id` - Ihre eindeutige Kennung (1-255 Zeichen)
</ParamField>

<ParamField body="brands" type="array" default="[]">
  Markennamen, nach denen im Seiteninhalt gesucht werden soll. Das LLM erkennt Erwähnungen dieser spezifischen Marken und gibt Übereinstimmungen im Feld `brand_mentions` zurück.
  
  ```json
  {
    "brands": ["Acme Corp", "TechBrand", "SaaS Pro"]
  }
  ```
</ParamField>

<ParamField body="country" type="string" default="RANDOM">
  Land für geo-targeted Scraping. Verwenden Sie ISO 3166-1 Alpha-2-Codes (`US`, `GB`, `DE`) oder `RANDOM`.
</ParamField>

<ParamField body="webhook" type="string">
  URL zum Empfangen von Webhook-Benachrichtigungen. Alle Lebenszyklusereignisse für diesen Batch werden hierhin gesendet.
</ParamField>

<ParamField body="metadata" type="object">
  Benutzerdefinierte Schlüssel-Wert-Paare, die dem Batch zugeordnet sind. Stripe-Stil: max. 50 Schlüssel, Schlüssel ≤40 Zeichen, Wert ≤500 Zeichen.
</ParamField>
