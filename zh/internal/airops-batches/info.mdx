title: "获取批处理状态"
description: "检索 AirOps 批处理的当前状态和进度"
openapi: "GET /v1/batches-airops/{batch_id}"
---

## 概述

返回 AirOps 批处理的当前阶段和进度指标。如果你没有使用 webhooks，可以使用此方法进行轮询。

## 响应字段

<ResponseField name="id" type="string">
  批处理标识符。
</ResponseField>

<ResponseField name="object" type="string">
  始终为 `"batch"`。
</ResponseField>

<ResponseField name="created_at" type="integer">
  批处理创建时的 Unix 时间戳（毫秒）。
</ResponseField>

<ResponseField name="total_urls" type="integer">
  提交的 URL 总数。
</ResponseField>

<ResponseField name="status" type="string">
  整体工作流状态：`pending`、`in_progress`、`completed` 或 `failed`。
</ResponseField>

<ResponseField name="batch_mode" type="boolean">
  是否启用了异步 LLM 批处理模式（`true`）或同步模式（`false`）。
</ResponseField>

<ResponseField name="scraping" type="object">
  完整的抓取阶段状态对象（参见 [Webhooks](/internal/airops-batches/webhooks#scraping-status)）。
</ResponseField>

<ResponseField name="analyzing" type="object">
  完整的分析阶段状态对象（参见 [Webhooks](/internal/airops-batches/webhooks#analyzing-status)）。
</ResponseField>

<ResponseField name="enriching" type="object">
  完整的丰富阶段状态对象（参见 [Webhooks](/internal/airops-batches/webhooks#enriching-status)）。
</ResponseField>
