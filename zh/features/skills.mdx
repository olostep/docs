通过命令行将 Olostep 技能文件夹直接安装到你的 AI 编码代理中。

- 将 Olostep 技能安装到 Cursor、Claude、Codex、Windsurf 等代理中
- 技能会存放在规范存储 (`~/.agents/skills`) 中，并通过符号链接或复制方式放入每个代理的技能目录
- 所有安装会记录在 `~/.agents/.skill-lock.json` 锁文件中

此功能可通过 [Olostep CLI](/sdks/cli) 使用。

## `olostep add skills`

将技能从源目录同步到 CLI，安装到规范存储中，然后部署到每个代理的技能目录。

### 安装 CLI

```bash
# 全局安装
npm install -g olostep-cli

# 或无需安装直接运行
npx -y olostep-cli@latest add skills
```

### 用法

```bash
# 将所有技能安装到所有检测到的代理中（默认）
olostep add skills

# 先认证，再安装
olostep add skills --login

# 指定特定代理
olostep add skills --agent cursor --agent claude

# 始终复制而不是使用符号链接
olostep add skills --link-mode copy

# 机器可读的 JSON 输出
olostep add skills --json
```

### 选项

| 选项 | 默认值 | 说明 |
| ------ | ------- | ----------- |
| `--login` | — | 安装前先执行浏览器登录 |
| `--source <path>` | CLI 内置技能目录 | 技能源目录 |
| `--cli-local-dir <path>` | CLI/skills | 用于 CLI 本地使用时同步源技能的目录 |
| `--agent <name>` | — | 指定特定代理（可重复） |
| `--all-agents` / `--no-all-agents` | `--all-agents` | 若未设置 `--agent`，则针对所有检测到的已安装代理 |
| `--global` / `--no-global` | `--global` | 安装到已知代理的全局技能目录 |
| `--canonical-dir <path>` | `~/.agents/skills` | Olostep 技能的规范存储位置 |
| `--agent-skills-dir <path>` | — | 自定义目标技能目录（需要 `--no-global`） |
| `--skill <name>` | — | 仅包含该技能（可重复） |
| `--exclude <name>` | — | 排除该技能（可重复） |
| `--overwrite` / `--no-overwrite` | `--overwrite` | 替换现有目标 |
| `--link-mode <auto\|symlink\|copy>` | `auto` | `auto` 会先尝试 symlink，失败后回退到 copy |
| `--json` | — | 输出机器可读的 JSON |

<Note>
**校验规则：**
- `--link-mode` 必须是 `auto`、`symlink` 或 `copy`
- `--agent-skills-dir` 需要 `--no-global`
- `--no-global` 需要 `--agent-skills-dir`
- 未知的代理名称会报错
- 在 `--skill` / `--exclude` 之后若技能选择为空会报错
</Note>

### JSON 输出

```json
{
  "sync": {
    "plugin_source_dir": "/path/to/CLI/skills",
    "cli_local_dir": "/path/to/CLI/skills"
  },
  "selected_skills": ["research-agent"],
  "canonical_dir": "~/.agents/skills",
  "lockfile_path": "~/.agents/.skill-lock.json",
  "installed": [
    {
      "skill": "research-agent",
      "canonical_path": "~/.agents/skills/olostep-research-agent",
      "targets": [
        { "agent": "cursor", "mode": "symlink", "path": "~/.cursor/skills/olostep-research-agent" },
        { "agent": "claude", "mode": "symlink", "path": "~/.claude/skills/olostep-research-agent" }
      ]
    }
  ]
}
```

## 技能发现与格式

系统通过扫描子目录中的 `SKILL.md` 文件来发现技能。该文件必须包含带有 `name` 和 `description` 字段的 frontmatter 块。清洗后重复的名称会被拒绝。

```markdown
---
name: research-agent
description: Autonomous research agent for deep web investigation
---

# Research Agent

...skill instructions here...
```

## 支持的代理

| 代理 | 键 |
| ----- | --- |
| Cursor | `cursor` |
| Claude | `claude` |
| Codex | `codex` |
| Windsurf | `windsurf` |
| Continue | `continue` |
| Augment | `augment` |
| Roo | `roo` |
| Gemini | `gemini` |
| Copilot | `copilot` |
| Factory | `factory` |

每个键都映射到一个已知的主目录技能路径。使用 `--all-agents`（默认）可针对所有检测到的已安装代理，或传入 `--agent <key>` 来指定特定代理。

## 命名规范

安装后的文件夹名称始终使用 `olostep-` 前缀：例如，`research-agent` 会变为 `olostep-research-agent`。名称会被规范化为小写，并将无效字符替换为 `-`。该前缀可清晰标识由 Olostep 管理的技能，并避免与其他工具冲突。

## `olostep remove skills`

从规范存储和各代理的技能目录中删除由 Olostep 安装的技能文件夹，并清理锁文件。

### 用法

```bash
# 从所有代理中移除所有 Olostep 技能
olostep remove skills

# 移除特定技能
olostep remove skills --skill research-agent

# 仅从特定代理中移除
olostep remove skills --agent cursor

# 机器可读的 JSON 输出
olostep remove skills --json
```

### 选项

| 选项 | 默认值 | 说明 |
| ------ | ------- | ----------- |
| `--agent <name>` | — | 仅从指定代理中移除（可重复） |
| `--all-agents` / `--no-all-agents` | `--all-agents` | 若未提供 `--agent` 或自定义目录，则使用所有检测到的已安装代理 |
| `--canonical-dir <path>` | `~/.agents/skills` | 要移除的规范技能目录 |
| `--agent-skills-dir <path>` | — | 用于移除的自定义目标技能目录 |
| `--skill <name>` | — | 仅移除匹配的技能（可重复） |
| `--json` | — | 输出机器可读的 JSON |

<Note>
**安全性：** 仅会处理带有 `olostep-` 前缀的文件夹。非 Olostep 的代理文件夹不会被修改。锁文件清理也只会移除带 `olostep-` 前缀的键。
</Note>
