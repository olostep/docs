经过验证的 [Olostep 应用在 Zapier 上](https://zapier.com/apps/olostep/integrations)为你提供了五个操作，可以在 Zapier 的可视化构建器中使用：抓取 URL、获取 AI 答案、批量抓取成千上万个 URL、爬取网站或映射其所有链接，然后将结果连接到任何 8,000 多个应用。

[在 Zapier 上查看 →](https://zapier.com/apps/olostep/integrations)

## 在你开始之前

- **拥有 Olostep 账户和 API 密钥：** [免费获取一个](https://olostep.com/dashboard)，无需信用卡。你的前 500 个积分已包含在内。
- **拥有 Zapier 账户：** 任何计划都可以使用。免费计划可以手动运行 Zaps；付费计划支持定时和多步骤 Zaps。
- **无需编写代码：** 本指南中的所有操作都通过 Zapier 的可视化编辑器完成。

## 设置

<Steps>
  <Step title="在 Zapier 中找到 Olostep">
    打开任何 Zap，点击 **+** 添加一个操作，然后搜索 **Olostep**。从结果中选择 **Olostep**。

    ![在 Zapier 操作选择器中搜索 Olostep](/images/integrations/zappier/step-1.png)
  </Step>

  <Step title="连接你的账户">
    点击 **登录到 Olostep**。将你的 API 密钥粘贴到字段中，然后点击 **是，继续到 Olostep**。Zapier 将验证密钥并保存凭据以供将来所有 Zaps 使用。

    ![Zapier 中 Olostep 账户连接屏幕，带有 API 密钥字段](/images/integrations/zappier/step-2.png)

    从 [Olostep 仪表板获取你的密钥 →](https://olostep.com/dashboard)
  </Step>

  <Step title="选择一个操作">
    连接账户后，从 **Action** 下拉菜单中选择五个 Olostep 操作之一，并填写所需字段。

    ![Zapier 中的 Olostep 操作下拉菜单，显示可用操作](/images/integrations/zappier/step-3.png)
  </Step>

  <Step title="配置抓取网站（URL + 格式）">
    - **要抓取的 URL：** 从你的触发器映射或手动输入
    - **输出格式：** 选择 `Markdown`、`HTML`、`JSON` 或 `Text`

    ![在 Zapier 中配置抓取网站，带有 URL 和输出格式字段](/images/integrations/zappier/step-4.png)
  </Step>

  <Step title="测试和发布">
    点击 **测试步骤** 以运行实时请求并确认输出。一旦看起来正确，连接下游步骤并发布你的 Zap。

    ![Zapier 测试步骤结果和发布流程](/images/integrations/zappier/step-5.png)
  </Step>
</Steps>

## 操作

<CardGroup cols={2}>
  <Card title="抓取网站" icon="file-lines">
    从任何 URL 提取内容为 Markdown、HTML、JSON 或纯文本。处理 JS 渲染的页面，支持可选的等待时间和国家定位。
  </Card>

  <Card title="询问 AI 答案" icon="sparkles">
    提出自然语言问题，并获得基于你提供的页面或实时网络搜索的引用答案。
  </Card>

  <Card title="批量抓取 URL" icon="layer-group">
    在一个作业中提交最多 100,000 个 URL，进行并行处理。返回一个 `batch_id`；异步检索结果。
  </Card>

  <Card title="创建爬网" icon="spider-web">
    从一个 URL 开始，跟随链接并抓取所有子页面。适用于文档站点、博客或全站点摄取。返回一个 `crawl_id`。
  </Card>

  <Card title="创建地图" icon="map">
    获取网站上的每个 URL，而不抓取内容。可在批处理作业前用于发现。返回一个 `map_id`。
  </Card>
</CardGroup>

<Note>
  **批处理、爬网和地图是异步的。** 存储返回的 ID，并使用延迟步骤或第二个 Zap 在处理完成后检索结果。
</Note>

## 示例工作流程：每日竞争对手简报

**它的功能：** 每天早上，这个 Zap 抓取竞争对手页面，询问 Olostep AI 答案以总结该页面的关键更新，并将最终简报连同引用一起发布到 Slack。

**使用的节点：** Schedule by Zapier -> Olostep Scrape Website -> Olostep Ask AI Answer -> Slack

![Zapier 画布：Schedule 触发器连接到 Olostep Scrape、Olostep Ask AI Answer 和 Slack](/images/integrations/zappier/workflow-example.png)

---

### 第 1 步：添加 Schedule 触发器

在 Zapier 中，创建一个新的 Zap，并将触发器设置为 **Schedule by Zapier**。设置为 **每天** 在适合你团队的时间运行（例如上午 8 点）。

### 第 2 步：添加 Olostep 抓取网站

添加一个 **Olostep** 操作并选择 **抓取网站**。设置：
- **URL：** `https://competitor.com/blog`
- **输出格式：** Markdown

Markdown 保持页面内容清晰，以便下一步 AI 答案使用。

### 第 3 步：添加 Olostep 询问 AI 答案

添加第二个 **Olostep** 操作并选择 **询问 AI 答案**。设置：
- **问题：** `今天这个页面上的关键更新、公告、产品更改或价格更改是什么？返回一个简洁的简报，使用项目符号并包括引用。`
- **上下文 URL（JSON 数组）：** 映射第 2 步的 **抓取 URL**
- **格式：** Markdown
- **包括引用：** true

### 第 4 步：发送到 Slack

添加一个 **Slack** 操作，设置为 **发送频道消息**。映射：
- **频道：** `#competitive-intel`
- **消息文本：** `每日竞争对手简报`
- **消息正文：** `{{Answer (Markdown)}}`

### 你得到的结果

每天早上，你的 Slack 频道会收到一条消息，如：

> **每日竞争对手简报**
> - 推出了一个新的电子商务工作流程集成
> - 更新了价格页面，增加了一个新的中档计划
> - 发布了两篇关于自动化最佳实践的新博客文章
>
> *来源：competitor.com/blog, competitor.com/pricing*

从这里你可以扩展这个 Zap：添加一个过滤器以仅在出现特定关键字时发布，将简报写入 Google Sheets 以进行跟踪，或为不同的竞争对手 URL 运行单独的 Zaps。

## 解析器

在任何抓取或批处理操作的 **Parser** 字段中添加一个解析器 ID，以获取结构化数据而非原始内容：

| 解析器 | 提取内容 |
|--------|----------|
| `@olostep/amazon-product` | 标题、价格、评分、评论、图片、变体 |
| `@olostep/google-search` | 结果标题、URL、摘要 |
| `@olostep/google-maps` | 商家名称、地址、评分、评论 |
| `@olostep/extract-emails` | 从任何页面提取电子邮件地址 |
| `@olostep/extract-socials` | 社交资料链接（X、GitHub、LinkedIn 等） |
| `@olostep/extract-calendars` | Google 日历和 ICS 链接 |

在 [Olostep 解析器商店 →](https://www.olostep.com/store) 查看完整列表

## Zapier 限制与解决方法

### 任务限制
Zapier 将每个操作计为一个任务，计入你计划的限制。

**解决方法：** 使用 **批量抓取 URL** 以单个任务处理多个 URL，而不是循环单个抓取操作。

### 执行超时
Zaps 在 30 秒后超时。爬网和大型批处理作业需要更长时间。

**解决方法：** 存储返回的 `crawl_id` 或 `batch_id`，并在由 webhook 或计划延迟触发的单独 Zap 中检索结果。

### 数据大小限制
Zapier 限制可以在步骤之间传递的数据大小，这可能是大抓取负载的问题。

**解决方法：** 使用 Olostep 返回的托管输出 URL 单独获取大内容，而不是在步骤之间传递原始内容。

### 轮询触发器
大多数 Zapier 触发器以 5-15 分钟的间隔轮询，而不是即时。

**解决方法：** 使用 Zapier 的 **Webhooks** 触发器进行即时通知，或在固定时间安排 Zaps，而不是依赖于近实时轮询。

## 故障排除

<AccordionGroup>
  <Accordion title="API 密钥被拒绝">
    直接从 [olostep.com/dashboard](https://olostep.com/dashboard) 复制密钥，确保没有尾随空格。如果错误仍然存在，请断开并重新连接 Olostep 账户。
  </Accordion>

  <Accordion title="抓取的内容为空">
    增加 **抓取前等待时间**（尝试 2000–5000ms 以应对 JS 密集的页面）。确认 URL 可以在不登录的情况下公开访问。如果特定域名持续失败，请联系 [info@olostep.com](mailto:info@olostep.com)。
  </Accordion>

  <Accordion title="批量 URL 格式错误">
    **要抓取的 URL** 字段需要一个 JSON 数组：
    ```json
    [
      { "url": "https://example.com/page-1", "custom_id": "p1" },
      { "url": "https://example.com/page-2", "custom_id": "p2" }
    ]
    ```
    如果需要，使用 **Code by Zapier** 步骤在上游从你的数据构建此数组。
  </Accordion>

  <Accordion title="达到速率限制">
    在抓取操作之间添加一个 **延迟** 步骤，或切换到 **批量抓取 URL** 而不是循环单个抓取。检查当前使用情况在 [仪表板](https://olostep.com/dashboard)。
  </Accordion>

  <Accordion title="Zap 在爬网或批处理中超时">
    这些操作是异步设计的。立即在操作后存储返回的 ID，然后使用第二个 Zap 或计划轮询稍后检索结果。
  </Accordion>
</AccordionGroup>
