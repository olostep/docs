## 安装

### 1. 在 Zapier 中找到 Olostep

在 Zapier 应用目录中或创建新 Zap 时搜索 "Olostep"：

1. 前往 [Olostep 在 Zapier](https://zapier.com/apps/olostep/integrations)
2. 点击 "Create Zap"
3. 在应用选择器中搜索 "Olostep"
4. 选择 Olostep 应用

或者，直接访问 [Olostep 集成页面](https://zapier.com/apps/olostep/integrations) 浏览可用操作并创建您的第一个 Zap。

### 2. 连接您的账户

首次在 Zap 中使用 Olostep 时，系统会提示您连接您的账户：

1. 点击 "Sign in to Olostep"
2. 输入您的 Olostep API 密钥
3. 点击 "Yes, Continue to Olostep"

从 [Olostep 仪表板](https://olostep.com/dashboard) 获取您的 API 密钥。

## 可用操作

### 抓取网站

从单个 URL 提取内容。支持多种格式和 JavaScript 渲染。

**使用场景：**
- 监控特定页面的变化
- 从电商网站提取产品信息
- 收集新闻文章或博客文章的数据
- 为内容聚合提取内容

**配置：**

<ParamField path="URL to Scrape" type="string" required>
  要抓取的网站 URL（必须包含 http:// 或 https://）
</ParamField>

<ParamField path="Output Format" type="dropdown" default="Markdown">
  选择格式：Markdown、HTML、JSON 或纯文本
</ParamField>

<ParamField path="Country Code" type="string">
  用于特定位置内容的国家代码（例如 "US", "GB", "CA"）
</ParamField>

<ParamField path="Wait Before Scraping" type="integer">
  JavaScript 渲染的等待时间（毫秒）（0-10000）
</ParamField>

<ParamField path="Parser" type="string">
  用于专业提取的可选解析器 ID（例如 "@olostep/amazon-product"）
</ParamField>

**输出字段：**
- 抓取 ID
- 抓取的 URL
- Markdown 内容
- HTML 内容
- JSON 内容
- 文本内容
- 状态
- 时间戳
- 截图 URL（如果可用）
- 页面元数据

**示例工作流：**

<AccordionGroup>
  <Accordion title="监控竞争对手定价">
    **触发器：** 计划（每天上午 9 点）
    
    **操作：** Olostep - 抓取网站
    - URL：竞争对手产品页面
    - 格式：JSON
    - 解析器：@olostep/amazon-product
    
    **操作：** Google Sheets - 创建行
    - 将价格数据添加到跟踪电子表格
    
    **操作：** Gmail - 发送邮件（如果价格下降）
    - 提醒团队价格变化
  </Accordion>

  <Accordion title="提取并保存博客文章">
    **触发器：** RSS by Zapier - 新项目在 Feed 中
    
    **操作：** Olostep - 抓取网站
    - URL：\{\{Post URL\}\}
    - 格式：Markdown
    
    **操作：** Notion - 创建页面
    - 将文章内容保存到 Notion 数据库
  </Accordion>

  <Accordion title="线索丰富">
    **触发器：** Google Sheets - 新行
    
    **操作：** Olostep - 抓取网站
    - URL：来自表格的公司网站
    - 格式：Markdown
    
    **操作：** OpenAI - 完成文本
    - 使用 AI 提取公司信息
    
    **操作：** Google Sheets - 更新行
    - 将丰富的数据添加回表格
  </Accordion>
</AccordionGroup>

### 批量抓取 URL

并行处理多个 URL（一次最多 100,000 个）。非常适合大规模数据提取。

**使用场景：**
- 抓取整个产品目录
- 从多个搜索结果中提取数据
- 处理电子表格中的 URL 列表
- 批量内容提取

**配置：**

<ParamField path="URLs to Scrape" type="text" required>
  包含 url 和 custom_id 字段的 JSON 对象数组。
  
  示例：`[{"url":"https://example.com","custom_id":"site1"}]`
</ParamField>

<ParamField path="Output Format" type="dropdown" default="Markdown">
  为所有 URL 选择格式：Markdown、HTML、JSON 或纯文本
</ParamField>

<ParamField path="Country Code" type="string">
  用于特定位置抓取的国家代码
</ParamField>

<ParamField path="Wait Before Scraping" type="integer">
  JavaScript 渲染的等待时间（毫秒）
</ParamField>

<ParamField path="Parser" type="string">
  用于专业提取的可选解析器 ID
</ParamField>

**输出字段：**
- 批次 ID（用于稍后检索结果）
- 状态
- 总 URL 数
- 创建时间
- 请求格式
- 国家代码
- 使用的解析器

**示例工作流：**

<AccordionGroup>
  <Accordion title="抓取产品目录">
    **触发器：** Webhook - 接收 POST 请求
    
    **操作：** Code by Zapier - 运行 Python
    - 将 CSV/列表转换为 JSON 数组格式
    
    **操作：** Olostep - 批量抓取 URL
    - URL：\{\{前一步的 JSON 数组\}\}
    - 格式：JSON
    - 解析器：@olostep/amazon-product
    
    **操作：** Webhook - POST
    - 将批次 ID 发送到您的系统以便检索
  </Accordion>

  <Accordion title="每日内容监控">
    **触发器：** 计划 - 每天上午 6 点
    
    **操作：** Google Sheets - 获取行
    - 获取要监控的 URL
    
    **操作：** Code by Zapier - 格式化 URL
    - 转换为批量数组格式
    
    **操作：** Olostep - 批量抓取 URL
    - 一次处理所有 URL
    
    **操作：** Slack - 发送消息
    - 通知团队抓取已完成
  </Accordion>
</AccordionGroup>

- 爬取并存档整个文档站点

- 监控网站结构变化

**配置：**

<ParamField path="Start URL" type="string" required>

</ParamField>

<ParamField path="Maximum Pages" type="integer" default="10">
  要爬取的最大页面数
</ParamField>

<ParamField path="Follow Links" type="boolean" default="true">
  是否跟随页面上的链接
</ParamField>

<ParamField path="Output Format" type="dropdown" default="Markdown">
  抓取内容的格式
</ParamField>

<ParamField path="Country Code" type="string">
  用于特定位置爬取的可选国家代码
</ParamField>

<ParamField path="Parser" type="string">
  用于专业内容提取的可选解析器 ID
</ParamField>

**输出字段：**
- 爬取 ID（用于稍后检索结果）
- 对象类型
- 状态
- 起始 URL
- 最大页面数
- 跟随链接
- 创建时间戳
- 格式

**示例工作流：**

### 创建地图

提取网站的所有 URL，用于内容发现和站点结构分析。

**使用场景：**
- 构建站点地图和站点结构图
- 在批量抓取前发现所有页面
- 查找损坏或缺失的页面
- SEO 审核和分析

**配置：**

<ParamField path="Website URL" type="string" required>
  要提取链接的网站 URL（必须包含 http:// 或 https://）
</ParamField>

<ParamField path="Search Query" type="string">
  用于过滤 URL 的可选搜索查询（例如 "blog"）
</ParamField>

<ParamField path="Top N URLs" type="integer">
  限制返回的 URL 数量
</ParamField>

<ParamField path="Include URL Patterns" type="string">
  包含特定路径的 Glob 模式（例如 "/blog/**"）
</ParamField>

<ParamField path="Exclude URL Patterns" type="string">
  排除特定路径的 Glob 模式（例如 "/admin/**"）
</ParamField>

**输出字段：**
- 地图 ID
- 对象类型
- 网站 URL
- 找到的总 URL 数
- URL（JSON 数组）
- 搜索查询
- Top N 限制

**示例工作流：**

<AccordionGroup>
  <Accordion title="发现并抓取">
    **触发器：** Zapier 中的按钮点击
    
    **操作：** Olostep - 创建地图
    - URL：https://example.com
    - 包含模式：/products/**
    - Top N：500
    
    **操作：** Code by Zapier - 提取 URL
    - 从地图结果中解析 URL
    
    **操作：** Olostep - 批量抓取 URL
    - URL：\{\{来自地图的 URL\}\}
    - 格式：JSON
    
    **操作：** Google Sheets - 创建行
    - 将所有产品数据添加到电子表格
  </Accordion>

  <Accordion title="SEO 站点审核">
    **触发器：** 计划 - 每月
    
    **操作：** Olostep - 创建地图
    - URL：您的网站
    - Top N：1000
    
    **操作：** Airtable - 创建记录
    - 存储所有 URL 以供跟踪
    
    **操作：** Slack - 发送消息
    - 报告找到的总页面数
  </Accordion>
</AccordionGroup>

### 询问 AI 答案

使用网络搜索或您提供的上下文 URL 获取 AI 驱动的答案。非常适合研究自动化、内容生成和数据分析。

**使用场景：**
- 带引用的研究自动化
- 从多个 URL 总结内容
- 生成带有来源参考的报告
- 使用特定网页回答问题
- 内容分析和提取

**配置：**

<ParamField path="Question" type="string" required>
  您希望 Olostep Answers 回答的问题
</ParamField>

<ParamField path="Context URLs (JSON Array)" type="string">
  用于回答的可选 JSON URL 数组（例如 ["https://example.com/page1", "https://example.com/page2"]）
</ParamField>

<ParamField path="Search Query" type="string">
  如果没有提供上下文 URL，则用于检索来源的可选查询（例如 "site:example.com pricing"）
</ParamField>

<ParamField path="Number of Sources" type="integer">
  用于生成答案的来源数量限制
</ParamField>

<ParamField path="Format" type="dropdown" default="Markdown">
  选择答案格式：Markdown、JSON 或纯文本
</ParamField>

<ParamField path="Include Citations" type="boolean" default="true">
  是否在响应中包含引用/来源
</ParamField>

**输出字段：**
- 答案 ID
- 对象类型
- 问题
- 答案（文本）
- 答案（Markdown）
- 答案（JSON）
- 引用（带有 URL、标题、摘要的 JSON 数组）
- 创建时间戳
- 格式
- 上下文 URL
- 搜索查询
- 来源数量

**示例工作流：**

<AccordionGroup>

  <Accordion title="从 URL 总结内容">

    **操作：** Olostep - 询问 AI 答案
    - 问题："总结这些页面的关键点"

    **操作：** Notion - 创建页面

## 多步骤工作流

### 完整的产品抓取管道

构建全面的产品数据管道：

<Steps>
  <Step title="发现产品 URL">
    使用 **创建地图** 找到目标网站上的所有产品页面
    - 包含模式：`/products/**`
    - 排除模式：`/cart/**`, `/checkout/**`
  </Step>
  
  <Step title="批量处理产品">
    使用 **批量抓取 URL** 提取所有产品数据
    - 格式：JSON
    - 解析器：如果可用，使用特定产品解析器
  </Step>
  
  <Step title="存储在数据库中">
    将批次 ID 发送到您的系统或等待并检索结果
    - 使用 Airtable、Google Sheets 或您的数据库
  </Step>
  
  <Step title="监控变化">
    安排每日抓取以跟踪价格/可用性变化
    - 与现有数据进行比较
    - 在重大变化时发出警报
  </Step>
</Steps>

### SEO 内容策略

分析竞争对手并规划内容：

<Steps>
  <Step title="映射竞争对手网站">
    在竞争对手网站上使用 **创建地图**
    - 提取所有博客文章和内容页面
  </Step>
  
  <Step title="抓取内容">
    使用 **批量抓取 URL** 获取完整内容
    - 格式：Markdown，便于分析
  </Step>
  
  <Step title="AI 分析">
    使用 OpenAI 分析主题和关键词
    - 识别内容差距
    - 找到热门话题
  </Step>
  
  <Step title="创建内容日历">
    将见解添加到 Notion 或 Airtable
    - 规划您的内容策略
  </Step>
</Steps>

## 专业解析器

Olostep 提供了针对流行网站的预构建解析器。在 `Parser` 字段中使用它们：

<CardGroup cols={2}>
  <Card title="Google 搜索" icon="google">
    `@olostep/google-search`
    
    提取：搜索结果、标题、摘要、URL
  </Card>
  
</CardGroup>

### 使用解析器

只需将解析器 ID 添加到 Parser 字段：

```
操作：Olostep - 抓取网站
  - URL：https://www.amazon.com/dp/PRODUCT_ID
  - 格式：JSON
  - 解析器：@olostep/amazon-product
```

解析器会自动提取特定于该网站类型的结构化数据。

## 与流行应用集成

### Google Sheets

非常适合数据收集和跟踪：

```
1. Olostep 抓取网站
2. 过滤或转换数据
3. Google Sheets - 创建/更新行
```

- 价格跟踪电子表格

- 内容库存

### Airtable

使用抓取的数据构建强大的数据库：

```
1. Olostep 抓取或爬取
2. 代码 - 格式化数据
3. Airtable - 创建记录
```

## 最佳实践

<AccordionGroup>
  <Accordion title="使用批量处理多个 URL">
    当抓取超过 3-5 个 URL 时，使用 **批量抓取 URL** 而不是多个 **抓取网站** 操作。批量处理是：
    - 更快（并行处理）
    - 更具成本效益
    - 更易于管理
    - 更适合速率限制
  </Accordion>

  <Accordion title="设置适当的等待时间">
    对于 JavaScript 密集型网站，使用 "Wait Before Scraping" 参数：
    - 简单网站：0-1000ms
    - 动态网站：2000-3000ms
    - 重 JavaScript：5000-8000ms
    
    测试不同的值以找到最佳等待时间。
  </Accordion>

  <Accordion title="使用专业解析器">
    对于流行网站（Amazon、LinkedIn、Google），使用预构建的解析器：
    - 自动获取结构化数据
    - 提取更可靠
    - 无需自定义解析
    - 由 Olostep 维护
  </Accordion>

  <Accordion title="在抓取前过滤">
    使用 Zapier 的过滤器操作以避免不必要的抓取：
    - 检查 URL 是否已更改
    - 验证数据是否最近已被抓取
    - 在抓取前应用业务逻辑
    
    这可以节省 API 积分和执行时间。
  </Accordion>

  <Accordion title="处理异步操作">
    批量、爬取和地图操作是异步的：
    - 存储返回的 ID（batch_id, crawl_id, map_id）
    - 如果立即检索，请使用延迟操作
    - 考虑使用 webhook 回调完成
    - 为检索设置单独的 Zaps
  </Accordion>

  <Accordion title="妥善存储结果">
    根据您的需求选择合适的存储：
    - **Google Sheets**：简单跟踪，团队协作
    - **Airtable**：关系数据，丰富格式
    - **数据库**：大规模，复杂查询
    - **Notion**：知识库，文档
  </Accordion>

  <Accordion title="监控和警报">
    为您的抓取工作流设置监控：
    - 在 Zaps 中使用错误路径
    - 在失败时发送警报到 Slack/Email
    - 在 Olostep 仪表板中跟踪 API 使用情况
    - 记录重要指标
  </Accordion>
</AccordionGroup>

## 故障排除

<AccordionGroup>
  <Accordion title="身份验证失败">
    **错误**："无效的 API 密钥"
    
    **解决方案**：
    - 从 [仪表板](https://olostep.com/dashboard) 检查 API 密钥
    - 确保 API 密钥中没有多余的空格
    - 在 Zapier 中重新连接您的 Olostep 账户
    - 验证 API 密钥是否有效
  </Accordion>

  <Accordion title="抓取返回空内容">
    **错误**：内容字段为空
    
    **解决方案**：
    - 增加 "Wait Before Scraping" 时间
    - 检查网站是否需要登录
    - 尝试不同的格式（HTML vs Markdown）
    - 验证 URL 是否可访问
    - 检查网站是否阻止自动访问
  </Accordion>

  <Accordion title="批量数组格式错误">
    **错误**："批量数组的 JSON 格式无效"
    
    **解决方案**：
    - 使用格式：`[{"url":"https://example.com","custom_id":"id1"}]`
    - 确保正确的 JSON 语法
    - 使用代码步骤正确格式化 URL
    - 使用在线验证器测试 JSON
  </Accordion>

  <Accordion title="超出速率限制">
    **错误**："超出速率限制"
    
    **解决方案**：
    - 通过延迟间隔 Zap 运行
    - 使用批量处理而不是单个抓取
    - 升级您的 Olostep 计划
    - 在仪表板中检查速率限制
  </Accordion>

  <Accordion title="URL 未抓取">
    **错误**：特定 URL 抓取失败
    
    **解决方案**：
    - 验证 URL 格式（包含 http:// 或 https://）
    - 检查 URL 是否需要身份验证
    - 首先在浏览器中测试 URL
    - 尝试使用国家参数
    - 对于被阻止的域名，联系支持
  </Accordion>
</AccordionGroup>

## Zapier 限制与解决方案

### 任务限制
Zapier 根据您的计划有任务限制。每个 Olostep 操作计为 1 个任务。

**解决方案**：使用批量处理将多个 URL 抓取为单个任务。

### 执行时间
Zaps 在 30 秒后超时。爬取和大型批次可能需要更长时间。

**解决方案**：存储 ID 并在单独的 Zap 中检索结果或使用 webhooks。

### 数据大小
Zapier 对每个任务的数据大小有限制。

**解决方案**：在输出中使用托管 URL 单独检索大内容。

### 轮询 vs 即时
触发器是基于轮询的（每 5-15 分钟检查一次）。

**解决方案**：使用 webhooks 获取即时通知或在特定时间安排。
