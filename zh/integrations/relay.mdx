## 安装

### 1. 在 Relay.app 中找到 Olostep

在创建新工作流时，在 Relay.app 中搜索“Olostep”：

1. 前往 [Olostep on Relay.app](https://www.relay.app/apps/olostep/integrations)
2. 点击“免费开始”或“创建工作流”
3. 在应用选择器中搜索“Olostep”
4. 选择 Olostep 应用

或者，直接访问 [Olostep 集成页面](https://www.relay.app/apps/olostep/integrations) 浏览可用操作并创建您的第一个工作流。

### 2. 连接您的账户

首次在工作流中使用 Olostep 时，系统会提示您连接账户：

1. 点击“连接 Olostep”
2. 输入您的 Olostep API 密钥
3. 授权连接

从 [Olostep Dashboard](https://olostep.com/dashboard) 获取您的 API 密钥。

## 可用操作

### 抓取 URL

从单个 URL 提取内容。支持多种格式和 JavaScript 渲染。

**使用案例：**
- 监控特定页面的更改
- 从电商网站提取产品信息
- 从新闻文章或博客文章中收集数据
- 为内容聚合拉取内容

**配置：**

<ParamField path="URL" type="string" required>
  要抓取的网站 URL（必须包含 http:// 或 https://）
</ParamField>

<ParamField path="Format" type="dropdown" default="Markdown">
  选择格式：Markdown、HTML、JSON 或纯文本
</ParamField>

<ParamField path="Country Code" type="string">
  用于特定位置内容的国家代码（例如，“US”、“GB”、“CA”）
</ParamField>

<ParamField path="Wait Before Scraping" type="integer">
  JavaScript 渲染的等待时间（0-10000 毫秒）
</ParamField>

<ParamField path="Parser" type="string">
  用于专门提取的可选解析器 ID（例如，“@olostep/amazon-product”）
</ParamField>

**输出字段：**
- 抓取 ID
- 抓取的 URL
- Markdown 内容
- HTML 内容
- JSON 内容
- 文本内容
- 状态
- 时间戳
- 截图 URL（如果可用）
- 页面元数据

**示例工作流：**

<AccordionGroup>
  <Accordion title="监控竞争对手定价">
    **触发器：** 计划（每天上午 9 点）
    
    **操作：** Olostep - 抓取 URL
    - URL：竞争对手产品页面
    - 格式：JSON
    - 解析器：@olostep/amazon-product
    
    **操作：** Google Sheets - 创建行
    - 将价格数据添加到跟踪电子表格
    
    **人工参与：** 审核价格变动
    - 批准显著的价格下降
    
    **操作：** Slack - 发送消息
    - 通知团队已批准的价格变动
  </Accordion>

  <Accordion title="提取并保存博客文章">
    **触发器：** RSS Feed - 新项目
    
    **操作：** Olostep - 抓取 URL
    - URL：\{\{Post URL\}\}
    - 格式：Markdown
    
    **操作：** Notion - 创建页面
    - 将文章内容保存到 Notion 数据库
  </Accordion>

  <Accordion title="使用 AI 进行线索丰富">
    **触发器：** Google Sheets - 新行
    
    **操作：** Olostep - 抓取 URL
    - URL：来自表格的公司网站
    - 格式：Markdown
    
    **AI 操作：** OpenAI - 提取数据
    - 提取：公司规模、行业、产品
    
    **操作：** Google Sheets - 更新行
    - 将丰富的数据添加回表格
  </Accordion>

## Relay.app 优势

Relay.app 相较于传统自动化平台具有多项优势：

### 人工参与工作流

在您的工作流中直接添加手动审核步骤：

```
1. Olostep 抓取网站
2. 人工审核步骤 ← 审核并批准数据
3. 仅在批准后保存到数据库
```

非常适合：
- 数据处理前的质量控制
- 对提取内容进行人工判断
- 敏感数据的审批工作流
- 审核 AI 生成的摘要

### 嵌入式 AI 功能

在您的工作流中直接使用 AI，无需外部集成：

- **AI 决策**：让 AI 根据抓取的内容决定工作流路径
- **智能过滤**：自动过滤和分类抓取的数据
- **内容摘要**：在存储前总结长篇文章
- **数据提取**：使用 AI 从非结构化内容中提取结构化数据

### 更好的用户体验和测试

Relay.app 的用户友好界面使工作流构建更容易：

- 清晰的测试信息
- 有用的错误消息
- 可视化工作流构建器
- 简单的调试和故障排除

## 流行的工作流示例

### 带审核的电商价格监控

通过人工审批监控竞争对手价格：

```
触发器：计划（每小时）
↓
操作：Olostep - 抓取 URL
  - URL：竞争对手产品页面
  - 格式：JSON
  - 解析器：@olostep/amazon-product
↓
人工审核步骤
  - 审核价格变动
  - 批准或拒绝
↓
操作：Slack - 发送消息（如果批准）
  - 警报：“价格变为 $\{\{price\}\}”
```

### 使用 AI 的内容聚合

从多个来源聚合和增强内容：

```
触发器：Google Sheets - 新行
↓
操作：Olostep - 抓取 URL
  - URL：\{\{URL from sheet\}\}
  - 格式：Markdown
↓
AI 操作：总结内容
  - 生成摘要
↓
操作：Airtable - 创建记录
  - 存储带有 AI 摘要的文章
```

### 线索丰富管道

通过网络信息和 AI 丰富线索数据：

```
触发器：HubSpot - 新联系人
↓
操作：Olostep - 抓取 URL
  - URL：\{\{Company website\}\}
  - 格式：Markdown
↓
AI 操作：提取公司信息
  - 提取：公司规模、行业、产品
↓
人工审核步骤
  - 验证 AI 提取的准确性
↓
操作：HubSpot - 更新联系人
  - 添加经过验证的丰富数据
```

## 故障排除

<AccordionGroup>
  <Accordion title="身份验证失败">
    **错误**：“无效的 API 密钥”
    
    **解决方案**：
    - 从 [dashboard](https://olostep.com/dashboard) 检查 API 密钥
    - 确保 API 密钥中没有多余的空格
    - 在 Relay.app 中重新连接您的 Olostep 账户
    - 验证 API 密钥是否有效
  </Accordion>

  <Accordion title="抓取返回空内容">
    **错误**：内容字段为空
    
    **解决方案**：
    - 增加“抓取前等待”时间
    - 检查网站是否需要登录
    - 尝试不同的格式（HTML 与 Markdown）
    - 验证 URL 是否可访问
    - 检查网站是否阻止自动访问
  </Accordion>

  <Accordion title="超出速率限制">
    **错误**：“超出速率限制”
    
    **解决方案**：
    - 通过延迟间隔工作流运行
    - 使用批量处理而非单个抓取
    - 升级您的 Olostep 计划
    - 在仪表板中检查速率限制
  </Accordion>

  <Accordion title="URL 未抓取">
    **错误**：特定 URL 抓取失败
    
    **解决方案**：
    - 验证 URL 格式（包括 http:// 或 https://）
    - 检查 URL 是否需要身份验证
    - 先在浏览器中测试 URL
    - 尝试使用国家参数
    - 对于被阻止的域联系支持
  </Accordion>
</AccordionGroup>

## Relay.app 与 Zapier 对比

Relay.app 是 Zapier 的现代替代方案，具有多项优势：

**Relay.app 优势：**
- **人工参与**：内置手动审核步骤
- **嵌入式 AI**：AI 功能集成到工作流中
- **更好的用户体验**：更直观的界面和更清晰的错误消息
- **更好的定价**：对许多使用案例更具成本效益
- **更深入的集成**：与流行应用的更全面集成

**何时选择 Zapier：**
- 需要与不常用的应用集成
- 偏好更成熟的平台
- 需要特定的 Zapier 专属功能

有关详细比较，请参见 [Relay.app 的 Zapier 比较](https://www.relay.app/blog/zapier-alternatives)。

## 定价

- **抓取**：按每次抓取收费

查看当前定价在 [olostep.com/pricing](https://olostep.com/pricing)。
