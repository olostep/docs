La integración de Olostep con n8n lleva potentes capacidades de búsqueda web, scraping y crawling a los flujos de trabajo de n8n. Construye flujos de trabajo automatizados que buscan, extraen y estructuran datos web de cualquier sitio sin escribir código.

[Comienza con Olostep en n8n →](https://www.npmjs.com/package/n8n-nodes-olostep)

## Características

La integración proporciona 6 operaciones potentes para la extracción automatizada de datos web:

## Instalación

### 1. Instala el Nodo

Instala el paquete de nodos de Olostep a través de npm:

```bash
npm install n8n-nodes-olostep
```

Luego reinicia n8n para cargar el nuevo nodo.

Alternativamente, sigue la [guía de instalación de nodos comunitarios de n8n](https://docs.n8n.io/integrations/community-nodes/installation/) para instrucciones detalladas.

### 2. Conecta Tu Cuenta

Cuando uses por primera vez el nodo de Olostep en un flujo de trabajo, necesitarás configurar credenciales:

1. Añade el nodo "Olostep Scrape" a tu flujo de trabajo
2. Haz clic en el nodo para abrir sus configuraciones
3. Haz clic en "Crear Nueva Credencial" o selecciona credenciales existentes
4. Ingresa tu clave API de Olostep
5. Haz clic en "Guardar" para almacenar la credencial

Obtén tu clave API desde el [Dashboard de Olostep](https://olostep.com/dashboard).

## Acciones Disponibles

### Extraer Sitio Web

Extrae contenido de una URL única. Soporta múltiples formatos y renderizado de JavaScript.

**Casos de Uso:**
- Monitorear cambios en páginas específicas
- Extraer información de productos de sitios de comercio electrónico
- Recopilar datos de artículos de noticias o blogs
- Obtener contenido para agregación de contenido

**Configuración:**

<ParamField path="URL para Extraer" type="string" required>
  URL del sitio web para extraer (debe incluir http:// o https://)
</ParamField>

<ParamField path="Formato de Salida" type="dropdown" default="Markdown">
  Elige formato: Markdown, HTML, JSON, o Texto Plano
</ParamField>

<ParamField path="Código de País" type="string">
  Código de país para contenido específico de la ubicación (por ejemplo, "US", "GB", "CA")
</ParamField>

<ParamField path="Esperar Antes de Extraer" type="integer">
  Tiempo de espera en milisegundos para renderizado de JavaScript (0-10000)
</ParamField>

<ParamField path="Parser" type="string">
  ID de parser opcional para extracción especializada (por ejemplo, "@olostep/amazon-product")
</ParamField>

**Campos de Salida:**
- ID de Extracción
- URL Extraída
- Contenido Markdown
- Contenido HTML
- Contenido JSON
- Contenido de Texto
- Estado
- Marca de Tiempo
- URL de Captura de Pantalla (si está disponible)
- Metadatos de la Página

**Ejemplos de Flujos de Trabajo:**

<AccordionGroup>
  <Accordion title="Monitorear Precios de Competencia">
    **Disparador:** Programar (Todos los días a las 9 AM)
    
    **Acción:** Olostep - Extraer Sitio Web
    - URL: Página de producto del competidor
    - Formato: JSON
    - Parser: @olostep/amazon-product
    
    **Acción:** Google Sheets - Añadir Fila
    - Añadir datos de precios a la hoja de seguimiento
    
    **Acción:** Email - Enviar Email (Si el precio baja)
    - Alertar al equipo sobre cambios de precio
  </Accordion>

  <Accordion title="Extraer y Guardar Entradas de Blog">
    **Disparador:** RSS Feed - Nuevo Elemento
    
    **Acción:** Olostep - Extraer Sitio Web
    - URL: \{\{$json.link\}\}
    - Formato: Markdown
    
    **Acción:** Notion - Crear Página
    - Guardar contenido del artículo en la base de datos de Notion
  </Accordion>

  <Accordion title="Enriquecimiento de Leads">
    **Disparador:** Google Sheets - Nueva Fila
    
    **Acción:** Olostep - Extraer Sitio Web
    - URL: Sitio web de la empresa desde la hoja
    - Formato: Markdown
    
    **Acción:** OpenAI - Completar Texto
    - Extraer información de la empresa usando IA
    
    **Acción:** Google Sheets - Actualizar Fila
    - Añadir datos enriquecidos de nuevo a la hoja
  </Accordion>
</AccordionGroup>

### Buscar

Busca en la Web una consulta dada y obtén resultados estructurados (resultados de búsqueda basados en parser, no IA).

**Casos de Uso:**
- Flujos de trabajo de investigación automatizados
- Descubrimiento y enriquecimiento de leads
- Análisis competitivo
- Investigación de contenido

**Configuración:**

<ParamField path="Consulta" type="string" required>
  Consulta de búsqueda
</ParamField>

**Salida:** Devuelve resultados de búsqueda estructurados como JSON con títulos, URLs, fragmentos y metadatos.

**Ejemplos de Flujos de Trabajo:**

    **Acción:** Olostep - Buscar
    - Consulta: "últimos desarrollos en IA"
    
    **Acción:** Código - Procesar Resultados
    - Extraer y dar formato a la información clave
    
    **Acción:** Notion - Crear Páginas

  <Accordion title="Descubrimiento de Leads">

### Extraer Lotes de URLs

Extrae hasta 10k URLs al mismo tiempo. Perfecto para la extracción de datos a gran escala.

**Casos de Uso:**
- Extraer catálogos de productos completos
- Extraer datos de múltiples resultados de búsqueda
- Procesar listas de URLs desde hojas de cálculo
- Extracción masiva de contenido

**Configuración:**

<ParamField path="URLs para Extraer" type="text" required>
  Array JSON de objetos con campos url y custom_id. 
  
  Ejemplo: `[{"url":"https://example.com","custom_id":"site1"}]`
</ParamField>

<ParamField path="Formato de Salida" type="dropdown" default="Markdown">
  Elige formato para todas las URLs: Markdown, HTML, JSON, o Texto Plano
</ParamField>

<ParamField path="Código de País" type="string">
  Código de país para extracción específica de la ubicación
</ParamField>

<ParamField path="Esperar Antes de Extraer" type="integer">
  Tiempo de espera en milisegundos para renderizado de JavaScript
</ParamField>

<ParamField path="Parser" type="string">
  ID de parser opcional para extracción especializada
</ParamField>

**Campos de Salida:**
- ID de Lote (usa esto para recuperar resultados más tarde)
- Estado
- Total de URLs
- Creado En
- Formato Solicitado
- Código de País
- Parser Usado

**Ejemplos de Flujos de Trabajo:**

<AccordionGroup>
  <Accordion title="Extraer Catálogo de Productos">
    **Disparador:** Webhook - Recibir Solicitud POST
    
    **Acción:** Código - Formatear URLs
    - Convertir CSV/lista a formato de array JSON
    
    **Acción:** Olostep - Extraer Lotes de URLs
    - URLs: \{\{$json.urlArray\}\}
    - Formato: JSON
    - Parser: @olostep/amazon-product
    
    **Acción:** Webhook - POST
    - Enviar ID de lote a tu sistema para recuperación
  </Accordion>

  <Accordion title="Monitoreo Diario de Contenido">
    **Disparador:** Programar - Todos los días a las 6 AM
    
    **Acción:** Google Sheets - Leer Filas
    - Obtener URLs para monitorear
    
    **Acción:** Código - Formatear URLs
    - Convertir a formato de array de lote
    
    **Acción:** Olostep - Extraer Lotes de URLs
    - Procesar todas las URLs a la vez
    
    **Acción:** Slack - Enviar Mensaje
    - Notificar al equipo que la extracción está completa
  </Accordion>
</AccordionGroup>

### Crear Rastreo

**Casos de Uso:**
- Rastrear y archivar sitios de documentación completos
- Extraer todos los artículos de blog de un sitio web
- Construir bases de conocimiento a partir de contenido web
- Monitorear cambios en la estructura del sitio web

**Configuración:**

<ParamField path="URL de Inicio" type="string" required>

</ParamField>

<ParamField path="Máximo de Páginas" type="integer" default="10">

<ParamField path="Formato de Salida" type="dropdown" default="Markdown">

</ParamField>

<ParamField path="Código de País" type="string">
  Código de país opcional para rastreo específico de la ubicación
</ParamField>

<ParamField path="Parser" type="string">
  ID de parser opcional para extracción de contenido especializada
</ParamField>

**Campos de Salida:**
- ID de Rastreo (usa esto para recuperar resultados más tarde)
- Tipo de Objeto
- Estado
- URL de Inicio

- Seguir Enlaces
- Marca de Tiempo de Creación
- Formatos

### Crear Mapa

Obtén todas las URLs en un sitio web. Extrae todas las URLs de un sitio web para el descubrimiento de contenido y análisis de la estructura del sitio.

**Casos de Uso:**
- Construir mapas de sitio y diagramas de estructura del sitio
- Descubrir todas las páginas antes de extraer en lote
- Encontrar páginas rotas o faltantes
- Auditorías y análisis de SEO

**Configuración:**

<ParamField path="URL del Sitio Web" type="string" required>
  URL del sitio web para extraer enlaces (debe incluir http:// o https://)
</ParamField>

<ParamField path="Consulta de Búsqueda" type="string">
  Consulta de búsqueda opcional para filtrar URLs (por ejemplo, "blog")
</ParamField>

<ParamField path="Top N URLs" type="integer">
  Limitar el número de URLs devueltas
</ParamField>

<ParamField path="Incluir Patrones de URL" type="string">
  Patrones glob para incluir rutas específicas (por ejemplo, "/blog/**")
</ParamField>

<ParamField path="Excluir Patrones de URL" type="string">
  Patrones glob para excluir rutas específicas (por ejemplo, "/admin/**")
</ParamField>

**Campos de Salida:**
- ID de Mapa
- Tipo de Objeto
- URL del Sitio Web
- Total de URLs Encontradas
- URLs (array JSON)
- Consulta de Búsqueda
- Límite de Top N

**Ejemplos de Flujos de Trabajo:**

<AccordionGroup>
  <Accordion title="Descubrir y Extraer">
    **Disparador:** Manual (Botón)
    
    **Acción:** Olostep - Crear Mapa

    - Incluir Patrones: /products/**
    - Top N: 500
    
    **Acción:** Código - Extraer URLs
    - Analizar URLs del resultado del mapa
    
    **Acción:** Olostep - Extraer Lotes de URLs
    - URLs: \{\{$json.urls\}\}
    - Formato: JSON
    
    **Acción:** Google Sheets - Añadir Filas
    - Añadir todos los datos de productos a la hoja de cálculo

  <Accordion title="Auditoría de Sitio SEO">
    **Disparador:** Programar - Mensualmente
    
    **Acción:** Olostep - Crear Mapa
    - URL: Tu sitio web
    - Top N: 1000
    
    **Acción:** Airtable - Crear Registros
    - Almacenar todas las URLs para seguimiento

</Steps>

### Usando Parsers

Simplemente añade el ID del parser al campo Parser. Dos ejemplos:

```
Acción: Olostep - Extraer Sitio Web
  - URL: https://www.amazon.com/dp/PRODUCT_ID
  - Formato: JSON
  - Parser: @olostep/amazon-product
```

```
Acción: Olostep - Extraer Sitio Web
  - URL: https://example.com/contact
  - Formato: JSON
  - Parser: @olostep/extract-emails
```

El parser extrae automáticamente datos estructurados específicos para la tarea.

## Mejores Prácticas

<AccordionGroup>
  <Accordion title="Usa Procesamiento por Lotes para Múltiples URLs">
    Cuando extraigas más de 3-5 URLs, usa **Extraer Lotes de URLs** en lugar de múltiples acciones de **Extraer Sitio Web**. El procesamiento por lotes es:
    - Mucho más rápido (procesamiento en paralelo)
    - Más rentable
    - Más fácil de gestionar
    - Mejor para límites de tasa
  </Accordion>

  <Accordion title="Configura Tiempos de Espera Apropiados">
    Para sitios pesados en JavaScript, usa el parámetro "Esperar Antes de Extraer":
    - Sitios simples: 0-1000ms
    - Sitios dinámicos: 2000-3000ms
    - JavaScript pesado: 5000-8000ms
    
    Prueba con diferentes valores para encontrar el tiempo de espera óptimo.
  </Accordion>

  <Accordion title="Usa Parsers Especializados">
    Usa parsers preconstruidos (por ejemplo, Amazon, Google, y parsers específicos de tareas de la Tienda Olostep como emails, redes sociales, calendarios):
    - Obtén datos estructurados automáticamente
    - Extracción más confiable
    - No necesitas parsing personalizado
    - Mantenidos por Olostep
  </Accordion>

  <Accordion title="Filtra Antes de Extraer">
    Usa el nodo IF de n8n para evitar extracciones innecesarias:
    - Verifica si la URL ha cambiado
    - Verifica que los datos no se hayan extraído recientemente
    - Aplica lógica de negocio antes de extraer
    
    Esto ahorra créditos de API y tiempo de ejecución.
  </Accordion>

  <Accordion title="Maneja Operaciones Asíncronas">
    Las operaciones de Lote, Rastreo y Mapa son asíncronas:
    - Almacena el ID devuelto (batch_id, crawl_id, map_id)
    - Usa un nodo de Espera si recuperas inmediatamente
    - Considera callbacks de webhook para finalización

## Solución de Problemas

<AccordionGroup>
  <Accordion title="Autenticación Fallida">
    **Error**: "Clave API inválida"
    
    **Soluciones**:
    - Verifica la clave API desde el [dashboard](https://olostep.com/dashboard)
    - Asegúrate de que no haya espacios extra en la clave API
    - Recrea la credencial en n8n
    - Verifica que la clave API esté activa
  </Accordion>

  <Accordion title="Extracción Devuelve Contenido Vacío">
    **Error**: Los campos de contenido están vacíos
    
    **Soluciones**:
    - Aumenta el tiempo de "Esperar Antes de Extraer"
    - Verifica si el sitio web requiere inicio de sesión
    - Prueba con un formato diferente (HTML vs Markdown)
    - Verifica que la URL sea accesible
    - Verifica si el sitio bloquea el acceso automatizado
  </Accordion>

  <Accordion title="Error de Formato de Array de Lote">
    **Error**: "Formato JSON inválido para array de lote"
    
    **Soluciones**:
    - Usa el formato: `[{"url":"https://example.com","custom_id":"id1"}]`
    - Asegúrate de la sintaxis JSON adecuada
    - Usa el nodo de Código para formatear URLs correctamente
    - Prueba el JSON con un validador en línea
  </Accordion>

  <Accordion title="Límite de Tasa Excedido">
    **Error**: "Límite de tasa excedido"
    
    **Soluciones**:
    - Espacia las ejecuciones de flujo de trabajo con nodos de Espera
    - Usa procesamiento por lotes en lugar de extracciones individuales
    - Actualiza tu plan de Olostep
    - Verifica el límite de tasa en el dashboard
  </Accordion>

  <Accordion title="URL No Extraída">
    **Error**: URLs específicas fallan al extraer
    
    **Soluciones**:
    - Verifica el formato de la URL (incluye http:// o https://)
    - Verifica si la URL requiere autenticación
    - Prueba la URL en el navegador primero
    - Prueba con el parámetro de país
    - Contacta al soporte para dominios bloqueados
  </Accordion>
</AccordionGroup>

## Ventajas de n8n

### Autoalojado
n8n es autoalojado, lo que te da control total sobre tus flujos de trabajo y datos. Sin bloqueo de proveedor, sin datos saliendo de tu infraestructura.

### Sin Límites de Tareas
A diferencia de las plataformas de automatización basadas en la nube, n8n no impone límites de tareas. Ejecuta tantos flujos de trabajo como necesites sin costos adicionales.

### Código Abierto
n8n es de código abierto, lo que te permite personalizarlo y extenderlo para adaptarlo a tus necesidades específicas.

### Rentable
n8n autoalojado es gratuito, con alojamiento en la nube opcional disponible. Solo paga por el uso de la API de Olostep.

## Precios

Olostep cobra basado en el uso de la API, independiente de n8n:

- **Extracciones**: Pago por extracción
- **Lotes**: Pago por URL en lote
- **Rastreos**: Pago por página rastreada
- **Mapas**: Pago por operación de mapa

Consulta los precios actuales en [olostep.com/pricing](https://olostep.com/pricing).

**n8n**: n8n autoalojado es gratuito. Alojamiento en la nube disponible con planes pagos opcionales.
