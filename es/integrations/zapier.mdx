La integración de Olostep con Zapier lleva potentes capacidades de scraping web al ecosistema de más de 8,000 aplicaciones de Zapier. Construye flujos de trabajo automatizados que extraen, monitorean y procesan datos web sin escribir código.

[Comienza con Olostep en Zapier →](https://zapier.com/apps/olostep/integrations)

## Características

La integración proporciona 5 acciones poderosas para la extracción automatizada de datos web y la investigación impulsada por IA:

## Instalación

### 1. Encuentra Olostep en Zapier

Busca "Olostep" en el directorio de aplicaciones de Zapier o al crear un nuevo Zap:

1. Ve a [Olostep en Zapier](https://zapier.com/apps/olostep/integrations)
2. Haz clic en "Crear Zap"
3. Busca "Olostep" en el selector de aplicaciones
4. Selecciona la aplicación Olostep

Alternativamente, visita la [página de integración de Olostep](https://zapier.com/apps/olostep/integrations) directamente para explorar las acciones disponibles y crear tu primer Zap.

### 2. Conecta Tu Cuenta

Cuando uses Olostep por primera vez en un Zap, se te pedirá que conectes tu cuenta:

1. Haz clic en "Iniciar sesión en Olostep"
2. Ingresa tu clave API de Olostep
3. Haz clic en "Sí, Continuar a Olostep"

Obtén tu clave API desde el [Panel de Olostep](https://olostep.com/dashboard).

## Acciones Disponibles

### Raspar Sitio Web

Extrae contenido de una URL única. Soporta múltiples formatos y renderizado de JavaScript.

**Casos de Uso:**
- Monitorea páginas específicas para cambios
- Extrae información de productos de sitios de comercio electrónico
- Recolecta datos de artículos de noticias o publicaciones de blogs
- Extrae contenido para agregación de contenido

**Configuración:**

<ParamField path="URL a Raspar" type="string" required>
  URL del sitio web a raspar (debe incluir http:// o https://)
</ParamField>

<ParamField path="Formato de Salida" type="dropdown" default="Markdown">
  Elige formato: Markdown, HTML, JSON o Texto Plano
</ParamField>

<ParamField path="Código de País" type="string">
  Código de país para contenido específico de ubicación (ej., "US", "GB", "CA")
</ParamField>

<ParamField path="Esperar Antes de Raspar" type="integer">
  Tiempo de espera en milisegundos para renderizado de JavaScript (0-10000)
</ParamField>

<ParamField path="Parser" type="string">
  ID de parser opcional para extracción especializada (ej., "@olostep/amazon-product")
</ParamField>

**Campos de Salida:**
- ID de Raspeo
- URL Raspada
- Contenido en Markdown
- Contenido en HTML
- Contenido en JSON
- Contenido en Texto
- Estado
- Marca de Tiempo
- URL de Captura de Pantalla (si está disponible)
- Metadatos de la Página

**Ejemplos de Flujos de Trabajo:**

<AccordionGroup>
  <Accordion title="Monitorear Precios de la Competencia">
    **Disparador:** Programar (Todos los días a las 9 AM)
    
    **Acción:** Olostep - Raspar Sitio Web
    - URL: Página de producto de la competencia
    - Formato: JSON
    - Parser: @olostep/amazon-product
    
    **Acción:** Google Sheets - Crear Fila
    - Agregar datos de precios a la hoja de seguimiento
    
    **Acción:** Gmail - Enviar Correo (Si el precio baja)
    - Alertar al equipo sobre cambios de precio
  </Accordion>

  <Accordion title="Extraer y Guardar Publicaciones de Blog">
    **Disparador:** RSS por Zapier - Nuevo Elemento en el Feed
    
    **Acción:** Olostep - Raspar Sitio Web
    - URL: \{\{Post URL\}\}
    - Formato: Markdown
    
    **Acción:** Notion - Crear Página
    - Guardar contenido del artículo en la base de datos de Notion
  </Accordion>

  <Accordion title="Enriquecimiento de Leads">
    **Disparador:** Google Sheets - Nueva Fila
    
    **Acción:** Olostep - Raspar Sitio Web
    - URL: Sitio web de la empresa desde la hoja
    - Formato: Markdown
    
    **Acción:** OpenAI - Completar Texto
    - Extraer información de la empresa usando IA
    
    **Acción:** Google Sheets - Actualizar Fila
    - Agregar datos enriquecidos de nuevo a la hoja
  </Accordion>
</AccordionGroup>

### Raspar URLs en Lote

Procesa múltiples URLs en paralelo (hasta 100,000 a la vez). Perfecto para la extracción de datos a gran escala.

**Casos de Uso:**
- Raspar catálogos de productos completos
- Extraer datos de múltiples resultados de búsqueda
- Procesar listas de URLs desde hojas de cálculo
- Extracción masiva de contenido

**Configuración:**

<ParamField path="URLs a Raspar" type="text" required>
  Arreglo JSON de objetos con campos url y custom_id. 
  
  Ejemplo: `[{"url":"https://example.com","custom_id":"site1"}]`
</ParamField>

<ParamField path="Formato de Salida" type="dropdown" default="Markdown">
  Elige formato para todas las URLs: Markdown, HTML, JSON o Texto Plano
</ParamField>

<ParamField path="Código de País" type="string">
  Código de país para raspado específico de ubicación
</ParamField>

<ParamField path="Esperar Antes de Raspar" type="integer">
  Tiempo de espera en milisegundos para renderizado de JavaScript
</ParamField>

<ParamField path="Parser" type="string">
  ID de parser opcional para extracción especializada
</ParamField>

**Campos de Salida:**
- ID de Lote (úsalo para recuperar resultados más tarde)
- Estado
- Total de URLs
- Creado En
- Formato Solicitado
- Código de País
- Parser Usado

**Ejemplos de Flujos de Trabajo:**

<AccordionGroup>
  <Accordion title="Raspar Catálogo de Productos">
    **Disparador:** Webhook - Recibir Solicitud POST
    
    **Acción:** Código por Zapier - Ejecutar Python
    - Convertir CSV/lista a formato de arreglo JSON
    
    **Acción:** Olostep - Raspar URLs en Lote
    - URLs: \{\{Arreglo JSON del paso anterior\}\}
    - Formato: JSON
    - Parser: @olostep/amazon-product
    
    **Acción:** Webhook - POST
    - Enviar ID de lote a tu sistema para recuperación
  </Accordion>

  <Accordion title="Monitoreo Diario de Contenido">
    **Disparador:** Programar - Todos los días a las 6 AM
    
    **Acción:** Google Sheets - Obtener Filas
    - Obtener URLs para monitorear
    
    **Acción:** Código por Zapier - Formatear URLs
    - Convertir a formato de arreglo de lote
    
    **Acción:** Olostep - Raspar URLs en Lote
    - Procesar todas las URLs a la vez
    
    **Acción:** Slack - Enviar Mensaje
    - Notificar al equipo que el raspado está completo
  </Accordion>
</AccordionGroup>

<ParamField path="Páginas Máximas" type="integer" default="10">

</ParamField>

<ParamField path="Formato de Salida" type="dropdown" default="Markdown">
  Formato para el contenido raspado
</ParamField>

<ParamField path="Código de País" type="string">
  Código de país opcional para rastreo específico de ubicación
</ParamField>

<ParamField path="Parser" type="string">
  ID de parser opcional para extracción de contenido especializada
</ParamField>

**Campos de Salida:**
- ID de Rastreo (úsalo para recuperar resultados más tarde)
- Tipo de Objeto
- Estado
- URL de Inicio
- Páginas Máximas
- Seguir Enlaces
- Marca de Tiempo de Creación
- Formatos

### Crear Mapa

Extrae todas las URLs de un sitio web para descubrimiento de contenido y análisis de estructura del sitio.

**Casos de Uso:**
- Construir mapas del sitio y diagramas de estructura del sitio
- Descubrir todas las páginas antes de raspar en lote
- Encontrar páginas rotas o faltantes
- Auditorías y análisis de SEO

**Configuración:**

<ParamField path="URL del Sitio Web" type="string" required>
  URL del sitio web para extraer enlaces (debe incluir http:// o https://)
</ParamField>

<ParamField path="Consulta de Búsqueda" type="string">
  Consulta de búsqueda opcional para filtrar URLs (ej., "blog")
</ParamField>

<ParamField path="Top N URLs" type="integer">
  Limitar el número de URLs devueltas
</ParamField>

<ParamField path="Incluir Patrones de URL" type="string">
  Patrones glob para incluir rutas específicas (ej., "/blog/**")
</ParamField>

<ParamField path="Excluir Patrones de URL" type="string">
  Patrones glob para excluir rutas específicas (ej., "/admin/**")
</ParamField>

**Campos de Salida:**
- ID de Mapa
- Tipo de Objeto
- URL del Sitio Web
- Total de URLs Encontradas
- URLs (arreglo JSON)
- Consulta de Búsqueda
- Límite Top N

**Ejemplos de Flujos de Trabajo:**

### Preguntar Respuesta de IA

Obtén respuestas impulsadas por IA a preguntas usando búsqueda web o tus URLs de contexto proporcionadas. Perfecto para automatización de investigación, generación de contenido y análisis de datos.

**Casos de Uso:**
- Automatización de investigación con citas
- Resumir contenido de múltiples URLs
- Generar informes con referencias de fuentes
- Responder preguntas usando páginas web específicas
- Análisis y extracción de contenido

**Configuración:**

<ParamField path="Pregunta" type="string" required>
  La pregunta que quieres que Olostep Respuestas responda
</ParamField>

<ParamField path="URLs de Contexto (Arreglo JSON)" type="string">
  Arreglo JSON opcional de URLs para fundamentar la respuesta (ej., ["https://example.com/page1", "https://example.com/page2"])
</ParamField>

<ParamField path="Consulta de Búsqueda" type="string">
  Consulta opcional para recuperar fuentes de la web si no se proporcionan URLs de contexto (ej., "site:example.com pricing")
</ParamField>

<ParamField path="Número de Fuentes" type="integer">
  Limitar el número de fuentes usadas para generar la respuesta
</ParamField>

<ParamField path="Formato" type="dropdown" default="Markdown">
  Elige formato de respuesta: Markdown, JSON o Texto Plano
</ParamField>

<ParamField path="Incluir Citas" type="boolean" default="true">
  Si se deben incluir citas/fuentes en la respuesta
</ParamField>

**Campos de Salida:**
- ID de Respuesta
- Tipo de Objeto
- Pregunta
- Respuesta (Texto)
- Respuesta (Markdown)
- Respuesta (JSON)
- Citas (arreglo JSON con URLs, títulos, fragmentos)
- Marca de Tiempo de Creación
- Formato
- URLs de Contexto
- Consulta de Búsqueda
- Número de Fuentes

**Ejemplos de Flujos de Trabajo:**

## Flujos de Trabajo de Múltiples Pasos

### Pipeline Completo de Raspado de Productos

Construye un pipeline de datos de productos completo:

<Steps>
  <Step title="Descubrir URLs de Productos">
    Usa **Crear Mapa** para encontrar todas las páginas de productos en el sitio objetivo
    - Incluir patrones: `/products/**`
    - Excluir patrones: `/cart/**`, `/checkout/**`
  </Step>
  
  <Step title="Procesar Productos en Lote">
    Usa **Raspar URLs en Lote** para extraer todos los datos de productos
    - Formato: JSON
    - Parser: Parser específico de producto si está disponible
  </Step>
  
  <Step title="Almacenar en Base de Datos">
    Envía el ID de lote a tu sistema o espera y recupera resultados
    - Usa Airtable, Google Sheets o tu base de datos
  </Step>
  
  <Step title="Monitorear Cambios">
    Programa raspados diarios para rastrear cambios de precio/disponibilidad
    - Compara con datos existentes
    - Alerta sobre cambios significativos
  </Step>
</Steps>

### Estrategia de Contenido SEO

<Steps>
  <Step title="Mapear Sitios de Competencia">

  </Step>
  
  <Step title="Raspar Contenido">
    Usa **Raspar URLs en Lote** para obtener contenido completo
    - Formato: Markdown para análisis fácil
  </Step>
  
  <Step title="Análisis de IA">
    Usa OpenAI para analizar temas y palabras clave
    - Identifica brechas de contenido

  </Step>
  
  <Step title="Crear Calendario de Contenido">
    Agrega conocimientos a Notion o Airtable

  </Step>
</Steps>

## Parsers Especializados

Olostep proporciona parsers pre-construidos para sitios web populares. Úsalos con el campo `Parser`:

<CardGroup cols={2}>
  <Card title="Búsqueda de Google" icon="google">
    `@olostep/google-search`
    
    Extrae: resultados de búsqueda, títulos, fragmentos, URLs
  </Card>
  
</CardGroup>

### Usando Parsers

Simplemente agrega el ID del parser al campo Parser:

```
Acción: Olostep - Raspar Sitio Web
  - URL: https://www.amazon.com/dp/PRODUCT_ID
  - Formato: JSON
  - Parser: @olostep/amazon-product
```

El parser extrae automáticamente datos estructurados específicos para ese tipo de sitio web.

## Mejores Prácticas

<AccordionGroup>
  <Accordion title="Usa Procesamiento en Lote para Múltiples URLs">
    Cuando raspes más de 3-5 URLs, usa **Raspar URLs en Lote** en lugar de múltiples acciones **Raspar Sitio Web**. El procesamiento en lote es:
    - Mucho más rápido (procesamiento en paralelo)
    - Más rentable
    - Más fácil de gestionar
    - Mejor para límites de tasa
  </Accordion>

  <Accordion title="Configura Tiempos de Espera Apropiados">
    Para sitios con mucho JavaScript, usa el parámetro "Esperar Antes de Raspar":
    - Sitios simples: 0-1000ms
    - Sitios dinámicos: 2000-3000ms
    - JavaScript pesado: 5000-8000ms
    
    Prueba con diferentes valores para encontrar el tiempo de espera óptimo.
  </Accordion>

  <Accordion title="Usa Parsers Especializados">
    Para sitios web populares (Amazon, LinkedIn, Google), usa parsers pre-construidos:
    - Obtén datos estructurados automáticamente
    - Extracción más confiable
    - No necesitas parsing personalizado
    - Mantenido por Olostep
  </Accordion>

  <Accordion title="Filtra Antes de Raspar">
    Usa la acción de Filtro de Zapier para evitar raspados innecesarios:
    - Verifica si la URL ha cambiado
    - Verifica que los datos no se hayan raspado recientemente
    - Aplica lógica de negocio antes de raspar
    
    Esto ahorra créditos de API y tiempo de ejecución.
  </Accordion>

  <Accordion title="Maneja Operaciones Asíncronas">
    Las operaciones de Lote, Rastreo y Mapa son asíncronas:
    - Almacena el ID devuelto (batch_id, crawl_id, map_id)
    - Usa una acción de Retraso si recuperas inmediatamente
    - Considera callbacks de webhook para finalización
    - Configura Zaps separados para recuperación
  </Accordion>

  <Accordion title="Almacena Resultados Apropiadamente">
    Elige el almacenamiento correcto según tus necesidades:
    - **Google Sheets**: Seguimiento simple, colaboración en equipo
    - **Airtable**: Datos relacionales, formato enriquecido
    - **Base de Datos**: Gran escala, consultas complejas
    - **Notion**: Base de conocimiento, documentación
  </Accordion>

  <Accordion title="Monitorea y Alerta">
    Configura monitoreo para tus flujos de trabajo de raspado:
    - Usa rutas de Error en Zaps
    - Envía alertas a Slack/Correo en caso de fallos
    - Rastrea el uso de la API en el panel de Olostep
    - Registra métricas importantes
  </Accordion>
</AccordionGroup>

## Solución de Problemas

<AccordionGroup>
  <Accordion title="Fallo de Autenticación">
    **Error**: "Clave API inválida"
    
    **Soluciones**:
    - Verifica la clave API desde el [panel](https://olostep.com/dashboard)
    - Asegúrate de que no haya espacios adicionales en la clave API
    - Reconecta tu cuenta de Olostep en Zapier
    - Verifica que la clave API esté activa
  </Accordion>

  <Accordion title="Raspeo Devuelve Contenido Vacío">
    **Error**: Los campos de contenido están vacíos
    
    **Soluciones**:
    - Aumenta el tiempo de "Esperar Antes de Raspar"
    - Verifica si el sitio web requiere inicio de sesión
    - Prueba con un formato diferente (HTML vs Markdown)
    - Verifica que la URL sea accesible
    - Verifica si el sitio bloquea el acceso automatizado
  </Accordion>

  <Accordion title="Error de Formato de Arreglo de Lote">
    **Error**: "Formato JSON inválido para arreglo de lote"
    
    **Soluciones**:
    - Usa el formato: `[{"url":"https://example.com","custom_id":"id1"}]`
    - Asegúrate de la sintaxis JSON correcta
    - Usa el paso de Código para formatear URLs correctamente
    - Prueba el JSON con un validador en línea
  </Accordion>

  <Accordion title="Límite de Tasa Excedido">
    **Error**: "Límite de tasa excedido"
    
    **Soluciones**:
    - Espacia las ejecuciones de Zap con retrasos
    - Usa procesamiento en lote en lugar de raspados individuales
    - Mejora tu plan de Olostep
    - Verifica el límite de tasa en el panel
  </Accordion>

  <Accordion title="URL No Raspada">
    **Error**: URLs específicas fallan al rasparse
    
    **Soluciones**:
    - Verifica el formato de la URL (incluye http:// o https://)
    - Verifica si la URL requiere autenticación
    - Prueba la URL en el navegador primero
    - Prueba con el parámetro de país
    - Contacta soporte para dominios bloqueados
  </Accordion>
</AccordionGroup>

## Limitaciones de Zapier y Soluciones

### Límites de Tareas
Zapier tiene límites de tareas según tu plan. Cada acción de Olostep cuenta como 1 tarea.

**Solución**: Usa procesamiento en lote para raspar múltiples URLs como una sola tarea.

### Tiempo de Ejecución
Los Zaps se agotan después de 30 segundos. Los rastreos y lotes grandes pueden tardar más.

**Solución**: Almacena el ID y recupera resultados en un Zap separado o usa webhooks.

### Tamaño de Datos
Zapier tiene límites en el tamaño de datos por tarea.

**Solución**: Usa URLs alojadas en la salida para recuperar contenido grande por separado.

### Polling vs Instantáneo
Los disparadores son basados en polling (verifican cada 5-15 minutos).

**Solución**: Usa webhooks para notificaciones instantáneas o programa en momentos específicos.
