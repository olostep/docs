La aplicación verificada de [Olostep en Relay](https://www.relay.app/apps/olostep/integrations) te ofrece dos acciones dentro del constructor visual de Relay: extraer datos de una URL o mapear todos los enlaces en un sitio.

[Ver en Relay →](https://www.relay.app/apps/olostep/integrations)

## Antes de empezar

- **Una cuenta de Olostep con una clave API:** [obtén una gratis](https://olostep.com/dashboard), no se requiere tarjeta de crédito. Tus primeros 500 créditos están incluidos.
- **Una cuenta de Relay:** crea una en [relay.app](https://www.relay.app/).
- **No se requiere programación:** todo en esta guía se realiza a través del editor visual de Relay.

---

## Configuración

<Steps>
  <Step title="Encuentra Olostep en Relay">
    Abre un flujo de trabajo en Relay, añade un paso y busca **Olostep**. Selecciona **Olostep** de la lista de aplicaciones.

    ![Placeholder: Buscar Olostep en el selector de aplicaciones de Relay](/images/integrations/relay/step-1.png)
  </Step>

  <Step title="Elige una acción">
    Después de seleccionar **Olostep**, elige una de las acciones disponibles de Olostep en la configuración del paso.

    ![Placeholder: Selector de acciones de Olostep en Relay](/images/integrations/relay/step-2.png)
  </Step>

  <Step title="Conecta tu cuenta">
    Haz clic en **Conectar cuenta**, pega tu clave API de Olostep y autoriza la conexión. Relay guardará esta cuenta para futuros flujos de trabajo.

    ![Placeholder: Modal de conexión de cuenta de Relay con campo de clave API de Olostep](/images/integrations/relay/step-3.png)

    Obtén tu clave desde el [tablero de Olostep →](https://olostep.com/dashboard)
  </Step>

  <Step title="Configura Scrape Website (URL + formato)">
    - **URL para extraer:** mapea desde un paso anterior o ingresa manualmente
    - **Formato de salida:** elige `Markdown`, `HTML`, `JSON` o `Texto`

    ![Placeholder: Campos de Scrape Website de Olostep en Relay con URL y formato de salida](/images/integrations/relay/step-4.png)
  </Step>

  <Step title="Prueba y publica">
    Ejecuta una prueba para verificar la salida, luego publica o activa el flujo de trabajo una vez que los resultados sean correctos.

    ![Placeholder: Salida de prueba de Relay y flujo de publicación](/images/integrations/relay/step-5.png)
  </Step>
</Steps>

## Acciones

<CardGroup cols={2}>
  <Card title="Scrape Website" icon="file-lines">
    Extrae contenido de cualquier URL como Markdown, HTML, JSON o texto plano. Maneja páginas renderizadas con JS con tiempos de espera opcionales y orientación por país.
  </Card>

  <Card title="Create Map" icon="map">
    Obtén cada URL en un sitio sin extraer contenido. Úsalo para descubrimiento antes de un trabajo por lotes. Devuelve un `map_id`.
  </Card>
</CardGroup>

<Note>
  **Map es asíncrono.** Almacena el `map_id` devuelto y usa un patrón de retraso/sondeo en Relay para recuperar resultados una vez que el procesamiento se complete.
</Note>

## Ejemplo de flujo de trabajo: Scrape programado de página de competidor

**Qué hace:** En un horario, este flujo de trabajo extrae una página de competidor y almacena contenido limpio que puedes reutilizar en pasos posteriores de Relay.

**Nodos utilizados:** Schedule -> Olostep Scrape Website

![Placeholder: Lienzo de Relay mostrando pasos de programación y extracción de Olostep](/images/integrations/relay/workflow-example.png)

---

### Paso 1: Añadir un disparador de programación

Crea un flujo de trabajo y añade un disparador de programación (por ejemplo, todos los días laborables a las 8 AM).

### Paso 2: Añadir Olostep Scrape Website

Añade un paso de **Olostep** y selecciona **Scrape Website**. Configura:
- **URL:** `https://competitor.com/blog`
- **Formato de salida:** Markdown

### Paso 3: Prueba y publica

Ejecuta una prueba para confirmar la salida, luego publica o activa el flujo de trabajo.

### Lo que obtienes

Cada ejecución produce contenido de página limpio listo para análisis o enrutamiento:

> **Resultado de Scrape**
> - URL de origen
> - Contenido recuperado en tu formato seleccionado
> - Salida de ejecución con marca de tiempo en Relay

## Parsers

Añade un ID de parser al campo **Parser** en la acción de Scrape para obtener datos estructurados en lugar de contenido sin procesar:

| Parser | Extrae |
|--------|--------|
| `@olostep/amazon-product` | Título, precio, calificación, reseñas, imágenes, variantes |
| `@olostep/google-search` | Títulos de resultados, URLs, fragmentos |
| `@olostep/google-maps` | Nombre del negocio, dirección, calificación, reseñas |
| `@olostep/extract-emails` | Direcciones de correo electrónico de cualquier página |
| `@olostep/extract-socials` | Enlaces de perfiles sociales (X, GitHub, LinkedIn, etc.) |
| `@olostep/extract-calendars` | Enlaces de Google Calendar e ICS |

Consulta la lista completa en la [tienda de parsers de Olostep →](https://www.olostep.com/store)

## Relay.app vs Zapier

Relay.app es una fuerte alternativa a Zapier para flujos de trabajo que necesitan revisión integrada y orquestación centrada en IA.

**Ventajas de Relay.app:**
- **Humano en el bucle:** pasos nativos de aprobación y revisión en el flujo de trabajo
- **Diseño de flujo de trabajo centrado en IA:** más fácil añadir decisiones y resúmenes de IA como pasos de primera clase
- **Claridad del flujo de trabajo:** constructor visual limpio con contexto de ejecución claro y salidas de pasos

**Cuándo Zapier puede ser mejor:**
- Necesitas cobertura para una aplicación de nicho solo disponible en Zapier
- Tu equipo ya tiene muchos Zaps existentes y herramientas operativas alrededor de Zapier

Si ambas plataformas soportan tu stack, elige Relay cuando la revisión + colaboración con IA sean centrales para tu proceso.

## Solución de problemas

<AccordionGroup>
  <Accordion title="Clave API rechazada">
    Copia la clave directamente desde [olostep.com/dashboard](https://olostep.com/dashboard) sin espacios finales. Reconecta la cuenta de Olostep en Relay si el error persiste.
  </Accordion>

  <Accordion title="El contenido extraído está vacío">
    Aumenta **Esperar antes de extraer** (prueba 2000–5000ms para páginas con mucho JS). Confirma que la URL sea accesible públicamente sin inicio de sesión. Si un dominio falla consistentemente, contacta a [info@olostep.com](mailto:info@olostep.com).
  </Accordion>

  <Accordion title="Límite de tasa alcanzado">
    Añade retrasos entre ejecuciones con mucho scraping y reintenta con retroceso para flujos de trabajo grandes. Verifica el uso actual en el [tablero](https://olostep.com/dashboard).
  </Accordion>

  <Accordion title="El paso de mapa tarda más de lo esperado">
    Los mapas son asíncronos. Almacena primero el `map_id` devuelto, luego recupera los resultados en un paso/ejecución posterior una vez que el procesamiento esté completo.
  </Accordion>
</AccordionGroup>
