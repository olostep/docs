---
title: 'Get Batch Status'
description: 'Retrieve the current status and progress of an AirOps batch'
openapi: GET /v1/batches-airops/{batch_id}
---

## Overview

Returns the current phase and progress metrics for an AirOps batch. Use this for polling if you're not using webhooks.

## Response Fields

<ResponseField name="id" type="string">
  The batch identifier.
</ResponseField>

<ResponseField name="object" type="string">
  Always `"batch"`.
</ResponseField>

<ResponseField name="created_at" type="integer">
  Unix timestamp (milliseconds) when the batch was created.
</ResponseField>

<ResponseField name="total_urls" type="integer">
  Total number of URLs submitted.
</ResponseField>

<ResponseField name="status" type="string">
  Overall workflow status: `pending`, `in_progress`, `completed`, or `failed`.
</ResponseField>

<ResponseField name="batch_mode" type="boolean">
  Whether async LLM batch mode is enabled (`true`) or sync mode (`false`).
</ResponseField>

<ResponseField name="scraping" type="object">
  Full scraping phase status object (see [Webhooks](/internal/airops-batches/webhooks#scraping-status)).
</ResponseField>

<ResponseField name="analyzing" type="object">
  Full analyzing phase status object (see [Webhooks](/internal/airops-batches/webhooks#analyzing-status)).
</ResponseField>

<ResponseField name="enriching" type="object">
  Full enriching phase status object (see [Webhooks](/internal/airops-batches/webhooks#enriching-status)).
</ResponseField>
